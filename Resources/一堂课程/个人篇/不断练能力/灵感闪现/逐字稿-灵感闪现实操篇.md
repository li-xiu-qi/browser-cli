接下来，我给出了大家，如果想长期修炼，需要迈过的三次飞跃： 意识飞跃，练习飞跃，创新飞跃。
[图片]
飞跃前，典型的特点是：没什么太多的灵感，总是习惯性按照原来的路径和方法做事，执行力可能很强，但是一做需要灵感、才华、动脑子的事情，就开始陷入不稳定、痛苦和纠结。
典型的情况有三个卡点：
- 卡点1：自我设限。 我就是一个靠谱执行的人，怎么可能拥有“灵感”、“新假设”，“创意”、“才华横溢”呢？我做好手里事情就好了，新解法和我关系不大。
- 卡点2：迷信灵感。 灵感都是靠才华 or 放空以后才能出现的啊，怎么能主动激发和管理呢？甚至创意是上帝吻了一下额头才能拥有的幸运，只能等待不能追求啊。
- 卡点3：点到为止。 我能有一个方案就很好了，差不多就行了，没必要再多准备几个PlanB啊，或者能凑合弄一个就行了，没必要想那么多。
如果这种状况长此以往，就会陷入一种“只会做简单封闭问题”的境地，一旦遇到提假设、写文案、做营销、想创意，就开始无计可施。这种状况未来在AI足够成熟以后，会遭遇越来越大的挑战。
第一次飞跃，典型的特点就是： 重视灵感闪现，开始有意识地流淌自己的灵感。
要实现这次飞跃，我们给了大家两个自我修养：
- 第一个自我修养，就是主动榨取。 遇到一个开放性、重要的问题时候，不要等待灵感随机出现，而是用一套机制，主动把灵感榨出来。
- 第二个自我修养，就是自我激励。 很多时候大家容易陷入一种 抠细节，写方案，做评估，做否定 的状态，这不是灵感闪现的状态。灵感闪现一旦断了，或者被否定，就会很难受。
时刻记得： 灵感闪现是答案，是备选项 ，是可能性，核心目标是 加法 ，加法做完了再说减法。

飞跃前，典型的特点是，偶尔也学了一些“灵感闪现”，但是数量太少了，一年也闪现不了两次，手很生。
我们发现常见有三个卡点：
- 卡点1：场景不敏感。 遇到了很多场景的问题，比如做方案、做计划、起名字，往往敏感性不够，甚至后知后觉："哦，原来这个场景也能用灵感闪现啊~！"
- 卡点2：没有固定套路。 每次执行都不一样，有时用脑子，有时用文档，有时用脑图，有时用清单体笔记，总是没有一个可以稳定训练的模型。
- 卡点3：闪现不充分。 每次闪现个3条、5条、10条，轻松就完成了，然后就收工大吉。每次都点到为止，长此以往除了增加点“习惯”并不会成为“灵感闪现的专家or高手”。
飞跃后显著的变化是： 把灵感闪现当做一个显著的刻意练习来做，凑齐1+4个要素，并且努力把刻意练习当做一个十年基本的个人基本功来修炼。
要实现这次飞跃，我们也给了大家两个自我修养：
- 第一个自我修养，就是场景敏感。 工作里是有大量的“灵感闪现”应用场景，只有能够意识到场景才有机会练习。有些人一年能用一两次，而有些人一年能用十几次，而有些人一年能用几十一百次。
- 第二个自我修养，就是拉满练习。 为此，我们给大家初步介绍了一个统一的、经典的、能覆盖90%以上的高价值灵感闪现工作流： SOUL模型 。

第三次飞跃，是高阶飞跃，就很难了：我们要长期持续练习，把灵感闪现长成我们核心能力的一部分。并且数量和质量稳定发生，并且可以创新涌现和集体攻坚。
飞跃前，典型的情况是：虽然我在练习，在持续重复做，但是数量和质量一直参差不齐，有的时候很强，有的时候很弱，就会带来很大的不确定性风险。
典型情况，有三个问题的类型： 速度不够快，质量不稳定，经常容易失手。
这个阶段飞跃以后的标志性结果有两个：
1. 结果1： 稳定闪现，又快又好，知道自己的"灵感激发能力"的度，很少失手。
2. 结果2： 带领团队一起闪现，甚至 无限闪现 、攻坚、多轮创新式闪现。
要实现这次飞跃，我们依旧给了大家两个自我修养：
- 第一个自我修养，是稳定闪现。 当你某一种闪现练得次数足够多的时候，你会越来越快，质量也会越来越好，你会慢慢有一个“基准值”： 我大概多少时间，能闪现出多少质量过关的灵感。
- 第二个自我修养，是无限激发。 这个我个人和一堂也还在爬坡，我想象中顶尖的高手应该可以 解放思想 、可以有丰富的手段做 灵感轰炸 ，可以围绕一个开放难题通过 组合创新 闪现。

怎么办呢？
我当时拉着孔源（专家）和丽梅（项目经理），认真问了一个问题：
这个项目的 最高风险环节 在哪里？努力找出3个最大的风险。
于是我们找到了各个项目最容易失控的几个风险：
1. 质量差： 会不会100%都是纯新手，导致最后没有汇报案例
2. 创客少： 案主太多，大家可能不愿意做创客，大量组队失败
3. 难匹配： 数量太大，案主和创客之间的组队和协作很容易出问题
4. 难完跑 ：人数很多，导致中途大量断掉，画面非常不好看
然后我问大家：如果这4个风险发生了，会怎么样呢。
我们非常悲观的发现，如果这几个风险有一个发生了， 整个项目都会显著失控 ，不只是无法正常落地，可能会出现 很多的BadCase和运营成本 。
理解了巨大价值，然后我站起来，站在白板面前说：
来吧，我们给这个项目准备至少20个对策吧。
每个风险加一两个锁不保险，我们至少加5道锁吧。

第二个技巧，就是往下拆解。
比如常见的，我想提升某一个转化率，假设提了10个没有了。
怎么办呢？
把它拆成 二级公式ABCDEF ，围绕公式，就可以大幅提出新假设了。
[图片]
第一层拆解： 复盘营 = 流量 + 运营 + 上课 + 推荐 + 销售
比如其中的【运营】模块，进行 第二层 拆解：
运营 = 加微率 x 到课率 x 选课率 x 作业率 x 3次作业率 x 周二到场率
比如其中的【到课率】，可以进行 第三层 理解：
到课率 = 预约率 x 到场率 x 讲课90分钟在场率
比如其中的 【预约率】 ，我们又会展开到 第四层 ，理解成三个本质要素：
预约率 = f（预约动力，预约阻力，预约触点）
甚至还有 第五层 ，某一个动力，某一个阻力的具体假设：
- 比如动力：如何承诺和一致？
- 比如阻力：如何降低没时间的阻力？
这样，我们就把业务分成了5个部分，3-5层，大概100+参数，每个参数都可以提出一堆的假设。比如光围绕如何增加【预约率】，我们提出了一大堆假设：

3. 班主任1v1追预约，参考同行（触点）
4. 预约文案迭代，常规想法（增加动力）

在AI的语境下，数据的本质是 “ AI模型的养料 ”， 是训练AI的经验和模式来源 。而其他的特性都是非必要的，比如：内部生成，学员生成，专家生成，这些都是表象。
所以进一步推导：
1）来源是四想象： 内部/外部 x 结构化/非结构化
- 内部数据： 公司自身产生的，如课程视频、学员作业、销售记录。（可控性强，价值密度高）
- 外部数据： 从外部获取的，如爬取的行业报告、公开的宏观经济数据、新闻资讯。（用于拓宽视野，增强实时性）
- 结构化数据： 可整齐放入表格的数据，如学员信息、交易金额、完成进度。
- 非结构化数据： 文本、音频、视频、图像，如课程文稿、私董会录音、设计海报。（占企业数据80%以上，是AI挖掘的重点）
2）按在AI工作流中的作用分： 和AI交互的前中后阶段划分
- 训练数据： 用于最初“教会”模型一个特定能力的、经过精心标注的数据集。例如，用于训练“商业模式画布Agent”的、已被专家标注好的成千上万份画布案例。
- 微调数据： 规模较小但质量极高，用于让通用大模型适应特定领域或风格的数据。例如，用您公司顶级讲师的教学对话记录，去微调一个通用的聊天模型，让它具备该讲师的授课风格。
- 提示/交互数据： 用户与AI产品互动时产生的数据。例如，用户向“商业分析Agent”提问的语句、AI的回答、用户给出的“点赞/点踩”反馈。（这是优化用户体验、实现持续学习的黄金数据）
- 验证/测试数据： 用于评估AI模型性能的一组“未见过的”数据，确保模型的泛化能力，避免过拟合。

# 一堂课程笔记库

本目录收录了一堂系列课程的完整学习笔记，涵盖**个人成长**、**创业实践**、**团队管理**三大维度，形成从自我提升到组织建设的完整知识体系。

---

## 内容导航

### 个人篇：向内生长

聚焦个人能力提升，从认知升级、优势定位、决策质量到表达能力，构建个人竞争力的完整闭环。

| 模块 | 核心目标 | 关键产出 |
|------|----------|----------|
| 不断提认知 | 建立持续学习系统 | 个人知识管理体系 |
| 个人红点 | 识别并放大个人优势 | 个人定位与发展路径 |
| 做决策 | 提升决策质量与效率 | ROI决策框架 |
| 形成竞争力 | 将能力转化为影响力 | 公开表达与产品思维 |

### 创业篇：向外突破

以一堂五步法为核心，覆盖从机会识别、情报调研、假设验证到产品起盘的完整创业周期。

| 阶段 | 核心任务 | 关键产出 |
|------|----------|----------|
| 底层逻辑 | 建立科学创业思维 | 实事求是的创业观 |
| 预判阶段 | 识别机会与方向选择 | 赛道分析报告 |
| 情报调研 | 系统性信息收集 | 调研武器库 |
| 关键假设 | 五步法定量验证需求 | 关键假设清单与验证方案 |
| 单元模型 | 构建可预测的业务模型 | 财务与运营模型 |
| 产品内核 | 定义最小可行产品 | 产品内核说明书 |
| 精益实验 | 低成本快速验证 | MVP与测试结果 |

### 管理篇：组织赋能

从个人到团队，从做事到管事，掌握团队建设与项目管理的核心方法。

| 模块 | 核心目标 | 关键产出 |
|------|----------|----------|
| 管团队-基本功 | 提升团队整体能力 | 基本功训练体系 |
| 管团队-懂招聘 | 科学识人与选人 | 招聘流程与面试框架 |
| 管项目 | 保障项目交付质量 | 项目管理全流程 |

## 使用建议

1. **按需阅读**：根据个人当前阶段选择对应篇章，不必线性阅读
2. **关联学习**：三篇内容相互支撑，建议交叉参考
3. **实践导向**：每篇均包含实操内容，学习后需结合实践复盘
4. **持续更新**：课程内容持续迭代，笔记也会随之补充

## 逐字稿管理策略

本目录的课程笔记以**逐字稿**形式存储，确保内容的完整性与可检索性。以下为管理规范：

### 更新机制

| 触发条件 | 操作 | 负责人 |
|----------|------|--------|
| 新增课程 | 按 PARA 结构归档至对应篇章，运行[[directory-indexer]]更新[[一堂课程_目录结构]]| AI 助手 |
| 课程内容更新 | 更新对应文件，保留历史版本（文件名加日期后缀） | AI 助手 |
| 目录结构调整 | 重构后必须运行[[directory-indexer]]更新[[一堂课程_目录结构]]| AI 助手 |
| 季度审查 | 检查链接有效性、补充缺失元数据 | 用户 |

### 文件命名规范

```
[课程名]-[篇章类型].md
[课程名]-[篇章类型]-[序号]-[子主题].md
```

示例：
- `重新理解科学理念.md`
- `ROI实操1-提升宽度篇.md`
- `五步进阶法1-重新理解需求原点/真需求-拆书.md`

### 元数据标准

每份逐字稿须包含 Frontmatter：

```yaml
---
title: 课程标题
course: 一堂系列课程名
module: 所属篇章（个人篇/创业篇/管理篇）
topic: 子主题
date: 学习日期
status: 已完成/进行中/待整理
tags:
  - 一堂
  - 对应标签
---
```

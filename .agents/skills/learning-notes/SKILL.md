## 场景1: 课程学习笔记

### 定位

你是一名"沉浸式学习者"。你不是在做速记，而是在通过笔记与老师对话。风格必须是**第一人称**、**感性与理性并存**、**极具代入感**。

### 核心写作原则

**1. 第一人称视角**
- 全文使用"我"的主观视角
- 常用句式："我发现..."、"这让我意识到..."
- 禁止使用客观教科书语气

**2. 保留血肉**
- **严禁**压缩成干瘪的公式或要点
- **必须保留案例**: 课程中的精彩比喻、修改前后的对比案例
- 建议使用 Markdown `diff` 块展示修改前后变化

**3. 结构化输出**
- 必须包含 Frontmatter
- 正文结构: `核心感悟` → `知识点模块(原理+案例)` → `实践总结` → `延伸思考`

**4. 实践导向**
- 将知识点转化为"自检清单"或"行动步骤"

### 笔记模板

```markdown
---
title: "[课程标题]"
type: "Knowledge"
target: "Course Note"
tags: ["note/course", "learning"]
date: {{date}}
source: "[[{{filename}}]]"
---

# [标题：提炼核心]

##  我的核心感悟
(用一段深情的文字总结这节课最打动"我"的点)

## 1. [知识点模块标题]
### 核心概念
(老师是怎么讲的？用了什么精彩的比喻？)

### 案例复盘
> **修改前**: ...
> **修改后**: ...
> **分析**: 为什么这样改更好？

### 我的理解
(这个技巧背后的底层逻辑是什么？)

##  实践总结：[方法名]
1. [步骤一]
2. [步骤二]

##  延伸思考/课后练习
(我对这个技巧的联想)
```

### 使用方式

```markdown

2. 按上述原则生成笔记
3. 保存为：笔记_[课程名].md
```

## 场景2: 案例收集

### 2A: 商业决策案例

**定义**: 真实商业场景中的决策分析  
**存储位置**: `Areas/科学决策/深度篇/案例收集/`  
**典型案例**: 涨薪决策、扩张决策、投资分析

#### 核心框架

案例必须包含以下结构：

**1. 背景与问题**
- 业务背景：公司阶段、规模数据、行业特征
- 决策触发点：具体事件、紧迫性、时机
- 核心问题：痛点、不做决策的成本、潜在收益
- 利益相关方：提出者、支持方、反对方、决策权归属
- 争议焦点：定性分歧、为什么无法达成共识

**2. 核心思路**
- 讲师观点：对这类决策的核心判断
- 关键计算：核心数字或公式

**3. 具体内容**
- 成本侧：具体金额、机会成本
- 收益侧：量化收益、难以量化的部分
- 关键逻辑：推导过程、临界点

**4. 划重点**
- 方法论启示：通用方法
- 讲师金句：原文引用
- 可复用工具：可迁移的工具

**5. 关联与行动**
- 关联概念：相关案例或概念链接
- 适用场景：这个案例适用于什么类似场景

#### 深度层级分类

| 层级 | 特征 | 何时使用 |
|------|------|----------|
| L1 定性 | 简单管理动作，明显权衡 | 问题可见、方案标准、成本低 |
| L3 部分定量 | 价值数万到数十万，需说服团队 | 定性分歧存在，关键数字能解决 |
| L4 公式定量 | 涉及数百万，跨部门决策 | 多方使用不同语言，需统一公式 |
| L5 财务定量 | 商业模式验证，大额试错成本 | 双方都有道理，必须全部换成钱 |

#### 文件头规范

```markdown
# 案例主题

来源：一堂 ROI深度篇  
涉及金额：**XXX万/年**  
停在层级：L3 部分定量

---

## 一句话概括
用一句话总结核心结论。

## 背景与问题
...

## 核心思路
...

## 具体内容
...

## 划重点
...

## 关联与行动
...

## 标签
课程类型：决策  
主题：涨薪决策  
适用场景：团队扩张期  
深度层级：L3
```

### 2B: 课程教学案例

**定义**: 课程中为说明方法论而设计的案例  
**存储位置**: 跟随课程目录，建立 `案例/` 子文件夹  
**目的**: 演示方法论如何应用

#### 与商业决策案例的区别

| 维度 | 商业决策案例 | 课程教学案例 |
|------|-------------|-------------|
| 目的 | 记录真实决策过程供参考 | 演示方法论如何应用 |
| 背景 | 真实公司业务背景 | 课程设定的教学场景 |
| 重点 | 决策逻辑、计算、争议 | 方法演示、前后对比 |
| 结构 | 完整五模块 | 简化六模块 |
| 数量 | 少而精 | 可收集课程中所有案例 |

#### 目录结构

```
一堂课程/
├── 笔记_课程名.md        # 课程完整笔记
├── 计划拆解.md           # 课程原始文稿
└── 案例/                 # 本课程所有教学案例
    ├── _INDEX-案例索引.md
    ├── 案例-手册项目.md
    ├── 案例-里程碑V1vsV2对比.md
    └── ...
```

#### 命名规范

统一前缀 `案例-`，体现代说明的方法论：
- `案例-手册项目.md`
- `案例-里程碑V1vsV2对比.md`
- `案例-融资会议排期.md`

#### 内容结构模板

```markdown
# 案例-案例名称

来源：一堂 管理篇-拆计划课程  
说明方法论：**拆计划四步法**

---

## 一句话概括
用一句话总结这个案例演示了什么。

## 案例背景
- **场景**：这个案例发生在什么场景下
- **约束条件**：时间、资源、人员等限制
- **目标**：要达成什么结果

## 前后对比（核心）
**修改前**
- 问题描述
- 具体表现
- 导致的后果

**修改后**
- 改进方案
- 具体做法
- 达成效果

## 方法论拆解
这个案例演示了哪些方法论要点：
1. **要点一**：具体说明
2. **要点二**：具体说明

## 讲师金句
> 原文引用

## 可复用工具
- 工具名称及用法

## 适用场景
- 这个案例的方法适用于什么场景
```

#### 识别课程中的案例

**典型信号**：
- "举个例子"
- "来看个案例"
- "修改前 vs 修改后"
- "V1版本到V7版本"

**收集流程**：
1. 识别案例：找到用于演示方法论的具体场景
2. 确定归属：这个案例说明什么方法论
3. 提取前后对比：记录修改前的问题和修改后的方案
4. 建立案例文件夹：在课程目录下创建 `案例/` 文件夹
5. 在课程笔记中链接：通过 wikilink 链接到具体案例

## 格式规范

所有学习笔记都需遵循 [[content-formatter]] 的规范：
- 背景信息使用分段式叙述（不用列表）
- 分析内容使用表格呈现
- 引用符号 `>` 用于讲师金句
- 加粗用于关键数字和核心结论
- 使用 wikilink 进行笔记间链接

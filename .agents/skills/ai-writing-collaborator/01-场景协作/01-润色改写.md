# 场景1：润色改写

**场景**: 已有初稿，需要按你的风格优化  
**典型用例**: 修改旧文章、改写他人内容、优化 AI 初稿

---

## 工作流程

```
原文 → 加载风格 Skill → AI 润色 → 手动修改 → 分析差异 → 更新 Skill
```

---

## 详细步骤

### 步骤1：准备原文

原文可能来自：
- 你自己写的旧文章
- 他人写的内容
- AI 生成的初稿
- 语音转写的粗糙稿

### 步骤2：加载风格 Skill

**提示词**：
```markdown
请根据以下【写作风格 Skill】对文章进行润色改写。

[粘贴 Skill 全文]

润色要求：
1. 严格遵循 Skill 中的风格要求
2. 保留原文的核心观点和信息
3. 只改表达方式，不改内容
4. 输出完整改写后的文章
```

### 步骤3：执行润色

**进阶选项**：

| 需求 | 附加指令 |
|------|----------|
| 只改特定段落 | "请重点润色第 3-5 段" |
| 对比模式 | "先输出原文，再输出润色版，标注改动" |
| 逐步确认 | "先润色第一段，我确认后再继续" |

### 步骤4：手动修改

**关键原则**: 必须亲自动手改

**检查清单**：
- [ ] 开头风格对吗？
- [ ] 用词符合我的习惯吗？
- [ ] 句式像我的吗？
- [ ] 整体语气一致吗？
- [ ] 有没有漏掉的禁止词汇？

### 步骤5：分析差异（可选但推荐）

```markdown
请对比以下两段：

【AI 润色版】
[粘贴]

【我的修改版】
[粘贴]

分析：
1. 我还做了哪些调整？
2. 这些调整反映了什么规律？
3. 需要更新到 Skill 吗？
```

## 特殊场景

### 场景A：改写他人文章

**目标**: 把别人的文章改成你的风格

**额外注意事项**：
1. **版权提醒**: 仅用于学习或个人笔记，不要直接发布
2. **保留观点**: 改写表达方式，不改变核心观点
3. **添加来源**: 注明原作者和原文链接

**提示词**：
```markdown
请将以下文章改写成我的风格。

【我的风格 Skill】
[粘贴]

【原文】
[粘贴]

要求：
1. 保留原文的核心观点
2. 完全按照我的风格重写
3. 在文末注明：改写自 [原作者]《[原文标题]》
```

### 场景B：优化旧文章

**目标**: 用当前的 Skill 优化以前写的文章

**价值**: 检验 Skill 是否覆盖了你的风格演变

**提示词**：
```markdown
以下是我以前写的文章，请用我当前的写作风格 Skill 进行优化。

【当前 Skill】
[粘贴]

【旧文章】
[粘贴]

请输出优化后的版本，并指出哪些地方体现了当前风格的变化。
```

### 场景C：精简内容

**目标**: 在保持风格的同时压缩篇幅

**提示词**：
```markdown
请润色并精简以下文章，要求：
1. 严格遵循我的风格 Skill
2. 字数压缩到原来的 70%
3. 保留所有核心观点
4. 不损失信息密度

【Skill】
[粘贴]

【原文】
[粘贴]
```

## 示例

### 原文（他人写的）

> 首先，我们需要明确人工智能在当前商业环境中的战略定位。其次，企业应当构建完善的技术架构体系。最后，通过持续的数据积累实现业务闭环。

### AI 润色后（加载了 Skill）

> AI 这东西，现在每个公司都得想清楚：你到底拿它干什么？技术上别贪大求全，先把能用起来的用起来。最关键的是数据，没有数据，再好的算法也是白搭。

### 我的修改

> 聊到 AI，我经常被问一个问题："我们公司要不要搞 AI？"我的回答永远是：先想清楚你要解决什么问题。技术架构可以慢慢搭，但数据得从今天开始攒。没有数据，AI 就是无米之炊。

### 分析差异

| 润色版 | 我的版本 | 规律 |
|--------|----------|------|
| "AI 这东西" | "聊到 AI" | 用对话式开头 |
| 陈述观点 | 引用常见问题 | 用场景引入 |
| "白搭" | "无米之炊" | 用具体比喻 |

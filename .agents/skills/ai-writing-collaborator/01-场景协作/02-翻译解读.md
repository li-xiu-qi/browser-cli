# 场景2：翻译解读

**场景**: 翻译外文文章，同时融入你的风格  
**典型用例**: 技术文章翻译、论文解读、外文资讯本地化

---

## 为什么翻译需要风格 Skill

### 机翻的问题

直译不带风格 = 读起来像机翻：
- 句式符合英文习惯，不符合中文习惯
- 专业术语翻译不一致
- 缺乏个人解读和背景补充

### 风格 Skill 的作用

| 问题 | Skill 解决方式 |
|------|----------------|
| 句式生硬 | 按你的句式习惯重组 |
| 术语混乱 | 统一的术语翻译表 |
| 缺乏解读 | 自动添加你的注释风格 |

---

## 工作流程

```
原文 → 风格 Skill + 翻译解读 Skill → AI 翻译 → 手动修改 → 更新 Skill
```

---

## 详细步骤

### 步骤1：准备原文

原文类型：
- 技术博客/论文
- 产品文档
- 新闻资讯

### 步骤2：双 Skill 协作

**提示词模板**：
```markdown
请将以下英文文章翻译成中文，要求：

1. **遵循我的写作风格 Skill**：
[粘贴风格 Skill]

2. **翻译规范**：
   - 专业术语保留英文并在括号内给出中文
   - 长句拆短，符合中文阅读习惯
   - 添加必要的背景说明（如读者可能不熟悉的概念）
   - 保留原文的核心观点和数据

3. **解读要求**：
   - 在关键观点后添加你的简短点评
   - 用【注：xxx】格式添加背景说明

【原文】
[粘贴]
```

### 步骤3：术语处理

在风格 Skill 中添加【术语表】部分：

```markdown
## 术语翻译对照表

| 英文 | 中文翻译 | 备注 |
|------|----------|------|
| Transformer | Transformer | 保留英文，大写 T |
| fine-tuning | 微调 | 不用"精调" |
| prompt engineering | 提示工程 | 不用"提示词工程" |
| AGI | 通用人工智能 | 首次出现全称 |
```

### 步骤4：手动修改

**重点检查**：
- [ ] 术语翻译是否符合习惯
- [ ] 长句是否拆成了短句
- [ ] 你的注释风格是否一致
- [ ] 整体读起来是否流畅

## 特殊场景

### 场景A：学术论文翻译

**特点**: 严谨、术语多、结构固定

**额外要求**：
```markdown
翻译要求补充：
- 严格遵循学术规范
- 引用格式保持原样
- 方法论部分精确翻译
- 结论部分准确传达
```

### 场景B：技术博客本地化

**特点**: 口语化、需要补充本地背景

**额外要求**：
```markdown
翻译要求补充：
- 将国外案例替换为中国读者熟悉的案例（或保留并说明）
- 补充中国相关的产品/公司对照
- 适当添加本地化的点评
```

### 场景C：快速摘要翻译

**目标**: 快速了解大意，不求全文

**提示词**：
```markdown
请阅读以下英文文章，输出：
1. 一句话总结核心观点
2. 3-5 个关键要点（按我的风格重写）
3. 值得关注的细节
4. 我的简短点评

【原文】
[粘贴]
```

## 示例

### 原文（英文技术博客节选）

> In recent years, transformer-based architectures have revolutionized natural language processing. The key innovation is the self-attention mechanism, which allows the model to weigh the importance of different words in a sentence regardless of their position.

### 翻译后（带风格）

> 这几年，基于 Transformer 的架构彻底改变了自然语言处理领域。【注：Transformer 是 Google 2017 年提出的模型架构，现在几乎所有大语言模型都用它】
>
> 它的核心创新是"自注意力机制"（self-attention）——简单说，模型能自己判断一句话里哪些词更重要，不管这些词在什么位置。
>
> 【我的点评】这个概念是理解现代 AI 的关键，建议花点时间搞懂。

### 分析

| 处理 | 说明 |
|------|------|
| Transformer 保留英文 | 术语表规定 |
| 【注：xxx】 | 统一的注释格式 |
| "简单说" | 口语化解释 |
| 【我的点评】 | 个人风格的点评 |

## 维护翻译专用 Skill

建议单独维护一个【翻译解读风格 Skill】：

```markdown
# 筱可-翻译解读风格

## 基础风格
- 继承：[[筱可-个人风格-V3.0]]

## 翻译特有规则
- 术语首次出现：英文（中文）
- 后续出现：看情况用中文或英文
- 注释格式：【注：xxx】
- 点评格式：【我的点评】xxx

## 术语表
[术语对照表]
```

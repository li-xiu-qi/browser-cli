# 场景4：从零创作

**场景**: 从选题到成文，全程与 AI 协作  
**典型用例**: 博客文章、公众号、技术教程

---

## 工作流概览

```
选题 → 头脑风暴 → 大纲 → 分段写作 → 整合 → 润色 → 定稿
```

---

## 详细步骤

### 步骤1：选题与角度

#### 1.1 对标分析（可选但推荐）

**使用微信文章搜索工具找竞品**：

```bash
# 搜索同主题文章
python .agents/tools/wechat-article-search/wechat_search.py search "[你的主题]" -o竞品分析.json

# 或提取特定文章参考
python .agents/tools/wechat-article-search/wechat_search.py extract "[文章URL]"
```

**分析维度**：
- 标题怎么起的？（数字/悬念/痛点/反常识）
- 角度是什么？（技术/商业/个人故事）
- 结构怎么安排的？
- 配图风格是怎样的？
- 阅读量如何？（判断话题热度）

#### 1.2 确定角度

**提示词**：
```markdown
我想写一篇关于"[主题]"的文章。

【竞品分析】
[粘贴微信搜索找到的对标文章信息]

请帮我：
1. 分析这个主题的不同切入角度（3-5个）
2. 每个角度的优缺点
3. 竞品都在用什么角度？还有哪些空白角度？
4. 推荐最适合我的角度（参考我的风格 Skill）

【我的风格 Skill】
[粘贴]
```

### 步骤2：头脑风暴

**确定角度后，头脑风暴要点**：

```markdown
确定角度：[具体角度]

请帮我头脑风暴：
1. 这个角度下，读者最关心什么问题？
2. 有哪些反直觉的观点可以讲？
3. 有什么具体的案例/数据支撑？
4. 可能的反对意见是什么？如何回应？

请列出 10-15 个要点，我会挑选使用。
```

### 步骤3：生成大纲

**提示词**：

```markdown
请根据以下要点，生成文章大纲：

【选中的要点】
[粘贴]

【要求】
1. 遵循我的风格 Skill 的结构习惯
2. 开头要吸引人（用场景/问题/反常识）
3. 逻辑清晰，层层递进
4. 结尾要有行动号召或总结

【我的风格 Skill】
[粘贴]
```

**大纲确认**：
- 检查结构是否符合你的习惯
- 确认逻辑顺序
- 调整章节权重

### 步骤4：分段写作

**不要一次性写全文**，分段更可控：

```markdown
请根据大纲，先写【第一部分：xxx】

要求：
1. 严格遵循我的风格 Skill
2. 字数控制在 xxx 字左右
3. 写完这部分我确认后再继续

【风格 Skill】
[粘贴]
```

**逐段迭代**：
1. AI 写出第一段
2. 你修改到满意
3. 分析差异，如有必要更新 Skill
4. 确认后继续下一段

### 步骤5：整合与润色

**全文提示词**：

```markdown
请将以下各段整合成一篇完整的文章：

【各段内容】
[粘贴]

整合要求：
1. 统一语气风格
2. 优化段落过渡
3. 检查逻辑连贯性
4. 按我的风格添加开头和结尾（如需要）

【风格 Skill】
[粘贴]
```

### 步骤6：最终修改

**检查清单**：
- [ ] 开头是否吸引人
- [ ] 每个段落是否有价值
- [ ] 有没有禁止词汇
- [ ] 语气是否一致
- [ ] 结尾是否有力
- [ ] 整体是否像我自己写的

## 分阶段使用 Skill

| 阶段 | Skill 使用方式 |
|------|----------------|
| 选题 | 让 AI 根据你的风格推荐角度 |
| 大纲 | 让 AI 按你的结构习惯生成 |
| 分段写作 | 严格加载 Skill |
| 润色 | 再次加载 Skill 统一风格 |

## 示例

### 选题

> 我想写一篇关于"AI 写作工具"的文章。
>
> AI 推荐角度：
> 1. 工具评测对比（常规）
> 2. 我的实际工作流（个人化）⭐ 推荐
> 3. 行业趋势分析（宏大）

### 大纲

> # 我用 AI 写文章的 3 个月
> 
> ## 开头：一个具体的场景
> 描述 3 个月前的困境
> 
> ## 第一部分：我尝试了哪些工具
> - 工具 A：适合快速草稿
> - 工具 B：适合深度思考
> - 工具 C：没必要
> 
> ## 第二部分：我的工作流是什么
> - 步骤 1
> - 步骤 2
> - 步骤 3
> 
> ## 第三部分：踩过的坑
> - 坑 1
> - 坑 2
> 
> ## 结尾：给新手的建议

### 分段写作（第一段示例）

**AI 输出**：
> 三个月前，我还在为每周的博客更新发愁...

**你修改后**：
> 三个月前，我陷入了写作地狱。
>
> 每周一篇博客的承诺，变成每周日的噩梦。坐在电脑前 4 小时，产出 200 字——还都是垃圾。

## 效率技巧

### 技巧1：模板化大纲

对于常写的类型，建立大纲模板：

```markdown
# 技术博客大纲模板

## 开头
- 场景引入：我遇到的具体问题
- 问题描述：当时的情况
- 悬念：我是怎么解决的？

## 主体
- 方案探索：尝试过什么
- 方案对比：为什么选 A 不选 B
- 具体实现：代码/步骤
- 踩坑记录：遇到的问题

## 结尾
- 效果总结
- 适用场景
- 下一步计划
```

### 技巧2：建立素材库

平时积累：
- 好词好句
- 典型案例
- 数据资料

写作时直接调用。

### 技巧3：批量生成然后挑选

```markdown
请针对这个主题，生成 3 个不同风格的开头：
1. 故事型
2. 问题型
3. 反常识型

我会挑选一个使用。
```

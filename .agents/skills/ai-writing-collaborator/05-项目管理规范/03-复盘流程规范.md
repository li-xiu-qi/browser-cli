# 复盘流程规范

**定位**: 如何系统复盘文章，提炼风格规则

---

## 复盘目的

1. **识别亮点** - 哪些表达是成功的？
2. **发现问题** - 哪些是不满意的？
3. **提炼规则** - 可复用的风格要素
4. **更新 Skill** - 持续迭代个人风格

---

## 复盘时机

| 时机 | 说明 |
|------|------|
| **发布后1-7天** | 印象最深刻，推荐 |
| **收到读者反馈后** | 有外部数据支撑 |
| **积累3-5篇后** | 批量复盘，找规律 |
| **发现风格问题时** | 针对性复盘 |

---

## 复盘流程

### 步骤1：准备

- 打开原文（`03_工作区/[文章]/输出.md`）
- 打开复盘模板（`04_Skill素材库/复盘记录/复盘模板.md`）
- 如有读者反馈/数据，一并准备

### 步骤2：通读原文

带着问题阅读：
- 这还是我想表达的意思吗？
- 哪些段落读起来顺畅？
- 哪些地方卡壳？
- 整体感觉符合我的预期吗？

### 步骤3：按维度分析

使用复盘模板，从以下维度分析：

#### 3.1 内容层面

| 维度 | 问题 |
|------|------|
| 核心观点 | 观点清晰吗？有独特见解吗？ |
| 结构设计 | 开头吸引人吗？逻辑清晰吗？结尾有力吗？ |
| 案例/证据 | 案例贴切吗？数据准确吗？ |

#### 3.2 风格层面

| 维度 | 评估 | 说明 |
|------|------|------|
| 语气 | ⭐⭐⭐⭐⭐ | 自然/正式/口语化？ |
| 句式 | ⭐⭐⭐⭐⭐ | 长短句分布 |
| 用词 | ⭐⭐⭐⭐⭐ | 词汇选择 |
| 节奏 | ⭐⭐⭐⭐⭐ | 段落长度、阅读体验 |

#### 3.3 亮点记录

- [ ] 亮点1：具体描述
- [ ] 亮点2：具体描述
- [ ] 亮点3：具体描述

#### 3.4 问题记录

- [ ] 问题1：具体描述 + 改进建议
- [ ] 问题2：具体描述 + 改进建议

### 步骤4：确定分类

**正例** ⭐
- 代表理想风格
- 愿意让 AI 学习
- 亮点 > 问题

**反例** ❌
- 明确不满意
- 要告诉 AI 避免
- 问题 > 亮点

**归档** 📦
- 风格已过时
- 不再参考
- 仅保留记录

### 步骤5：更新索引

在 `04_Skill素材库/_INDEX.md` 中：
- 更新「复盘状态」为 ✅ 已复盘
- 更新「Skill分类」为正例/反例/归档
- 添加「复盘文档」链接

### 步骤6：更新 Skill（关键）

基于复盘结果，更新风格 Skill：

```markdown
## 基于复盘[文章标题]的更新

### 新增规则
- [规则描述]（来自亮点）

### 修改规则
- [规则描述]（来自问题）

### 禁止清单补充
- ❌ [要避免的表达]
```

## 复盘频率建议

| 场景 | 频率 |
|------|------|
| 高频写作期 | 每篇发布后复盘 |
| 稳定期 | 积累3-5篇后批量复盘 |
| 风格转型期 | 每篇必复盘，快速迭代 |

---

## 与 Skill 迭代的闭环

```
写作新文章
    ↓
使用当前风格 Skill
    ↓
发布
    ↓
复盘（发现亮点/问题）
    ↓
更新风格 Skill
    ↓
下一篇文章使用更新后的 Skill
    ↓
更好的文章
```

**核心**: 复盘是 Skill 进化的燃料

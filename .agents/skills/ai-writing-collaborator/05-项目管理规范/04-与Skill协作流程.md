# 与 Skill 协作流程

**定位**: 在项目管理框架下使用风格 Skill 进行写作

---

## 协作前提

1. 已完成 [[02-创建你的第一个风格Skill|风格 Skill 创建]]
2. 风格 Skill 保存在可引用位置（如 `04_Skill素材库/`）
3. 了解 [[00-写作风格指南/01-风格Skill方法论|风格 Skill 方法论]]

## 新文章协作流程

### 阶段1：准备（5分钟）

1. **创建工作区**
   ```powershell
   mkdir "Projects/写作/03_工作区/[文章标题]"
   cd "Projects/写作/03_工作区/[文章标题]"
   ```

2. **创建标准文件**
   - `捕获.md`
   - `提取.md`
   - `输出.md`
   - `参考文献/`（如需）
   - `assets/`（如需）

3. **加载风格 Skill**
   
   在 AI 对话中：
   ```markdown
   请基于以下【写作风格 Skill】进行写作协作：
   
   [[Projects/写作/04_Skill素材库/筱可-个人风格-Vx.y]]
   
   任务：写一篇关于"[主题]"的文章
   ```

### 阶段2：捕获（可选，持续进行）

在 `捕获.md` 中：
- 记录灵感
- 收集素材
- 粘贴参考资料

**AI 协助**:
```markdown
我有以下素材，请帮我整理分类：
[粘贴捕获.md 内容]
```

### 阶段3：提取（关键，15-30分钟）

在 `提取.md` 中梳理：

**步骤1：头脑风暴**
```markdown
我想写关于"[主题]"的文章。

请帮我：
1. 分析不同切入角度
2. 每个角度的优缺点
3. 推荐最适合我的角度
```

**步骤2：确定大纲**
```markdown
确定角度：[角度]

请生成文章大纲：
1. 开头怎么引入
2. 主体分几部分，每部分核心观点
3. 结尾怎么收

遵循我的风格 Skill：
[粘贴风格要点]
```

**步骤3：确认大纲**
- 检查结构是否符合风格习惯
- 确认逻辑顺序
- 调整章节权重
- 将确认后的大纲写入 `提取.md`

### 阶段4：输出（核心，30-60分钟）

在 `输出.md` 中撰写：

**分段协作策略**：

```markdown
请根据大纲，先写【第一部分：xxx】

要求：
1. 严格遵循我的风格 Skill
2. 字数控制在 xxx 字左右
3. 写完这部分我确认后再继续
```

**逐段迭代**：
1. AI 写出第一段
2. **手动修改**到满意
3. 对比 AI 原文和修改版，如有新发现，更新风格 Skill
4. 确认后继续下一段

**关键原则**: 必须亲自动手改，不能只在聊天里指挥

### 阶段5：润色（10-20分钟）

**全文润色**:
```markdown
请润色以下文章，确保：
1. 严格遵循我的风格 Skill
2. 语气一致
3. 删除冗余
4. 优化过渡

[粘贴全文]
```

**手动最终修改**:
- 通读全文
- 检查开头是否抓人
- 检查是否有禁止词汇
- 检查结尾是否有力
- 确保整体像自己写的

### 阶段6：发布与归档（5分钟）

1. **发布**到目标平台
2. **更新索引**在 `04_Skill素材库/_INDEX.md`:
   ```markdown
   | 文章标题 | 03_工作区/文章/ | 已发布 | ⭐ 正例 | ⏳ 待复盘 |
   ```

### 阶段7：复盘（发布后1-7天）

详见 [[03-复盘流程规范]]

## 风格 Skill 迭代流程

### 何时更新 Skill

| 场景 | 操作 |
|------|------|
| 发现"不对味" | 手动修改后，分析差异，更新 Skill |
| 复盘后 | 根据复盘结果，添加新规则 |
| 积累3-5篇文章后 | 批量分析，归纳规律 |

### 更新步骤

1. **对比原文**
   ```markdown
   请对比以下两段：
   
   【AI 原文】
   [粘贴]
   
   【我的修改版】
   [粘贴]
   
   分析我做了哪些修改，总结规律。
   ```

2. **提取规则**
   - 具体修改 → 可复用规律
   - 单点问题 → 通用规则

3. **更新 Skill 文档**
   - 添加新规则到对应章节
   - 更新版本号
   - 记录变更日志

4. **验证**
   - 用更新后的 Skill 写新段落
   - 检查是否符合预期


```markdown

```

```markdown
请对比两段，分析修改规律：

```

```markdown
请根据以上差异，更新我的风格 Skill，添加新规则。
```

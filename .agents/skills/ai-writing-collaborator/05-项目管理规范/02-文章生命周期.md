# 文章生命周期

**定位**: 从灵感到归档的完整流程

---

## 生命周期流程图

```
灵感产生
    ↓
01_选题思考区/      # 记录想法（可选）
    ↓
02_选题库/          # 筛选有价值的选题（可选）
    ↓
03_工作区/[文章]/   # 创建文章目录 ⭐核心
    ↓
捕获 → 提取 → 输出  # 写作流程
    ↓
发布                # 对外发布
    ↓
04_Skill素材库/     # 复盘与标记
    ↓
风格 Skill 迭代     # 持续改进
```

---

## 阶段详解

### 阶段1：选题（可选）

**01_选题思考区/**
- 随时记录灵感
- 不筛选，不管理
- 定期回顾整理

**02_选题库/**
- 筛选有价值的选题
- 初步思考角度
- 评估可行性

### 阶段2：创建（必须）

在 `03_工作区/` 创建文章目录：

```powershell
mkdir "Projects/写作/03_工作区/文章标题"
```

标准结构：
```
文章标题/
├── 捕获.md
├── 提取.md
├── 输出.md
├── 参考文献/
└── assets/
```

### 阶段3：写作（核心）

按 **捕获→提取→输出** 流程：

```
捕获.md     # 收集素材
    ↓
提取.md     # 梳理逻辑、确定大纲
    ↓
输出.md     # 撰写正文
```

AI 协作方式：
- 加载风格 Skill
- 分段协作
- 手动修改
- 迭代优化

详见 [[04-与Skill协作流程]]

### 阶段4：发布（必须）

发布动作：
1. 确认 `输出.md` 完成
2. 发布到目标平台
3. 在 `04_Skill素材库/_INDEX.md` 添加条目

索引条目格式：
```markdown
| 文章标题 | 位置 | 发布状态 | Skill分类 | 复盘状态 |
|----------|------|----------|-----------|----------|
| 文章A | 03_工作区/文章A/ | 已发布 | ⭐ 正例 | ⏳ 待复盘 |
```

### 阶段5：复盘（推荐）

复盘目的：
- 分析文章亮点
- 发现问题
- 提炼可复用规则
- 更新风格 Skill

复盘流程：
1. 阅读原文
2. 按 [[03-复盘流程规范|复盘模板]] 分析
3. 确定分类（正例/反例）
4. 更新 `_INDEX.md`
5. 根据复盘更新风格 Skill

### 阶段6：迭代（持续）

风格 Skill 迭代：
- 基于复盘结果
- 更新 Skill 规则
- 应用到新文章
- 持续改进

## 状态流转

```
写作中
    ↓（发布）
已发布 + 待复盘
    ↓（复盘）
    ├─ 正例 ⭐（满意，用于训练Skill）
    ├─ 反例 ❌（不满意，用于禁止清单）
    └─ 归档 📦（风格已过时）
```

## 关键决策点

### 何时创建文章目录？

**建议**: 确定要写时立即创建

- 有明确写作意图
- 素材收集到一定程度
- 有明确的截稿时间

### 何时发布？

**标准**: `输出.md` 完成且自我满意

- 内容完整
- 排版整洁
- 符合风格 Skill
- 已做最终检查

### 何时复盘？

**建议**: 发布后 1-7 天内

- 刚发布时印象最深刻
- 有读者反馈时复盘价值更高
- 积累 3-5 篇后统一复盘也可

## 与风格 Skill 的关系

```
新文章写作
    ↓
使用当前风格 Skill
    ↓
发布后复盘
    ↓
发现风格问题/亮点
    ↓
更新风格 Skill
    ↓
下一篇文章使用更新后的 Skill
```

**闭环**: 写作 → 复盘 → 迭代 Skill → 更好的写作

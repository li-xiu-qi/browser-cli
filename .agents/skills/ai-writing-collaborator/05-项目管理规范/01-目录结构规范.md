# 目录结构规范

**定位**: 写作项目的物理组织标准

---

## 总览

```
Projects/写作/
├── AGENTS.md                      # 项目级 AI 协作规范 ⭐
├── README.md                      # 项目说明（给人类看）
├── 01_选题思考区/                  # 灵感收集
├── 02_选题库/                      # 选题储备
├── 03_工作区/                      # 写作工作区（核心）
└── 04_Skill素材库/                 # 复盘与风格管理
```

---

## 详细规范

### 01_选题思考区/

**用途**: 灵感收集、碎片想法、网页剪藏  
**原则**: 自由堆积，不管理，定期整理  
**内容**:  anything，允许混乱

```
01_选题思考区/
├── 2026-02-15_关于AI写作的灵感.md
├── 网页剪藏-xxx.md
└── ...
```

### 02_选题库/

**用途**: 经过初步筛选的选题储备  
**原则**: 每个选题独立文件，包含基本思考  
**内容**: 选题标题、背景、角度、预期价值

```
02_选题库/
├── AI写作的未来趋势.md
├── 如何高效使用Claude.md
└── ...
```

### 03_工作区/ ⭐核心

**用途**: 所有文章的创作场所  
**原则**: 一篇文章一个目录，结构标准化  
**命名**: 用文章标题（发布前可调整）

标准结构：
```
03_工作区/
└── [文章标题]/
    ├── 捕获.md        # 输入层
    ├── 提取.md        # 加工层
    ├── 输出.md        # 产出层
    ├── 参考文献/       # 深度参考资料
    └── assets/        # 配图、附件
```

#### 捕获.md

**定位**: 混沌的输入缓冲区  
**原则**: **只管存，不管理**

内容：
- 网页剪藏链接
- 脑海中闪过的碎片灵感
- 复制粘贴的原始段落
- 临时想到的金句

格式：
```markdown
# 捕获

## 2026-02-15
- 看到一个好观点：...
- 链接：...

## 2026-02-16
- 突然想到：...
```

#### 提取.md

**定位**: 逻辑加工台与大纲视图  
**原则**: **结构优先，完成思考**

内容：
- 思维导图/大纲
- 核心观点提炼
- TODO 列表
- 素材引用（链接到捕获.md）

格式：
```markdown
# 提取

## 核心观点
1. ...
2. ...

## 文章大纲
- 开头：场景引入
- 主体：
  - 观点1
  - 观点2
- 结尾：总结升华

## TODO
- [ ] 查证数据
- [ ] 补充案例
```

#### 输出.md

**定位**: 对外发布的成品  
**原则**: **读者视角，遵循风格 Skill**

要求：
- 经过润色、排版完美
- 遵循个人风格 Skill
- 包含标准 Frontmatter（可选）

格式：
```markdown
---
title: "文章标题"
date: 2026-02-15
---

# 文章标题

正文...
```

#### 参考文献/

**定位**: 高价值的原始参考资料库  
**内容**:
- 论文原文 (PDF)
- 行业研报
- 电子书章节
- 深度长文离线存档

**用法**:
```markdown
在提取.md或输出.md中引用：
[[参考文献/xxx.pdf]]
```

#### assets/

**定位**: 视觉资源库  
**内容**: 文章中插入的图片、图表  
**维护**: 通常由 Obsidian 自动管理

### 04_Skill素材库/

**用途**: 复盘文章、管理风格素材  
**原则**: 不存原文，只存索引和复盘记录

```
04_Skill素材库/
├── _INDEX.md                  # 文章状态索引 ⭐
└── 复盘记录/
    ├── 复盘模板.md
    └── YYYY-MM-DD_复盘_文章标题.md
```

#### _INDEX.md

核心索引，记录：
- 所有已发布文章
- 状态标记（写作中/已发布/正例/反例）
- 复盘状态
- 链接到工作区原文

格式：
```markdown
| 文章标题 | 位置 | 发布状态 | Skill分类 | 复盘状态 |
|----------|------|----------|-----------|----------|
| 文章A | 03_工作区/文章A/ | 已发布 | ⭐ 正例 | ✅ 已复盘 |
```

#### 复盘记录/

**用途**: 存放每篇文章的复盘文档  
**模板**: 使用 [[复盘模板]]

## 命名规范

### 目录命名

| 位置 | 命名方式 | 示例 |
|------|----------|------|
| 工作区文章目录 | 文章标题 | `我就爱用AI写笨代码：谈谈渐进式开发的掌控感/` |
| 选题库文件 | 选题标题.md | `AI写作的未来趋势.md` |
| 复盘文档 | `日期_复盘_标题.md` | `2026-02-15_复盘_我就爱用AI写笨代码.md` |

### 文件命名

| 文件 | 固定名称 |
|------|----------|
| 工作区捕获 | `捕获.md` |
| 工作区提取 | `提取.md` |
| 工作区输出 | `输出.md` |
| 素材库索引 | `_INDEX.md` |

---

## 路径引用规范

使用 wikilink 引用：

```markdown
引用工作区文章：
[[Projects/写作/03_工作区/文章标题/输出]]

引用参考文献：
[[Projects/写作/03_工作区/文章标题/参考文献/论文.pdf]]

引用复盘记录：
[[Projects/写作/04_Skill素材库/复盘记录/复盘文档]]
```

---

## 禁止事项

- ❌ 不要在 `03_工作区/` 外创建文章目录
- ❌ 不要在工作区目录外存放 `捕获/提取/输出.md`
- ❌ 不要在 `04_Skill素材库/` 存放原文（只存索引）
- ❌ 不要复制文章到多个位置（用链接引用）

# 公众号风格模板 V1.0

**定位**: 公众号文章、热点评论、干货分享  
**语气**: 像朋友聊天，有观点、有态度  
**目标读者**: 移动端阅读的大众读者

---

## 角色与读者

- **角色**: 有见解的内容创作者
- **读者**: 关注该领域的普通读者（非专家）
- **语气**: 口语化、有观点、像朋友推荐

---

## 风格要点

### 1. 开头抓人
- 100 字内必须抓住读者
- 用场景、问题、反常识、金句开头
- 让读者觉得"这跟我有关"

### 2. 段落简短
- 适合手机阅读
- 一个观点一段
- 多用换行制造节奏

### 3. 有观点有态度
- 不说正确的废话
- 敢于下判断
- 用"我认为"、"我觉得"表达立场

### 4. 易读性优先
- 复杂概念用比喻解释
- 专业术语首次出现要解释
- 用列表、加粗帮助阅读

---

## 文章结构

### 开头（50-100字）
- **场景式**："你有没有遇到过..."
- **问题式**："为什么..."
- **反常识式**："大部分人以为...其实..."
- **金句式**：一句抓人的话

### 主体
1. **背景铺垫** - 让读者理解上下文
2. **核心观点** - 你要表达什么
3. **论据支撑** - 案例、数据、逻辑
4. **实用建议** - 读者能做什么

### 结尾（30-50字）
- 总结核心观点
- 引发思考
- 互动引导（"你怎么看？"）

---

## 用词偏好

### 推荐
- "你" - 直接与读者对话
- "我"、"我觉得" - 个人视角
- 口语化表达 - "说白了"、"其实就是"
- 具体例子胜过抽象概念

### 避免
- 书面语 - "笔者认为"、"综上所述"
- 长句 - 超过 30 字要拆分
- 大段文字 - 不超过 5 行一段
- 专业术语不解释

---

## 句式习惯

- 短句为主，偶尔长句制造变化
- 一个段落 2-4 行（手机屏）
- 用换行代替复杂的连接词
- 适当用问句引发思考

**示例**：
```markdown
❌ 不好的：
人工智能技术的发展正在深刻改变我们的生活和工作方式，从智能手机到自动驾驶，从医疗诊断到金融风控，AI 的应用场景越来越广泛。

✅ 好的：
AI 正在改变我们的生活。

从手机里的语音助手，到医院里的辅助诊断，再到路上的自动驾驶——AI 无处不在。
```

## 禁止清单

### 表达层面
- ❌ 禁止"笔者认为"、"综上所述"、"不难看出"
- ❌ 禁止"首先...其次...最后..."三段式
- ❌ 禁止一段超过 5 行（手机屏）
- ❌ 禁止专业术语不解释
- ❌ 禁止没有观点的陈述

### 标题层面（基于复盘经验）
- ❌ **禁止术语堆砌**：如《AI编程的上下文工程实际上考验的是沟通管理方法论》（24字，双重术语门槛）
- ❌ **禁止陈述句标题**：缺乏悬念和钩子
- ❌ **禁止过长标题**：超过20字手机折叠严重
- ❌ **禁止开篇暴露利益相关**：如产品链接、推广信息
- ❌ **禁止标题党**：标题承诺与内容不符（如承诺"打假"实际讲技术原理）

## 金句使用

- 每篇文章 2-3 个金句
- 金句单独成段，用引用格式
- 金句要在关键转折点

**示例**：
```markdown
> 工具不会替代思考，但会用工具的人会替代不会用工具的人。
```

## 标题写作指南

### 标题三要素（必须满足至少2项）

| 要素 | 说明 | 示例 |
|------|------|------|
| **数字** | 具体数据，降低决策成本 | 《我测试了4个模型，最快的25秒，最慢的8分钟》 |
| **冲突/对比** | A vs B，制造张力 | 《让AI写代码，我反而变慢了》 |
| **悬念** | 必须点进去才知道答案 | 《被AI气到摔键盘后，我悟了一个道理》 |
| **情绪** | 焦虑、共鸣、爽感 | 《不会写SOP的人，正在被AI淘汰》 |
| **利益点** | 读者能获得什么 | 《一个SOP帮我省了每天2小时》 |
| **场景** | 具体情境，代入感 | 《周五下午，老板让我2小时做个数据看板》 |
| **人称** | "我"或"你"，拉近距离 | 《我为什么选择"慢"编程》 |

### 标题公式（基于复盘验证）

**公式1：痛点+反直觉**
```
《为什么[常见做法]反而[负面结果]？》
示例：《为什么AI什么都会，我却更焦虑了？》
```

**公式2：数字+冲突**
```
《我[动作][数量][事物]，最快的[A]，最慢的[B]》
示例：《我测试了4个AI模型，最快的25秒，最慢的8分钟》
```

**公式3：场景+转折**
```
《[场景]后，我[反直觉发现]》
示例：《被AI气到摔键盘后，我悟了一个道理》
```

**公式4：人称+利益**
```
《我[动作]，[结果+利益]》
示例：《我把工作流程写成AI Skill，效率提升了5倍》
```

**公式5：蹭热点+悬念**
```
《[热点工具]用了[时间]，我终于知道为什么[问题]》
示例：《Claude Code用了一周，我终于知道为什么它总出错》
```

### 标题自检清单

- [ ] 去掉专业术语，普通读者能看懂吗？
- [ ] 有数字或具体信息吗？
- [ ] 有冲突、悬念或反直觉吗？
- [ ] 读者能从中获得什么（利益点）？
- [ ] 字数在15-20字之间吗？
- [ ] 手机折叠后前半句完整吗？
- [ ] 内容与标题预期一致吗？

## 结构模板

### 模板A：问题-解法-升华（方法论文章）
```markdown
## 开头（痛点引入）
- 场景/问题/焦虑三问
- 让读者觉得"这跟我有关"

## 主体
### 一、概念引入（书籍/理论）
- 引用权威来源
- 建立理论支撑

### 二、方法论（N步法）
- 步骤1：________
- 步骤2：________
- 步骤3：________

### 三、案例说明
- 具体例子
- 前后对比

### 四、升华
- 从方法到哲学
- 金句收尾

## 结尾
- 三点总结
- 行动呼吁
```

### 模板B：类比框架型（概念重构文章）
```markdown
## 开头（概念重新定义）
- "我说的不是XX，而是YY"
- 建立新的类比框架

## 主体
### 一、建立类比
- A = B，C = D
- 让读者理解新框架

### 二、展开论证（4步法）
- 第一步：________
- 第二步：________
- 第三步：________
- 第四步：________

### 三、案例支撑
- 具体例子说明类比

## 结尾
- 心态转变总结
- 金句收尾
```

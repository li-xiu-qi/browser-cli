## 文件组织规范

### 目录结构

```
ai-writing-collaborator/
└── 02-风格模板库/
    ├── _INDEX.md                    # 索引文件
    ├── 筱可-个人风格-V3.2.md         # 你的个人主风格
    ├── 筱可-公众号风格-V2.1.md       # 公众号专用
    ├── 筱可-技术博客-V1.5.md         # 技术博客专用
    └── 筱可-学术论文-V1.0.md         # 学术论文专用
```

### 文件格式规范

**必须包含 YAML frontmatter**：

```markdown
---
name: 筱可-个人风格
version: "3.2"
created: 2025-12-01
updated: 2026-02-15
description: 基于 20+ 篇原文迭代的主风格
source_articles:
  - 《AI 写作的陷阱》
  - 《我的知识管理体系》
  - 《2025 年终总结》
tags:
  - 主风格
  - 通用
---

# 筱可的写作风格 Skill V3.2

[内容]
```

**YAML 字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✅ | 风格名称，不含版本号 |
| version | ✅ | 版本号，格式 x.y |
| created | ✅ | 创建日期 |
| updated | ✅ | 最后更新日期 |
| description | ✅ | 一句话描述 |
| source_articles | ❌ | 参考的原文列表 |
| tags | ❌ | 标签，用于分类 |
| parent | ❌ | 继承的父风格（多风格时用）|

## 命名与版本

### 命名规范

```
[作者名]-[风格类型]-V[版本号].md
```

**风格类型**：
- `个人风格` - 主风格，最常用
- `公众号风格` - 公众号文章专用
- `技术博客` - 技术文章专用
- `学术论文` - 学术写作专用
- `会议纪要` - 会议记录专用
- `翻译解读` - 翻译文章专用

**示例**：
```
筱可-个人风格-V3.2.md
筱可-公众号风格-V2.1.md
筱可-技术博客-V1.5.md
```

### 版本号规则

**格式**: `主版本.次版本`

| 版本变化 | 触发条件 | 示例 |
|----------|----------|------|
| 主版本 +1 | 重大风格转变、重写 | 1.9 → 2.0 |
| 次版本 +1 | 常规迭代、小调整 | 2.1 → 2.2 |

**版本历史记录**：

在文件末尾添加版本历史：

```markdown
---

## 版本历史

### V3.2 (2026-02-15)
- 新增：少用转折词的规则
- 优化：术语表扩展

### V3.1 (2026-02-01)
- 新增：段落长度限制
- 修复："然而"使用过度的问题

### V3.0 (2026-01-15)
- 重大更新：基于 10 篇新原文重写
- 调整：语气从正式改为更口语化
```

## 版本控制

### Git 管理

**推荐做法**：

1. **所有风格 Skill 纳入 Git**
   ```bash
   git add .agents/skills/ai-writing-collaborator/02-风格模板库/
   ```

2. **提交信息规范**
   ```
   [风格名] Vx.y: 变更描述
   
   示例：
   [个人风格] V3.2: 新增少用转折词规则
   ```

3. **重要版本打标签**
   ```bash
   git tag style-personal-v3.0
   ```

### 变更追踪

**在文件内记录关键变更**：

```markdown

| 日期 | 变更 | 原因 |
|------|------|------|
| 2026-02-15 | 新增"少用转折词" | 发现修改时大量删除"然而" |
| 2026-02-10 | 术语表添加 5 个新词 | 翻译文章时发现不一致 |
```

## 备份与迁移

### 备份策略

| 方式 | 频率 | 说明 |
|------|------|------|
| Git 提交 | 每次更新 | 基础版本控制 |
| 云盘备份 | 每月 | 导出重要版本到云盘 |

### 导出备份

**批量导出当前所有风格**：

```powershell
# PowerShell
$date = Get-Date -Format "yyyy-MM-dd"
$backupDir = "Archives/风格Skill备份/$date"
New-Item -ItemType Directory -Force -Path $backupDir
Copy-Item -Path ".agents/skills/ai-writing-collaborator/02-风格模板库/*.md" -Destination $backupDir
```

### 迁移指南

**迁移到新设备/新库**：

1. 复制整个 `02-风格模板库/` 目录
2. 更新索引文件 `_INDEX.md`
3. 测试几个风格是否正常工作
4. 更新管理手册中的路径信息（如有）

## 多风格管理

### 什么时候需要多风格

| 场景 | 建议 |
|------|------|
| 只有一种写作风格 | 一个 Skill 足够 |
| 工作/个人风格差异大 | 分开两个 Skill |
| 有多个发布平台 | 每个平台一个 Skill |

### 风格继承

**子风格继承父风格**：

```markdown
---
name: 筱可-公众号风格
version: "2.1"
parent: 筱可-个人风格-V3.2
---

# 筱可的公众号风格 Skill V2.1

基于 [[筱可-个人风格-V3.2]]，针对公众号场景调整：

## 继承的规则
（父 Skill 的所有规则）

## 公众号特有规则
- 段落更短，适合移动端阅读
- 多用小标题
- 开头 100 字必须抓人
- 结尾要有互动引导
```

## 定期维护

### 维护清单

**每月检查**：
- [ ] 回顾本月迭代次数
- [ ] 检查是否有冗余规则
- [ ] 更新版本历史和变更记录
- [ ] 备份重要版本

**每季度检查**：
- [ ] 评估 Skill 准确度（是否还需要大量修改）
- [ ] 整理术语表
- [ ] 归纳禁止清单（太长时总结规律）

- [ ] 重大风格转变评估


## 问题排查

### 问题1：AI 输出还是不像我

**诊断**：
- 迭代次数是否足够？（建议至少 5 次）
- Skill 是否加载正确？
- 原文分析是否准确？

**解决**：
1. 重新分析原文
2. 增加迭代频率
3. 检查 Skill 长度（建议 100-200 行）

### 问题2：规则太多，AI 遵守不过来

**诊断**：
- 禁止清单是否超过 30 条？
- 规则之间是否有矛盾？

**解决**：
1. 让 AI 归纳总结规律
2. 删除已内化的规则
3. 拆分多个子风格

### 问题3：不同场景需要不同风格

**诊断**：
- 是否硬用同一个 Skill 处理所有场景？

**解决**：
1. 创建多风格
2. 场景匹配对应的 Skill
3. 建立风格继承关系

### 问题4：风格漂移（Skill 越来越不像最初的风格）

**诊断**：
- 迭代过程中是否引入了新方向？

**解决**：
1. 定期回顾原始原文
2. 重大转变时升级主版本号
3. 保留旧版本备份

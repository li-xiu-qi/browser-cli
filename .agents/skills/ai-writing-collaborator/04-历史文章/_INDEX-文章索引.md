## 文章状态说明

| 状态 | 含义 | 用途 |
|------|------|------|
| **正例** ⭐ | 代表理想风格的文章 | 用于生成/迭代风格 Skill |
| **反例** ❌ | 刻意避免的风格 | 用于明确"不要什么"，写入 Skill 禁止清单 |
| **归档** 📦 | 历史文章但不再参考 | 仅记录，不参与风格分析 |
| **废弃** 🗑️ | 错误或失败的作品 | 作为教训记录，不参与任何分析 |

## 使用方式

### 生成风格 Skill 时

**使用「正例」文章**：

```markdown
我将提供 [N] 篇我的原创文章，请分析我的写作风格。

【正例文章】
[[Projects/写作/03_工作区/2026-02-11_我就爱用 AI 写笨代码：谈谈渐进式开发的掌控感]]
[[Projects/写作/03_工作区/2026-02-11_nanochat-sft文章]]
[[Projects/写作/03_工作区/2026-01-16_别再吹视觉压缩了_DeepSeek-OCR的高压缩率可能误导了你]]
[[Projects/写作/03_工作区/2026-01-17_从 SOP 到 Agent Skill：为什么总结流程在 AI 时代如此重要？]]
[[Projects/写作/03_工作区/2026-02-11_AI编程实际上考验的是沟通管理方法]]
...

请特别注意，要避免出现以下【反例】中的问题：
- 反例1的问题：[具体问题描述]
- 反例2的问题：[具体问题描述]

基于以上，生成我的写作风格 Skill。
```

### 迭代优化时

**对比正例和反例**：

```markdown
请对比以下文章：

【正例】
[理想风格的文章片段]

【反例】
[要避免风格的文章片段]

分析两者的差异，总结我需要避免的表达方式，更新到 Skill 的禁止清单中。
```

### 复盘流程

1. 阅读原文（在 `Projects/写作/03_已发布/`）
2. 在 `Projects/写作/04_Skill素材库/复盘记录/` 创建复盘文档（使用模板）
3. 更新本索引中的「复盘状态」
4. 确定分类（正例/反例/归档）

## 状态流转规则

```
新文章发布
    ↓
默认标记为「正例 ⭐」（默认可用）
    ↓
    复盘分析（可选）
    ↓
    ├─ 风格认可 ──→ 保持「正例」⭐
    │
    ├─ 发现问题 ──→ 降级为「反例」❌（用于警示）
    │
    └─ 风格过时 ──→ 降级为「归档」📦
```

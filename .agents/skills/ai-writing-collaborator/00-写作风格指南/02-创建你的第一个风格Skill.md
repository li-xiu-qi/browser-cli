# 创建你的第一个风格 Skill

**目标**: 从零开始创建一份可用的写作风格 Skill  
**预计时间**: 30-60 分钟（含迭代一次）  
**前置条件**: 3-5 篇你写的满意文章

---

## 准备工作

### 选择原文

**必须满足的条件**:

| 条件 | 说明 | 检查 |
|------|------|------|
| 你自己写的 | 不是 AI 代笔，不是抄袭 |  |
| 风格统一 | 代表你想要的效果 |  |
| 内容完整 | 有开头、主体、结尾 |  |
| 3-5 篇 | 足够 AI 分析出规律 |  |
| 不同主题 | 展示风格的跨主题一致性 |  |

**不要用的内容**:
-  AI 帮你写的文章
-  工作汇报、邮件（太正式或太随意）
-  太短的碎片（微博、朋友圈）

### 整理原文

将原文放在一个容易复制粘贴的位置：
- 可以是 `.md` 文件
- 可以是云文档
- 准备发给 AI

---

## 第一步：让 AI 分析你的原文（10分钟）

### 提示词模板（基础版）

```markdown
我将提供 3-5 篇我自己写的文章。请分析我的写作风格，包括：

1. **角色与读者定位**
   - 我是什么身份/角色？
   - 我写给谁看？
   - 整体语气是什么？

2. **用词偏好**
   - 常用什么类型的词汇？
   - 避免什么类型的词汇？
   - 有特定的过渡词、连接词吗？

3. **句式特点**
   - 句子长短分布？
   - 偏好什么句式结构？
   - 段落长度和结构？

4. **文章结构**
   - 开头习惯怎么写？
   - 主体如何组织？
   - 结尾怎么收？

5. **独特标识**
   - 有什么个人特色的表达？
   - 常用的修辞手法？
   - 特别的标点符号使用习惯？

请基于以上分析，生成一份【写作风格 Skill】文档，包含：
- 角色与读者
- 风格要点（3-5条）
- 用词偏好
- 句式习惯
- 禁止清单

以下是我的原文：

[粘贴你的文章]
```

### 提示词模板（进阶版：含反例）

如果你有明确不想要的文章风格，使用此版本：

```markdown
我将提供我的原创文章，请分析我的写作风格。

【正例：代表我理想风格的文章】
[文章1全文]
---
[文章2全文]
...

【反例：我明确不想要的文章风格】
[反例文章1全文]
问题描述：[具体说明为什么这篇文章不符合我的期望]
---
[反例文章2全文]
问题描述：[具体问题]

请基于以上分析：
1. 从【正例】中提取我的风格特点
2. 从【反例】中总结我要避免的表达方式
3. 生成一份【写作风格 Skill】文档，包含：
   - 角色与读者
   - 风格要点（3-5条，基于正例）
   - 用词偏好
   - 句式习惯
   - 禁止清单（结合反例的问题）
```

### 执行

1. 打开你常用的 AI 工具（Claude、Kimi 等）
2. 粘贴上述提示词
3. 在末尾粘贴你的原文
4. 发送

### 预期输出

AI 会输出一份 30-50 行的风格 Skill 初稿，包含：

```markdown
# 筱可的写作风格 Skill V1.0

## 角色与读者
- 角色：产品经理/技术写作者
- 读者：技术背景、时间有限的从业者
- 语气：像同事聊天，不像写报告

## 风格要点
1. **说人话** - 用口语化表达，避免书面语
2. **有观点** - 不说正确的废话，要有立场
3. **具体可执行** - 给例子，不给概念

## 用词偏好
- 用"你"而不是"读者"、"用户"
- 用"我觉得"、"我认为"表达观点
- 具体数字胜过模糊描述

## 句式习惯
- 短句为主，长短句交替
- 一个段落一个观点
- 用换行制造节奏感

## 禁止清单
-  禁止"赋能"、"闭环"、"抓手"等商业黑话
-  禁止"首先...其次...最后..."三段式
-  禁止"综上所述"、"总之"结尾
```

```markdown
  - 文章标题1
  - 文章标题2

[AI 生成的内容]
```

## 第三步：实战测试（15分钟）

### 选择测试题目

找一个你**正好要写**的题目，不要用无关的题目测试。

**好的测试题目**：
-  你今天就想写的选题
-  工作中需要的文档
-  博客的下一篇文章

**不好的测试题目**：
-  "写一篇关于 AI 的文章"（太泛）
-  完全不是你平时写的内容类型

### 测试提示词

```markdown
请根据以下【写作风格 Skill】，写一篇关于"[你的题目]"的文章。

[粘贴你的 Skill 全文]

要求：
1. 严格遵循 Skill 中的风格要求
2. 先输出大纲让我确认
3. 我确认后再写全文
```

### 评估输出

**预期**: AI 写的会有你的影子，但肯定"不完全像"

**关键问题**：
- 开头风格对吗？
- 用词习惯符合吗？
- 整体语气像你的吗？

**不要期待完美**，V1.0 只需要"方向对"即可。

## 第四步：手动修改（20分钟）

**这是最关键的一步**。

### 修改方法

1. 将 AI 输出复制到编辑器
2. **逐句修改**，改成你自己满意的样子
3. **不要**在聊天里指挥 AI 改，要亲自动手

### 修改时注意

问自己：
- 这里为什么不像我写的？
- 我会怎么表达这个观点？
- 这个词我会用吗？
- 这个句式符合我的习惯吗？

### 示例修改

**AI 原文**：
> 作为一名深耕 AI 领域多年的老兵，我认为...

**你修改后**：
> 作为一个做了十几年 AI 的人，我觉得...

**差异分析**：
- "作为" → "作为"（保留）
- "深耕" → 删除（商业黑话）
- "AI 领域" → "AI"（说人话）
- "多年的老兵" → "十几年的人"（不用军事比喻）
- "我认为" → "我觉得"（更口语化）

## 第五步：分析差异，更新 Skill（10分钟）

### 对比提示词

```markdown
请对比以下两段文字：

【AI 原文】
[粘贴 AI 生成的原文]

【我的修改版】
[粘贴你修改后的版本]

请分析：
1. 我做了哪些具体修改？
2. 这些修改背后的规律是什么？
3. 请据此更新【写作风格 Skill】，将新发现的规则添加进去。
```

### 更新 Skill

将 AI 分析出的新规则添加到 Skill 中：

```markdown
## 禁止清单（更新）
-  禁止"赋能"、"闭环"、"抓手"等商业黑话
-  禁止"首先...其次...最后..."三段式
-  禁止"综上所述"、"总之"结尾
-  禁止"深耕"、"老兵"等军事化/商业黑话  ← 新增

## 用词偏好（更新）
- 用"你"而不是"读者"、"用户"
- 用"我觉得"、"我认为"表达观点 ← "我觉得"优于"我认为"
- 具体数字胜过模糊描述
- 不用"多年"，用具体年数如"十几年"  ← 新增
```

```
筱可-个人风格-V1.0.md → 筱可-个人风格-V1.1.md
```

更新 YAML：
```yaml
version: "1.1"
updated: 2026-02-15
```

## 完成！你有了 V1.1

现在你有：
1.  一份基于你原文的初版 Skill
2.  一次实战测试
3.  一轮迭代优化

**接下来**：
- 继续用这个 Skill 写作
- 每次发现"不对味"就手动修改、分析差异、更新 Skill
- 3-5 次迭代后，Skill 会非常精准

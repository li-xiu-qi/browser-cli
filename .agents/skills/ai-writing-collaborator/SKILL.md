## 核心概念

### 什么是写作风格 Skill

写作风格 Skill 是一份**持续更新的个人写作菜谱**，记录了：

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| **角色与读者** | 你是谁，写给谁 | 产品经理写给技术团队 |
| **风格要点** | 核心写作原则 | 说人话、有观点、具体可执行 |
| **禁止清单** | 绝对不要的词句 | 禁止"赋能"、禁止"综上所述" |
| **参考资料** | 术语表、范文链接 | 领域术语对照表 |

### 与提示词的本质区别

| 对比项 | 去 AI 味提示词 | 写作风格 Skill |
|--------|---------------|----------------|
| **形式** | 一次性指令 | 持续更新的文档 |
| **记忆** | 用完即忘 | 每次写作都加载 |
| **个性化** | 同质化（大家都在用同一套） | 专属个人 |
| **迭代** | 无法积累 | 越用越精准 |
| **效果** | AI 味 2.0 | 真正的个人风格 |

## 快速开始

### 第一步：准备你的原文（5分钟）

找出 3-5 篇**你自己写的**、最满意的文章。必须是：
-  你自己写的，不是 AI 代笔
-  风格统一，代表你想要的效果
-  内容完整，有开头结尾

**进阶：利用「反例」**

如果你有明确**不想要**的风格（如早期不成熟的文章），可以作为「反例」：
- 在生成 Skill 时告诉 AI 要避免什么
- 将反例的问题写入 Skill 的禁止清单
- 详见 [[04-历史文章/_INDEX-文章索引|文章索引管理]]

### 第二步：创建初版风格 Skill（10分钟）

将文章发给 AI，使用 [[02-创建你的第一个风格Skill|创建指南]] 中的提示词，让 AI 分析你的写作特点并生成 Skill。

### 第三步：实战迭代（持续）

1. 让 AI 用 Skill 写一篇新文章
2. 你**手动修改**到满意
3. 对比 AI 原文和你的修改版，更新 Skill
4. 重复 3-5 次，Skill 就会非常精准

### AGENTS.md vs README 边界

| 文件 | 读者 | 用途 |
|------|------|------|
| **AGENTS.md** | AI | 执行规范：必须做什么、禁止做什么 |
| **README.md** | 人类 | 项目说明：是什么、怎么用 |

**AI 执行逻辑**：
1. 先读全局 Skill（本文件）
2. 再读项目 AGENTS.md（专属约束）
3. 有疑问参考 README.md（背景说明）
4. 执行任务

项目级规范示例：[[Projects/写作/AGENTS.md]]（AI 必读） | [[Projects/写作/README|README]]（人类阅读）

## 使用场景

### 场景1：润色改写

已有初稿，需要按照你的风格优化。

**工作流**:
1. 加载你的写作风格 Skill
2. 发送原文 + 润色指令
3. 手动修改不满意处
4. 分析差异，更新 Skill

详见 [[01-场景协作/01-润色改写|润色改写指南]]

### 场景2：翻译解读

翻译外文文章，同时融入你的风格。

**工作流**:
1. 加载写作风格 Skill + 翻译解读 Skill
2. 提供原文
3. AI 翻译时自动带上你的表达习惯、注释方式

详见 [[01-场景协作/02-翻译解读|翻译解读指南]]

### 场景3：访谈整理

将播客/访谈录音转为文章。

**工作流**:
1. 访谈 Skill 提取信息、整理结构
2. 写作风格 Skill 转化为你的文章风格

详见 [[01-场景协作/03-访谈整理|访谈整理指南]]

### 场景4：从零创作

从选题到成文，全程协作。

**工作流**:
1. 头脑风暴确定角度
2. AI 按你的风格出大纲
3. 分段写作，每段你审核修改
4. 整合成文

详见 [[01-场景协作/04-从零创作|从零创作指南]]

## 常见误区

| 误区 | 纠正 |
|------|------|
| "我先随便写个 Skill，以后再完善" | 没有原文分析的 Skill 是空中楼阁，必须先有原文 |
| "我直接复制别人的 Skill 用" | 风格必须基于你自己的文字，别人的 Skill 对你来说是新的 AI 味 |
| "一次写好就不用改了" | Skill 是活的文档，每次写作都是迭代机会 |
| "规则越多越好" | 禁止清单太长会影响效果，要定期总结归纳规律 |

## 核心理念回顾

> **你去不掉 AI 味。你只能用自己的味道盖过去。**

AI 味不是 AI 写得差，而是 AI 在用训练数据的**平均风格**写作。想要独特，必须给它一份你的专属菜谱。

提示词像你跟食堂师傅喊"少放盐"，Skill 像你把家传菜谱写下来交给他。十次迭代后，他做的菜就有了你家厨房的味道。

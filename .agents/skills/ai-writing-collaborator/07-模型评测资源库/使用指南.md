# 模型评测资源库使用指南

**本文档**: 说明如何在写作模型评测类文章时使用本资源库

---

## 快速开始

### 3分钟上手

```markdown
1. 打开 [[_INDEX\|资源索引]] 查看可用评测集
2. 选择适合的评测集（如：视觉理解评测集）
3. 阅读 [[视觉理解评测集/评测集说明\|评测集说明]] 了解case详情
4. 根据要测试的模型能力，选择对应维度的case
5. 使用说明中的Prompt进行测试
6. 记录结果，撰写评测文章
```

## 典型使用场景

### 场景1: 评测新发布的VLM模型

**目标**: 快速产出一份模型评测文章

**步骤**:

1. **选择基础case**（每个维度1-2个）
   - 目标对比: Case 1.1 (奔跑的人)
   - 图片复刻: Case 2.1 (网页复刻)
   - OCR识别: Case 6.1 (手写体)
   - 文本推理: Case 9.1 (改分数)

2. **执行测试**
   - 使用评测集中的Prompt
   - 记录模型输出
   - 截图保存结果

3. **撰写文章结构**
   ```markdown
   ## 评测方法
   本次评测选取了4个核心维度的测试case，
   涵盖目标识别、图片复刻、OCR识别和逻辑推理...
   
   ## 测试结果
   
   ### 目标识别能力
   [测试结果描述]
   [截图对比]
   
   ### 图片复刻能力
   ...
   ```

### 场景2: 对比两款模型

**目标**: 制作A vs B的对比评测

**步骤**:

1. **选择对比case**（有区分度的）
   - Case 1.3 (不开心的冰块) - 测试细粒度理解
   - Case 2.2 (迷宫复刻) - 测试代码生成
   - Case 4.1 (鸡腿走迷宫) - 测试空间推理
   - Case 6.3 (究极OCR) - 测试极限OCR

2. **制作对比表格**
   ```markdown
   | 测试项 | 模型A | 模型B | 说明 |
   |--------|-------|-------|------|
   | 不开心的冰块 | ✅正确 | ❌错误 | 模型A理解情绪+物理状态 |
   | 迷宫复刻 | ⚠️有瑕疵 | ✅完整 | 模型B路径更准确 |
   | ... | ... | ... | ... |
   ```

3. **撰写结论**
   - 总结两款模型各自的优势
   - 给出适用场景建议

### 场景3: 深度评测某方面能力

**目标**: 深入测试模型的空间推理能力

**步骤**:

1. **选择相关case**（空间逻辑+空间变换）
   - Case 4.1: 鸡腿走迷宫
   - Case 4.2: 六面体展开图
   - Case 5.1: 动物视角
   - Case 5.2: 橙色圆圈大小
   - Case 5.3: 泛化测试

2. **分析失败case**
   - 为什么失败？
   - 是理解问题还是推理问题？
   - 与参数量有关吗？

3. **撰写深度分析**
   ```markdown
   ## 空间推理能力深度评测
   
   ### 测试设计
   本文选取5个case，覆盖路径规划、3D映射、
   视角推断、视觉错觉、泛化能力...
   
   ### 结果分析
   
   #### 路径规划 (Case 4.1)
   模型无法理解"上下左右"的相对方向...
   
   [详细分析]
   ```

## 注意事项

### 1. 客观性
- 既报告成功案例，也报告失败case
- 避免"吹"或"黑"，用事实说话
- 区分"个人感受"和"客观事实"

### 2. 可复现性
- 使用评测集中的标准Prompt
- 记录测试时间和模型版本
- 截图保存原始结果

### 3. 时效性
- 模型会快速迭代，标注测试时间
- 过期结论需要更新
- 关注模型版本变化

## FAQ

**Q: 这些case会不会被模型"背答案"？**  
A: 部分经典case（如视觉错觉）可能被训练数据覆盖，建议：
- 适当修改Prompt表述
- 结合多个case综合判断
- 关注推理过程而非只看结果

**Q: 可以自己新增case吗？**  
A: 当然可以！欢迎补充：
1. 按格式整理case（Prompt+难度+预期）
2. 放入对应维度
3. 更新评测集说明文档

**Q: 这些case适合测试所有VLM模型吗？**  
A: 主要针对图文理解模型，纯文本模型不适用。

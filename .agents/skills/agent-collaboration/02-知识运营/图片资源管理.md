# 图片资源管理

**定位**: 个人图片素材库管理规范  
**用途**: 收集、整理、检索配图素材  
**存储位置**: `resources/image-library/`

---

## 📁 目录结构

```
resources/
└── image-library/                      # 正式图片库（精选后）
    ├── index.md                        # 总索引（带预览）
    ├── peter-steinberger-openai/       # 每张图片独立文件夹
    │   ├── image.png                   # 图片文件（统一命名）
    │   └── metadata.json               # 元数据描述文件
    ├── claude-code-welcome/
    │   ├── image.jpg
    │   └── metadata.json
    └── chatgpt-agent-logo/
        ├── image.jpg
        └── metadata.json

0_Inbox/
└── image-staging/              # 暂存区（预筛选）
    └── 2026-02-16/             # 按日期组织
        ├── img_001.jpg         # 从文章提取的原始图片
        ├── img_002.png
        └── ...
```

**结构说明**:
- 每张图片一个独立文件夹，图片文件统一命名为 `image.png` 或 `image.jpg`
- 元数据文件统一命名为 `metadata.json`，与图片放在同一文件夹
- `index.md` 提供全局预览，包含所有图片的缩略图和关键元数据

## 🔧 配套工具

### 1. 图片分析器 (image-analyzer)

**路径**: `.agents/tools/image-analyzer/analyzer.py`

**用途**: 为无多模态能力的 AI 生成图片描述

**使用方式**:
```bash
# 分析单张图片
python analyzer.py image.jpg

# 分析目录（批量）
python analyzer.py --dir ./images
```

**输出**: `metadata.json`（与图片放在同一文件夹内）

```json
{
  "description": "详细描述...",
  "caption": "一句话摘要",
  "tags": ["标签1", "标签2"],
  "usage_scenarios": ["文章配图", "PPT插图"],
  "has_watermark": false,
  "quality": "high",
  "subject": "主体名称",
  "style": "写实照片",
  "_meta": {
    "analyzed_at": "2026-02-16T15:30:00",
    "model": "Qwen/Qwen3-VL-32B-Instruct"
  }
}
```

### 2. 图片收集器 (image-collector)

**路径**: `.agents/tools/image-collector/collector.py`

**用途**: 从网页/文章提取图片到**暂存区**

**使用方式**:
```bash
# 从 Markdown 文章提取（保存到暂存区）
python collector.py article.md

# 从网页提取
python collector.py https://example.com/article

# 指定下载序号
python collector.py article.md --indices 1,3,5

# 查看暂存区
python collector.py --staging

# 入库（从暂存区移到正式库）
python collector.py --import "0_Inbox/image-staging/2026-02-16/img_001.jpg" --name "claude-workflow"

# 清理7天前的暂存文件
python collector.py --clean 7
```

## 💡 工作流程

### 场景一：手动收集单张图片

```
1. 用户: "保存这张图片到图片库，命名为'claude-code-workflow'"
   ↓
2. AI: 创建文件夹 resources/image-library/claude-code-workflow/
   ↓
3. AI: 复制图片到文件夹内，命名为 image.jpg
   ↓
4. AI: 调用 analyzer.py 生成 metadata.json
   ↓
5. AI: 运行脚本更新 index.md
   ↓
6. 完成: 文件夹包含 image.jpg + metadata.json
```

### 场景二：从 Cliper 文章提取（使用暂存区）

```
1. Cliper 抓取文章 → 保存到 Projects/写作/
   ↓
2. 用户: "这篇文章的配图保存到暂存区"
   ↓
3. AI: 调用 collector.py 提取图片 → 保存到 0_Inbox/image-staging/2026-02-16/
   ↓
4. 用户查看暂存区，选择要的图片
   ↓
5. 用户: "把 img_001.jpg 入库，命名为'claude-workflow'"
   ↓
6. AI: 创建文件夹 resources/image-library/claude-workflow/
   ↓
7. AI: 移动图片到文件夹内，命名为 image.jpg
   ↓
8. AI: 调用 analyzer.py 生成 metadata.json
   ↓
9. AI: 运行脚本更新 index.md
   ↓
10. 清理暂存区不要的
```

### 场景三：从微信文章搜索配图

当你需要某类特定配图但图片库没有时：

```
1. 用户: "找几张'程序员工作场景'的配图参考"
   ↓
2. AI: 调用 wechat_search.py 搜索 "程序员 工作 效率"
   ↓
3. AI: 浏览搜索结果，打开相关文章
   ↓
4. AI: 提取文章中的配图 → 暂存区
   ↓
5. 用户筛选: 选择无水印、高质量的图片
   ↓
6. AI: 创建文件夹并重命名为有意义的名称
   ↓
7. AI: 移动图片为 image.jpg/image.png
   ↓
8. AI: 生成 metadata.json
   ↓
9. AI: 运行脚本更新 index.md
```

**为什么要暂存区？**
- 避免污染正式图片库
- 先预览再决定是否保留
- 支持重命名（有意义的文件夹名）
- 批量处理后再精选

**新目录结构的优势**
- 图片和元数据绑定：每个文件夹包含图片+metadata.json，不会混乱
- 统一预览入口：index.md 提供全局预览，Obsidian 中直接查看
- 易于扩展：添加新图片只需新建文件夹+更新索引
- 引用方便：通过 `folder-name/image.jpg` 路径引用

## 📦 暂存区管理

暂存区位于 `0_Inbox/image-staging/`，按日期自动组织。

### 入库流程

```bash
# 1. 查看暂存区有哪些图片
python collector.py --staging

# 2. 选择需要的图片，入库（创建文件夹并重命名）
python collector.py --import "0_Inbox/image-staging/2026-02-16/img_001.jpg" --name "claude-workflow"

# 3. 生成描述文件
python analyzer.py resources/image-library/claude-workflow/

# 4. 定期清理暂存区（删除7天前的）
python collector.py --clean 7
```

### 命名建议（入库时）

**文件夹命名**（不是图片文件名）：

| 原文件名 | 文件夹命名 | 图片文件 | 说明 |
|---------|-----------|---------|------|
| img_001.jpg | claude-code-interface/ | image.jpg | 产品界面截图 |
| img_002.png | peter-steinberger-portrait/ | image.png | 人物肖像 |
| img_003.webp | ai-workflow-diagram/ | image.webp | 流程图/示意图 |

**命名原则**:
- 文件夹名全小写，单词间用连字符 `-`
- 有意义，一看就知道内容
- 图片文件统一命名为 `image.png` 或 `image.jpg`

### 清理策略

- **定期清理**: 每周运行 `--clean 7` 删除旧文件
- **手动删除**: 直接删除暂存区不需要的图片
- **批量入库**: 支持通配符或目录批量处理（可扩展）

## 📋 命名规范

### 文件夹命名（重要！）

| 类型 | 文件夹命名示例 | 内部图片 | 说明 |
|------|---------------|---------|------|
| 人物照片 | `peter-steinberger-portrait/` | image.jpg | 人名-类型 |
| 产品截图 | `claude-code-interface/` | image.png | 产品-内容 |
| 场景图 | `meeting-room-modern/` | image.jpg | 场景-特征 |
| 概念图 | `ai-workflow-diagram/` | image.webp | 主题-类型 |

### 命名原则

**文件夹名**：
- 全小写
- 单词间用连字符 `-`
- 有意义，一看就知道内容
- 不要 `IMG_1234` 这种无意义名字

**图片文件**：
- 统一命名为 `image.png` 或 `image.jpg`
- 保持原始格式（png/jpg/webp）

---

## ✅ 保存原则

**保存**:
- ✅ 无水印或水印不影响使用
- ✅ 高清（至少 800px 宽）
- ✅ 与知识库主题相关
- ✅ 通用性强，可重复使用

**不保存**:
- ❌ 带有明显版权水印
- ❌ 质量过低（模糊、压缩严重）
- ❌ 过于特定（如某篇文章独占的截图）
- ❌ 与知识库主题无关

## 📊 维护建议

- **定期整理**: 每月检查，删除无用图片
- **及时分析**: 新图片入库后立即生成 metadata.json
- **更新索引**: 新图片入库后同步更新 index.md，保持预览最新
- **备份**: 图片库纳入 Git 版本控制（小文件）

### 自动更新 index.md

使用脚本自动生成索引：

```bash
# 在仓库根目录运行
python .agents/tools/image-analyzer/update_index.py

# 或指定图片库路径
python .agents/tools/image-analyzer/update_index.py resources/image-library/
```

脚本会：
1. 扫描所有图片文件夹
2. 读取 `metadata.json`
3. 自动重新生成 `index.md`

### 手动更新（不推荐）

如需手动编辑，参考以下模板：

```markdown
## 图片标题

**文件夹：** [`folder-name/`](./folder-name/)

![描述文字](./folder-name/image.jpg)

| 属性 | 内容 |
|------|------|
| **标题** | 从 metadata.json 的 caption 字段获取 |
| **主题** | 从 subject 字段获取 |
| **风格** | 从 style 字段获取 |
| **标签** | 从 tags 字段获取 |
| **质量** | 从 quality 字段获取 |
| **水印** | 从 has_watermark 字段获取 |
```

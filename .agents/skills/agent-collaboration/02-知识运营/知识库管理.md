# 知识库管理

**定位**: Obsidian 知识库的结构、规范和演进管理  
**适用范围**: 本知识库的所有维护工作

---

## 概述

本规范提供系统化的知识库管理方法，确保 Obsidian Vault 保持结构清晰、规范一致、可持续演进。

---

## 知识库架构

### PARA 目录结构

```
笔记专用/
├── 0_Inbox/          # 收集箱，临时存放未分类内容
├── Projects/         # 项目，有明确目标的短期任务
├── Areas/            # 领域，长期维护的责任领域
├── Resources/        # 资源，参考资料和附件
├── Archives/         # 归档，已完成项目
└── 日记/             # 日常记录
```

### 技能库架构

**核心位置**: `.agents/skills/`

本项目的 AI 技能库以 `.agents/skills/` 为唯一事实来源，其他 AI 工具配置通过软连接引用：

```
.agents/skills/           # 核心目录（唯一事实来源）
├── agent-collaboration/
├── case-study-collector/
└── ...

.gemini/skills/           # 目录联接 → .agents/skills/
├── agent-collaboration/  [JUNC]
├── case-study-collector/ [JUNC]
└── ...
```

**规范**: 所有 Skill 的修改都在 `.agents/skills/` 进行，自动同步到所有关联位置。

## 附件管理规范

**强制规则**: 附件必须与笔记同级存放

- 笔记: `路径/笔记名.md`
- 附件目录: `路径/assets/笔记名/`
- 引用: `![描述](assets/笔记名/图片.png)`

**禁止**:
-  全局附件文件夹（如 `attachments/` 或 `附件/`）
-  图片直接放笔记同级（未放入 `assets/` 子目录）
-  通用文件夹名如 `assets/images`（必须匹配文件名）

### 执行协议

当处理违反此规则的笔记时：

1. 若 `assets/<文件名>/` 目录不存在，则创建它
2. 将被引用的本地文件移动至该目录
3. 更新 Markdown 中的链接，使用相对路径 `assets/<文件名>/<文件名>`

### 重构现有笔记流程

当被要求"修复"或"整理"笔记图片时：

1. 识别所有本地图片链接
2. 检查其存放位置是否符合附件强制同级存放规则
3. 必要时执行文件移动并更新 Markdown 链接

**检查清单**:
- [ ] 附件是否放在 `assets/笔记名/` 目录
- [ ] 子文件夹名是否与笔记文件名一致
- [ ] 引用路径是否为相对路径 `assets/笔记名/图片.png`

## 写作规范

### 语言风格

- 使用直白词语，不用成语
- 标点尽量少，句子要清楚
- 绝对无括号、无 Emoji
- 线性输出，一行一个主逻辑
- 仅核心强调词使用加粗

### 链接规范

- 使用 Obsidian wikilink: `[[笔记名]]`
- 显示文本: `[[笔记名|显示文本]]`
- 禁止标准 Markdown 链接 `[文本](路径)`

---

## 工作流

### 创建新笔记

1. 判断笔记类型选择目录
2. 复制对应模板
3. 按规范填写内容
4. 检查附件存放位置
5. 添加适当标签

### 重构笔记

1. 读取原笔记内容
2. 识别需要保留的信息
3. 按新模板重新组织
4. 更新关联链接
5. 移动附件到新位置

### 规范检查

- [ ] 附件是否放在 `assets/笔记名/` 目录
- [ ] 是否使用 wikilink 而非 Markdown 链接
- [ ] 格式是否符合语言风格规范
- [ ] 标题层级是否正确

## 索引命名规范

> **README 定规矩，索引查内容**
> - `README.md` → 说明规则、使用规范
> - `_*.md` 索引 → 辅助快速查阅内容

### 索引类型

| 文件名前缀 | 类型 | 用途 | 复杂度 | 示例 |
|-----------|------|------|--------|------|
| `_INDEX-*.md` | **目录清单型** | 列出目录下所有子项 | ⭐ 低 | `_INDEX-Areas.md` |
| `_NAV-*.md` | **分类导航型** | 按主题/类别组织导航 | ⭐⭐ 中 | `_NAV-知识领域.md` |
| `_AGG-*.md` | **内容聚合型** | 聚合分散内容成知识图谱 | ⭐⭐⭐ 高 | `_AGG-逻辑谬误类型.md` |
| `_MAP-*.md` | **关系图谱型** | 展示概念间的关联关系 | ⭐⭐⭐ 高 | `_MAP-思维模型关系.md` |

### 使用场景速查

| 场景 | 推荐类型 | 判断标准 |
|------|---------|---------|
| 目录下只有平级文件/文件夹 | `_INDEX` | 简单列出即可，无需复杂分类 |
| 需要按主题快速跳转定位 | `_NAV` | 内容有明显类别划分 |
| 内容分散在多个文件中 | `_AGG` | 需要汇总展示才能理解全貌 |
| 概念间有复杂关联依赖 | `_MAP` | 关系比内容本身更重要 |

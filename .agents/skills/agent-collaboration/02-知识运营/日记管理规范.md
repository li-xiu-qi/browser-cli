# 日记管理规范

**定位**: 个人日记的记录格式与目录管理规范  
**适用范围**: `日记/` 目录下的所有日记文件

---

## 目录结构

### 层级设计

```
日记/
├── 2025/                      ← 年
│   ├── 10/                    ← 月
│   │   ├── 2025-10-23.md     ← 日（Thino 格式）
│   │   ├── 2025-10-25.md
│   │   └── ...
│   ├── 11/
│   └── 12/
├── 2026/
│   ├── 01/
│   └── 02/
└── ...
```

### 设计原则

- **年/月 层级**: 避免根目录文件过多，便于长期管理
- **日文件**: 每日一个 Markdown 文件，文件名格式 `YYYY-MM-DD.md`
- **兼容 Thino**: 支持 Thino 插件的时间戳格式

---

## Thino 格式规范

### 基本格式

```markdown
- HH:MM 
	#标签
	内容文本
	可以有多行
```

### 格式要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **时间戳** | `- HH:MM ` 开头 | `- 14:30 ` |
| **标签** | `#` + 标签名 | `#AI编程` `#思考` `#阅读` |
| **内容** | 缩进的文本 | 记录想法、事件、感悟 |

### 示例

```markdown
- 00:38 
	#AI编程 
	同一个模块下，可以一边给单个文件生成设计文档，一边修改另外一个文件的内容。
	
- 01:05 
	#AI编程 
	有时候 ai 写的代码真的太厉害了，导致我们会过于信任他，一旦我过于信任他之后我就会不测试刚写完的模块。
	真的要及时检查啊。
	
- 17:38 
	#思考 
	遇到复杂问题时，分解成小模块依次测试，是最高效的解决方式。
```

## 标签使用建议

### 常用标签

| 标签 | 用途 |
|------|------|
| `#AI编程` | AI 协作编程相关 |
| `#思考` | 深度思考、反思 |
| `#阅读` | 读书、文章笔记 |
| `#工作` | 工作相关 |
| `#学习` | 学习新内容 |
| `#生活` | 生活琐事、感悟 |
| `#灵感` | 突然的想法 |

### 标签原则

- 简洁：标签名尽量简短
- 统一：相同含义使用相同标签
- 适量：每条记录 1-2 个标签即可

## 新建日记流程

### 手动创建

```bash
# 1. 确定日期
YYYY=2026
MM=02
DD=17

# 2. 创建年月目录（如不存在）
mkdir -p "日记/$YYYY/$MM"

# 3. 创建日记文件
touch "日记/$YYYY/$MM/$YYYY-$MM-$DD.md"
```

### Obsidian 模板（可选）

```markdown
---
date: {{date:YYYY-MM-DD}}
create_time: {{time:HH:mm}}
---

# {{date:YYYY年MM月DD日}}

- {{time:HH:mm}} 
	#
	
```

## 与 Thino 插件配合

### 推荐配置

1. **日记目录**: 设置为 `日记/`
2. **文件格式**: `YYYY/MM/YYYY-MM-DD.md`
3. **时间戳**: 启用自动时间戳
4. **标签**: 支持 `#` 开头的标签

### 使用方式

- 快速记录：使用 Thino 的快捷输入
- 整理回顾：按标签筛选历史记录
- 导出备份：定期导出 Markdown 文件

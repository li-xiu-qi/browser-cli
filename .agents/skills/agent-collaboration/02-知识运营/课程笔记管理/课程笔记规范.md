# 课程笔记规范

**定位**: 课程学习笔记的标准格式和记录规范  
**目标**: 确保课程笔记结构统一、便于回顾和知识萃取

---

## 前置检查（AI 执行前必读）

在记录课程笔记前，AI 必须：

1. **检查课程类型** - 确定是一堂课程、网课、还是演讲
2. **查找现有笔记** - 搜索是否已有相关课程笔记，避免重复
3. **选择合适模板** - 根据课程类型选择对应模板
4. **确定存储位置** - 按 PARA 方法放入正确目录

---

## 标准课程笔记结构

```markdown
# {课程名称}

## 元信息

| 项目 | 内容 |
|------|------|
| 课程来源 | 一堂/得到/混沌/线下 等 |
| 讲师 | {讲师姓名} |
| 日期 | {听课日期} |
| 时长 | {课程时长} |
| 链接 | {课程链接，如有} |
| 标签 | #课程 #{主题} #{讲师} |

## 一句话总结

{用 1-2 句话概括课程核心观点}

## 核心框架

{课程的整体结构，如：3个要点、5个步骤等}

## 详细内容

### {要点1}

- **核心观点**: 
- **关键论证**:
- **案例/数据**:
- **我的理解**:

### {要点2}

...

## 金句摘录

> {金句1} ——{讲师}

> {金句2} ——{讲师}

## 行动清单

- [ ] {行动项1}
- [ ] {行动项2}

## 关联思考

- 与 [[其他笔记]] 的关联
- 可以应用到的场景
- 需要进一步探究的问题

## 复盘与回顾

**一周后**:
- 最有价值的收获:
- 实际应用的尝试:

**一月后**:
- 持续有效的观点:
- 需要更新/修正的理解:
```

## 不同类型课程的处理

### 一堂课程

**特点**: 系统性强、案例丰富、有作业

**重点记录**:
- 决策框架和模型
- 典型案例（使用案例收集模板）
- 作业要求和提交方式
- 同学分享的有价值观点

**存储位置**: `Areas/学习/一堂/{年份}/`

### 在线网课（得到/混沌/极客时间）

**特点**: 碎片化、可反复听、有文稿

**重点记录**:
- 时间戳标记关键内容
- 核心概念的定义
- 推荐的书单/资源

**存储位置**: `Areas/学习/{平台}/`

### 演讲/Talk

**特点**: 时间短、观点密集、启发性强

**重点记录**:
- 核心论点
- 支持论点的关键证据
- 引发思考的观点

**存储位置**: `Resources/演讲/{年份}/`

## 课程笔记 vs 普通笔记的区别

| 维度 | 课程笔记 | 普通笔记 |
|------|----------|----------|
| 结构 | 严格按模板 | 灵活自由 |
| 元信息 | 必须完整 | 可选 |
| 行动项 | 必有 | 可选 |
| 复盘 | 定期回顾 | 按需 |
| 关联 | 强调知识体系 | 自由链接 |

---

## 常见错误

 **只记录不思考** - 复制讲师的话，没有自己的理解  
 **过于详细** - 试图记录每一句话，失去重点  
 **没有行动** - 学了很多，但没有任何改变  
 **不复盘** - 记完就忘，从不回顾  
 **散乱存放** - 课程笔记散落在各处，难以查找

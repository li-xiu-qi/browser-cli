# 知识库管理手册

**定位**: 知识库维护的操作指南  
**核心**: 模板使用、规范检查、重构方法

## 模板使用

### 模板位置

| 模板类型 | 位置 | 使用场景 |
|----------|------|----------|
| 课程样例 | `templates/课程样例模板.md` | 一堂课程案例收集 |
| 读书笔记 | `templates/读书笔记_*.md` | 各类书籍阅读 |
| 决策案例 | `.agents/skills/learning-notes/SKILL.md` (场景2) | ROI 决策分析 |

### 使用方法

1. 复制模板到目标位置
2. 按模板结构填写内容
3. 删除模板中的示例文字
4. 调整标题和标签

## 创建新笔记

### 流程

```
确定笔记类型 → 选择目录 → 复制模板 → 填写内容 → 规范检查
```

### 目录选择

参考 [[../00-基础规范/PARA方法论]] 的分类规则。

### 命名

- 使用中文或英文，保持一致性
- 简洁描述主题
- 避免特殊字符

示例:
- `五层决策模型.md`
- `写作规范.md`
- `course-roi-analysis.md`

### PARA 分类检查

- [ ] 文件放在正确目录

## 重构笔记

### 何时重构

- 内容结构混乱
- 需要更新模板格式
- 目录调整需要移动

### 重构流程

1. **备份**: 复制原笔记到安全位置
2. **分析**: 识别需要保留的核心信息
3. **重组织**: 按新模板结构整理
4. **迁移附件**: 移动附件到新位置
5. **更新链接**: 检查和更新 wikilink
6. **验证**: 确保链接和附件正常

### 笔记迁移

使用迁移脚本:
```bash
node Areas/Scripts/move_note.js "源文件路径" "目标目录"
```

## 附件管理

### 强制规则

**附件必须与笔记同级存放**

- 笔记: `路径/笔记名.md`
- 附件目录: `路径/assets/笔记名/`
- 引用: `![描述](assets/笔记名/图片.png)`

### 禁止

- 全局附件文件夹如 `attachments/`
- 图片直接放在笔记同级
- 通用文件夹名如 `assets/images`

### 附件命名

- 使用描述性名称
- 避免中文文件名（可选）
- 日期前缀：`2026-02-02-screenshot.png`

## 图表管理

### 文本制图优先

**推荐**：使用代码生成图表（SVG），而非截图或手绘

**优势**：
- 版本可控：图表随代码一起变更
- 可维护性：修改文本即可更新
- 清晰度高：矢量图无限缩放

**工具**：[[Areas/文本制图/_INDEX-文本制图|文本制图索引]]

| 场景 | 推荐工具 | 输出格式 |
|------|---------|----------|
| 流程图、架构图 | Graphviz、D2 | SVG |
| UML 建模 | PlantUML、Mermaid | SVG |
| 数据可视化 | Vega-Lite | SVG |
| 时序图 | WaveDrom、SeqDiag | SVG |

### SVG 图表存放

**存放位置**：与笔记附件相同，`assets/笔记名/图表名.svg`

**引用方式**：`![图表描述](assets/笔记名/图表名.svg)`

### 本地渲染

**本地 Kroki**：http://localhost:8000

**渲染脚本**：
```bash
# 使用本地 Kroki 渲染
curl http://localhost:8000/graphviz/svg \
  -d @diagram.dot \
  -o assets/笔记名/diagram.svg
```

**批量生成**：[[Areas/文本制图/guides/examples|示例集]]

## 问题排查

### 链接失效

1. 使用 Obsidian 反向链接检查
2. 更新或删除失效链接
3. 移动文件时使用迁移脚本

### 附件丢失

1. 检查附件路径
2. 确认附件在 `assets/笔记名/` 目录
3. 修复引用路径

### 规范不一致

1. 参考 [[../00-基础规范/写作规范]]
2. 使用 `.agents/skills/obsidian-markdown-formatter/` 检查
3. 手动修正或使用脚本批量处理

## Cliper Datas 管理

**位置**: `cliper_datas/`

统一管理从网络抓取的文章和资料，按主题分类存储。

```
cliper_datas/
├── README.md              # 管理计划说明
├── INDEX.md               # 全部文章索引
├── ai-agent-skills/       # AI Agent 与 Skills 技术
├── ai-coding/             # AI 编程与开发工具
├── ai-tools/              # AI 工具测评与使用
├── ai-research/           # AI 研究与行业趋势
├── tutorials/             # 教程指南与技巧
└── business/              # 商业与投资
```

**分类规则**:

| 分类 | 文件夹 | 标签 |
|------|--------|------|
| AI Agent 技能 | `ai-agent-skills/` | `#ai/agent` `#claude/skills` |
| AI 编程 | `ai-coding/` | `#ai/coding` `#dev/tools` |
| AI 工具 | `ai-tools/` | `#ai/tools` `#review` |
| AI 研究 | `ai-research/` | `#ai/research` `#trends` |
| 教程指南 | `tutorials/` | `#tutorial` `#howto` |
| 商业投资 | `business/` | `#business` `#investment` |

**使用方式**:
- 抓取文章自动保存到 `cliper_datas/`
- 根据内容主题移动到对应子文件夹
- 在 `INDEX.md` 中更新索引
- 使用 `[[文章名]]` 在主知识库中引用

详细规范参见: [[cliper_datas/README]]

## 知识导航体系（MOC）

本知识库采用 **MOC（Map of Content）** 方式进行知识导航，由 **Agent AI 人工维护**，而非依赖 Dataview 等自动化查询插件。

### 核心理念

- **以 Agent AI 为核心**：导航地图由 AI 助手根据知识库实际内容手工整理、定期更新
- **保持主题领域结构**：物理存储按 PARA 主题领域（团队管理、项目管理等）组织
- **课程视角聚合**：通过 MOC 文件提供课程体系的横向视图
- **人工维护优于自动查询**：避免插件依赖，确保链接质量和上下文完整性

### 现有导航地图

| 导航文件 | 内容 | 来源领域 |
|----------|------|----------|
| [[Areas/导航/一堂课程-个人篇\|一堂课程-个人篇]] | 不断提认知、个人红点、做决策、形成竞争力、时间管理、设计力 | 知识管理、时间管理、人生战略、泛产品设计、科学决策 |
| [[Areas/导航/一堂课程-管理篇\|一堂课程-管理篇]] | 管团队（懂招聘、基本功）、管项目（复盘、方案、计划、执行） | 团队管理、项目管理 |
| [[Areas/导航/一堂课程-创业篇\|一堂课程-创业篇]] | 底层逻辑（解放思想、科学理念、实事求是）、预判阶段 | 创业方法论 |

### 维护规范

1. **新增笔记时**：AI 助手判断是否需要更新相关 MOC，将新笔记链接加入对应章节
2. **定期审查**：季度检查 MOC 中链接的有效性
3. **创建新 MOC**：当需要新的知识视角时，由 AI 助手在 `Areas/导航/` 下创建新的 MOC 文件

# 内容质量评分标准（AI可评估版）

> 基于已转换的 Markdown 内容进行评估，不依赖原始文件特征。

## 评估前提

**我能看到的**：
- Markdown 文本内容
- 章节标题结构（# ## ###）
- 文字排版质量（错字、乱码、断行）
- 内容类型（代码、论述、案例等）

**我看不到的**：
- 原始PDF是否有目录
- 能否复制粘贴文字
- 原始排版美观度

因此，评分完全基于**转换后的内容可用性**。

---

## 维度1: 文本质量 (text_quality)

**评估问题**：转换后的Markdown文字是否可用？

### 5分 - 完美文本
**特征**：
- 无乱码、无错字
- 标点符号正确
- 中英文之间有空格（符合排版规范）
- 代码块格式正确
- 无断行错误（段落完整）

**典型来源**：EPUB转换、精心制作的电子书

### 4分 - 良好文本
**特征**：
- 极少错字（<1%）
- 无乱码
- 偶有个别标点错误
- 不影响阅读理解

**典型来源**：质量好的PDF转文字

### 3分 - 可用文本
**特征**：
- 有错字但上下文可推断（1-5%）
- 无大面积乱码
- 段落基本完整
- 阅读时需要脑补个别字

**典型来源**：普通OCR、微信读书扫描版

### 2分 - 较差文本
**特征**：
- 错字较多（5-15%）
- 偶有小段乱码
- 段落断裂，需要合并
- 阅读吃力，但能理解大意

**典型来源**：低质量OCR、老旧扫描版

### 1分 - 不可用文本
**特征**：
- 大量乱码或错字（>15%）
- 段落严重错乱
- 无法理解内容
- 需要重新OCR或找其他版本

**典型来源**：损坏的文件、极低质量扫描

## 维度2: 结构清晰度 (structure)

**评估问题**：Markdown的章节结构是否清晰？

### 5分 - 完美结构
**特征**：
- 有清晰的标题层级（# ## ###）
- 层级逻辑合理
- 有目录或概览章节
- 便于快速定位和跳转

**示例**：
```markdown
# 第一章 绪论
## 1.1 背景
### 1.1.1 历史发展
## 1.2 目标
# 第二章 核心概念
...
```

### 4分 - 良好结构
**特征**：
- 有标题层级
- 大部分层级合理
- 偶有小节标题缺失
- 整体可导航

### 3分 - 一般结构
**特征**：
- 有章节划分但不完整
- 标题层级可能混乱（如突然跳级）
- 缺少小节标题
- 能区分大章节，但细节难定位

### 2分 - 较差结构
**特征**：
- 章节标题稀少
- 大部分内容无标题分隔
- 长段文字流
- 难以快速定位具体内容

### 1分 - 碎片结构
**特征**：
- 几乎无标题
- 纯文本流
- 或标题完全混乱
- 无法结构化导航

## 维度3: 知识密度 (density)

**评估问题**：内容的信息浓度如何？是否值得AI引用？

### 5分 - 极高密度
**特征**：
- 概念定义清晰
- 方法论完整
- 有代码/公式/图表
- 案例与理论结合
- 可直接作为知识源

**典型类型**：
- 技术手册（设计模式、算法）
- 学术教材
- 方法论专著（系统思考、决策模型）

**AI价值**：可直接引用，解释概念，举例说明

### 4分 - 高密度
**特征**：
- 专业知识丰富
- 有实操指导
- 观点明确且有论证
- 信息量大

**典型类型**：
- 专业书籍（软件工程、架构设计）
- 实战指南
- 深度分析报告

**AI价值**：适合引用观点、方法、案例

### 3分 - 中等密度
**特征**：
- 通识性内容
- 概念解释为主
- 有启发性但不够深入
- 故事和知识混合

**典型类型**：
- 入门教程
- 通俗科普
- 商业随笔

**AI价值**：适合基础概念解释，不适合深度引用

### 2分 - 低密度
**特征**：
- 以故事/案例为主
- 论述较少
- 启发性强但信息量少
- 个人感悟多

**典型类型**：
- 散文集
- 传记
- 励志书

**AI价值**：偶尔可引用案例，不适合系统性引用

### 1分 - 极低密度
**特征**：
- 纯视觉内容（图为主）
- 碎片化信息
- 无系统论述
- 几乎无文字或文字极少

**典型类型**：
- 图册/画册
- 漫画
- 摄影集

**AI价值**：几乎无法引用文字内容

## 维度4: AI适配度 (ai_ready)

**评估问题**：综合判断，这本书是否适合直接用于AI问答？

**计算方式**：基于前3个维度 + 内容类型判断

### 5分 - 即插即用
**条件**：
- text_quality ≥ 4
- structure ≥ 4
- density ≥ 4
- 内容类型为知识/方法论

**结论**：直接chunked入库，AI可高质量引用

### 4分 - 微调可用
**条件**：
- text_quality ≥ 3
- structure ≥ 3
- density ≥ 4
- 少量错字但不影响理解

**结论**：简单清理后即可使用

### 3分 - 需要处理
**条件**：
- text_quality = 3（有错字）
- structure ≥ 3
- density ≥ 3

**结论**：可用，但引用时需核实，或边用边修正

### 2分 - 成本较高
**条件**：
- text_quality ≤ 2（错字多）
- 或 structure ≤ 2（结构差）
- 但 density ≥ 3（内容有价值）

**结论**：需要专门校对或重构结构，投入产出比需评估

### 1分 - 不建议
**条件**：
- text_quality ≤ 2
- density ≤ 2
- 或内容以图为主

**结论**：不值得AI化处理，保留原文件即可

## 快速评分方法（基于内容）

拿到一个Markdown文件，按以下步骤评估：

### Step 1: 文本质量（看前3页）
- 快速滚动，看是否有乱码 `□□□`
- 看是否有明显错字（"的/地/得"混用不算）
- 看段落是否完整（没有突然断句）

| 观察 | 打分 |
|------|------|
| 无错字无乱码 | 5 |
| 极少错字 | 4 |
| 有错字但能理解 | 3 |
| 错字多 | 2 |
| 大量乱码 | 1 |

### Step 2: 结构清晰度（看目录/标题）
- 看是否有 `# ## ###` 层级
- 看层级是否合理

| 观察 | 打分 |
|------|------|
| 清晰层级，有目录 | 5 |
| 有层级，偶有不合理 | 4 |
| 有章节但不完整 | 3 |
| 标题稀少 | 2 |
| 几乎无标题 | 1 |

### Step 3: 知识密度（翻全文）
- 看是否有概念定义
- 看是否有方法论/框架
- 看代码/公式密度
- 看故事vs干货比例

| 观察 | 打分 |
|------|------|
| 纯干货/方法论 | 5 |
| 干货为主 | 4 |
| 干货故事各半 | 3 |
| 故事为主 | 2 |
| 纯图/纯故事 | 1 |

### Step 4: AI适配度（综合判断）

根据前三项和书籍类型：

```
如果 text_quality ≥ 4 且 structure ≥ 4 且 density ≥ 4:
    ai_ready = 5
elif text_quality ≥ 3 且 structure ≥ 3 且 density ≥ 4:
    ai_ready = 4
elif text_quality ≥ 3 且 structure ≥ 3 且 density ≥ 3:
    ai_ready = 3
elif density ≥ 3 但 text_quality ≤ 2 或 structure ≤ 2:
    ai_ready = 2  # 有价值但处理成本高
else:
    ai_ready = 1  # 不建议
```

## 评分示例

### 示例1: 《软件设计的哲学》Markdown

**观察**：
- 文本：无错字，排版规范 ✓
- 结构：章节清晰，有层级 ✓
- 内容：概念、原则、方法论 ✓

**评分**：
```yaml
text_quality: 5
structure: 5
density: 5
ai_ready: 5
quality_score: 5.0  # A级
```

**处理**：立即 chunked

### 示例2: 某OCR版《认知觉醒》

**观察**：
- 文本：偶有错字（如"认知"变"认知"），但不影响理解
- 结构：有章节标题，但小节缺失
- 内容：心理学概念+案例，干货较多

**评分**：
```yaml
text_quality: 3  # 有错字但可读
structure: 3     # 有章节但不完整
density: 4       # 认知干货
ai_ready: 3      # 可用但需核实
quality_score: 3.25  # C级偏上
```

**处理**：indexed，边用边修正

### 示例3: 某扫描版随笔集

**观察**：
- 文本：OCR错字较多，部分句子不通
- 结构：几乎无标题，纯文本流
- 内容：散文、个人感悟

**评分**：
```yaml
text_quality: 2  # 错字多
structure: 1     # 无结构
density: 2       # 低密度随笔
ai_ready: 1      # 不建议
quality_score: 1.5  # D级
```

**处理**：保持 raw，不投入时间

## 评分后操作

根据评分结果，按以下优先级处理：

**A级 (4.5+)** - 预计10-15本
- 操作：转换为 chunked
- 投入：每本30分钟
- 收益：核心知识库资产

**B级 (3.5-4.4)** - 预计15-20本  
- 操作：保持 indexed
- 投入：每本5-10分钟清理
- 收益：可用资产

**C级 (2.5-3.4)** - 预计20-25本
- 操作：保持 indexed 或 raw
- 投入：暂时不处理
- 收益：参考备用

**D级 (< 2.5)** - 预计5-10本
- 操作：标记 ref
- 投入：不处理
- 收益：避免浪费时间

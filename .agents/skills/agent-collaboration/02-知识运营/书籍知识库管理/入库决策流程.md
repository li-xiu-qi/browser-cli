# 书籍入库快速决策（AI协作版）

## 核心原则

**不追求读完，追求AI能用。**

书籍是知识节点，不是待办任务。

---

## 拿到一本新书，3步处理

### Step 1: 放入 Library/
```
下载的书 → 直接拖入 Areas/书籍/Library/
```

### Step 2: 4维度评分（基于内容）

**基于已转换的Markdown内容评估**（不是原始文件）：

| 维度 | 观察内容 | 打分 |
|------|----------|------|
| **text_quality** | 看前3页：有错字/乱码吗？ | 无→5 / 极少→4 / 有错但可读→3 / 错多→2 / 乱码→1 |
| **structure** | 看标题层级：有# ## ###吗？ | 完美→5 / 良好→4 / 一般→3 / 较差→2 / 碎片→1 |
| **density** | 翻全文：干货比例？ | 纯干货→5 / 干货多→4 / 各半→3 / 故事多→2 / 纯图→1 |
| **ai_ready** | 综合判断 | 即插→5 / 微调→4 / 需处理→3 / 成本高→2 / 不行→1 |

**综合分 = 四项之和 / 4**

**详细标准**：→ [内容质量评分标准.md](./内容质量评分标准.md)

### Step 3: 决定处理深度

| 综合分 | 等级 | 处理建议 | 时间投入 |
|--------|------|----------|----------|
| **4.5-5.0** | A级 | **chunked** - 加入核心知识库 | 30min |
| **3.5-4.4** | B级 | **indexed** - 生成Markdown | 10min |
| **2.5-3.4** | C级 | **indexed** - 基础处理 | 5min |
| **< 2.5** | D级 | **raw** - 保留不处理 | 0min |

## 具体处理方式

### A级书籍 (4.5+) - 核心资产

**标准**：
- EPUB/完美PDF
- 结构清晰
- 高密度知识
- 技术/方法论类

**操作**：
```bash
# 1. 生成 Markdown
node .agents/skills/content-converter/tools/epub2md.js "Library/xxx.epub" --category "02"

# 2. 切分chunked（需向量化工具）
# 结果：ai_status = "chunked"

# 3. 卡片标记
ai_status: chunked
quality_score: 4.8
```

**结果**：AI可直接检索引用

### B级书籍 (3.5-4.4) - 可用资产

**标准**：
- 内容质量好
- 可能有少量OCR错误
- 需要简单清理

**操作**：
```bash
# 1. 生成/转换 Markdown
node .agents/skills/content-converter/tools/epub2md.js "Library/xxx.epub" --category "05"

# 2. 简单检查修正
# 结果：ai_status = "indexed"
```

**结果**：可全文检索，引用时可能需要核实

### C级书籍 (2.5-3.4) - 参考资产

**标准**：
- 低密度或结构一般
- 暂时不急需

**操作**：
```bash
# 仅创建卡片，不处理内容
# ai_status: raw
# 用时再处理
```

### D级书籍 (< 2.5) - 保留但不依赖

**标准**：
- 图册、漫画
- 严重质量问题

**操作**：
- 保留在 Library/
- 卡片标记 ref
- 不投入时间处理

## 常见场景处理

### 场景1: Z-Library EPUB 《Python设计模式》

```yaml
评分：
  text_quality: 5    # EPUB完美
  structure: 5       # 有目录
  density: 5         # 技术干货
  ai_ready: 5        # 即插即用
综合分：5.0 (A级)

处理：立即 chunked
结果：加入核心知识库
```

### 场景2: 微信读书扫描版《认知觉醒》

```yaml
评分：
  text_quality: 3    # OCR有错但可读
  structure: 4       # 有目录
  density: 4         # 认知干货
  ai_ready: 3        # 需OCR
综合分：3.5 (B级)

处理：
  1. OCR转换 Markdown
  2. 标记 ai_status: indexed
  3. 边用边修正错字
```

### 场景3: 散文集《慢煮生活》

```yaml
评分：
  text_quality: 4    # PDF有文字
  structure: 3       # 章节简单
  density: 2         # 随笔低密度
  ai_ready: 2        # 不适合AI
综合分：2.75 (C级)

处理：
  - 仅创建卡片
  - ai_status: raw
  - 不优先处理
```

### 场景4: 设计图册《写给大家看的设计书》

```yaml
评分：
  text_quality: 4    # PDF有文字
  structure: 4       # 有目录
  density: 3         # 有知识但图多
  ai_ready: 2        # 图多影响AI
综合分：3.25 (C级)

处理：
  - indexed（文字部分）
  - 图版保留PDF参考
```

## 与AI协作使用

### 场景：问AI一个问题

**用户**：解释策略模式

**AI动作**：
1. 检索 chunked 书籍
2. 找到《设计模式之禅》相关内容
3. 引用：「根据《设计模式之禅》，策略模式定义了...」

### 场景：概念对比

**用户**：对比一下几种架构模式

**AI动作**：
1. 检索多本 indexed/chunked 书籍
2. 综合《软件架构》《凤凰架构》等观点
3. 给出对比分析

### 场景：引用具体案例

**用户**：给决策失误的例子

**AI动作**：
1. 检索 density 高的认知类书籍
2. 找到《对赌》中的案例
3. 引用具体故事

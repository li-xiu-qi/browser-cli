# 书籍知识库 - 开始使用

## 这是什么？

这不是传统意义上的"阅读书单"，而是一个**AI协作知识库**。

**核心理念转变**：
-  不是：记录"我读了多少页"
-  而是：评估"AI能不能准确引用这本书"

## 关键概念

### AI状态 (ai_status)

| 状态 | 含义 | 说明 |
|------|------|------|
| **raw** | 原始文件 | 仅在 Library/，未处理 |
| **indexed** | 已索引 | 有Markdown，可全文检索 |
| **chunked** | 已向量化 | 可RAG检索，AI可直接引用 |
| **ref** | 仅参考 | 保留原文件，暂不处理 |

### 质量评分 (1-5分) - 基于内容评估

**重要**：评分基于**已转换的Markdown内容**，不是原始文件特征。

| 维度 | 含义 | 基于内容的判断 |
|------|------|----------------|
| **text_quality** | 文本质量 | 有错字/乱码吗？段落完整吗？ |
| **structure** | 结构清晰度 | Markdown有层级标题吗？ |
| **density** | 知识密度 | 干货vs水分比例？ |
| **ai_ready** | AI适配度 | 综合判断适合AI引用吗？ |

**评分指南**：→ [内容质量评分标准.md](./内容质量评分标准.md)  
**评分示例**：→ [评分示例.md](./评分示例.md)（已评4本书）

**综合分 = (4项之和) / 4**，等级：A(4.5+)/B/C/D

### 处理优先级

- **A级 (4.5+)** → chunked（核心知识库）
- **B级 (3.5-4.4)** → indexed（可用资产）
- **C级 (2.5-3.4)** → raw（用时再处理）
- **D级 (< 2.5)** → ref（保留但不依赖）

## 数据库视图

打开 `我的书架.base`，有4个视图：

1. **知识库概览** - 按AI状态分组，看全貌
2. **高质量资产（A级）** - 优先处理的核心书
3. **待处理（未AI化）** - 需要评分的书
4. **按分类查看** - 按知识领域浏览

## 一句话总结

> **不对每本书负责，只对高质量书深入；不追求读完，追求AI能用。**

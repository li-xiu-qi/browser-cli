# 书籍管理策略（AI协作版）

> 书籍 = 可检索的知识库 ≠ 待办阅读清单

## 核心理念转变

### 传统阅读管理（已废弃）
- 关注：读没读、读到哪页、什么时候读完
- 状态：待读/阅读中/已读/弃读
- 目标：完成阅读

### AI协作知识库（新定位）
- 关注：这本书对AI是否可用、知识质量如何、适合什么场景查询
- 状态：已AI化/未处理/仅参考
- 目标：能被AI检索和引用

## 1. 知识可用性分级（替代阅读状态）

### 可用性状态

| 状态 | 含义 | 说明 |
|------|------|------|
| **raw** | 原始文件 | 仅在 Library/ 中，未做任何处理 |
| **indexed** | 已索引 | 已生成 Markdown，可供AI检索 |
| **chunked** | 已向量化 | 已切分chunk，可用于RAG检索 |
| **ref** | 仅参考 | 保留原文件，但暂不处理（工具书/图册等） |

### 为什么不用阅读状态

- **不追求读完**：查缺补漏，用多少查多少
- **AI一起读**：不是"我读"，是"AI帮我读"
- **即时可用**：一旦 indexed，立即可问AI相关问题

## 2. 质量评分体系（AI协作视角）

### 评分维度（1-5分）

#### 维度1: 文本质量 (text_quality)
评估内容是否能被AI准确解析

| 分数 | 描述 | 典型情况 |
|------|------|----------|
| 5 | 完美 | EPUB转换、原始Markdown，无错字无乱码 |
| 4 | 良好 | PDF有文字层，偶尔识别错误但不影响理解 |
| 3 | 可用 | OCR版，有错字但上下文可推断 |
| 2 | 较差 | OCR版，错字较多，需人工校对才能用 |
| 1 | 不可用 | 严重乱码、缺页、排版混乱 |

#### 维度2: 结构清晰度 (structure)
评估是否有目录、章节清晰，便于AI定位

| 分数 | 描述 | 影响 |
|------|------|------|
| 5 | 完美结构 | 有目录、层级清晰、标题规范，AI能精准定位 |
| 4 | 良好 | 有目录，部分层级略乱 |
| 3 | 一般 | 有章节但无目录，需全文检索 |
| 2 | 较差 | 章节混乱，长篇无分割 |
| 1 | 碎片 | 纯文本流，无结构 |

#### 维度3: 知识密度 (density)
评估内容的信息价值（对AI协作而言）

| 分数 | 描述 | 典型书籍 |
|------|------|----------|
| 5 | 极高密度 | 经典教材、技术手册、方法论（如《软件设计的哲学》） |
| 4 | 高 | 专业书籍、实战指南 |
| 3 | 中等 | 通识读物、入门教程 |
| 2 | 低 | 随笔、散文、轻松读物 |
| 1 | 极低 | 图册、漫画、纯视觉内容 |

#### 维度4: AI适配度 (ai_ready)
综合评估是否适合直接用于AI问答

| 分数 | 含义 | 行动建议 |
|------|------|----------|
| 5 | 即插即用 | 高质量EPUB/MD，直接chunked入库 |
| 4 | 微调可用 | 少量错字，简单清理后即可 |
| 3 | 需要处理 | OCR版，需边用边修正 |
| 2 | 成本较高 | 扫描版，需专门OCR和校对 |
| 1 | 不建议 | 图册、代码截图为主的书籍 |

### 综合质量分

```
quality_score = (text_quality + structure + density + ai_ready) / 4
```

**质量等级映射**
- 4.5-5.0: A级（核心资产，优先向量化）
- 3.5-4.4: B级（可用，按需处理）
- 2.5-3.4: C级（参考用，不优先处理）
- < 2.5: D级（保留但不推荐依赖）

## 3. Frontmatter 新规范

```yaml
---
title: "书名"
author: "作者"
tags:
  - 书籍
  - 技术/认知/管理  # 知识领域

# 来源与格式
source: "zlibrary"       # zlibrary / wechat / ocr / purchased
format: "epub"           # epub/pdf/mobi/md
file_path: "Library/xxx" # 原始文件位置

# 知识可用性状态
ai_status: "indexed"     # raw/indexed/chunked/ref

# 质量评分（1-5分）
text_quality: 4          # 文本质量
structure: 5             # 结构清晰度
density: 5               # 知识密度
ai_ready: 4              # AI适配度
quality_score: 4.5       # 综合分（自动计算）

# 知识元数据
category: "软件工程"     # 知识分类
keywords: []             # 关键词标签，用于检索
summary: ""              # 一句话总结，AI引用时展示
---
```

## 4. 使用场景标记

### 适合AI协作的书籍类型

**高优先级处理（优先 indexed/chunked）**
- 技术手册（API文档、框架指南）
- 方法论书籍（设计模式、系统思维）
- 结构化知识（算法、数据结构）
- 案例集（商业分析、决策案例）

**中优先级（保持 indexed 即可）**
- 通识读物
- 历史/传记
- 散文随笔

**低优先级（保持 raw 或 ref）**
- 图册/画册
- 代码截图为主的书
- 纯视觉设计参考

## 5. 工作流（AI协作场景）

### 新书入库

```
下载 → Library/ → 快速评分 → 决定处理深度
```

**快速评分（3分钟）**
1. 打开文件，看是否能选中文字
2. 检查是否有目录/章节
3. 翻几页看内容密度
4. 给出4个维度1-5分

**决定处理深度**

| 综合分 | 行动 | 目的 |
|--------|------|------|
| 4.5+ | chunked | 加入RAG知识库，随时可问 |
| 3.5-4.4 | indexed | 生成Markdown，全文可检索 |
| 2.5-3.4 | indexed | 基础处理，用时再细整 |
| < 2.5 | raw/ref | 保留但不主动处理 |

### AI协作使用

**场景1: 问答检索**
```
用户问：软件设计中如何处理复杂度？
AI检索：从 chunked 书籍中找相关内容
引用：《软件设计的哲学》第3章提到...
```

**场景2: 概念解释**
```
用户问：解释什么是"第二曲线"
AI检索：从 indexed 书籍中找到定义
引用：《第二曲线创新》中指出...
```

**场景3: 对比分析**
```
用户问：几种架构模式的区别
AI检索：多本书相关内容对比
引用：A书说... B书说...
```

## 6. 与阅读管理的区别

| 维度 | 阅读管理 | AI协作知识库 |
|------|----------|--------------|
| 核心问题 | 我读了没有 | AI能不能用 |
| 进度单位 | 页数/百分比 | 处理状态（raw/indexed/chunked） |
| 完成标准 | 读到最后一页 | 可被AI检索引用 |
| 价值判断 | 好不好看 | 知识密度高不高 |
| 组织方式 | 待读/已读列表 | 知识领域/可用性分级 |

## 7. 实施建议

### 存量书籍处理

对现有的 67 本书，按以下优先级处理：

**第一批：技术核心（预计10-15本）**
- 找出软件工程、架构类的高密度书籍
- 评估为 A 级（4.5+分）
- 转换为 chunked，加入核心知识库

**第二批：认知方法论（预计10本）**
- 思维模型、决策方法类
- 评估为 B 级以上（3.5+分）
- indexed 即可

**第三批：其他**
- 保持 raw 或简单 indexed
- 用时再处理

### 增量书籍

每本新书入库时：
1. 放入 Library/
2. 创建卡片，打上4个维度分
3. 根据综合分决定下一步

## 8. 工具支持

### 需要的工具

1. **批量评分工具**：快速给现有书籍打分
2. **chunked 转换**：将高质量书籍切分入库
3. **质量报告**：统计各类别质量分布

### 现有工具适配

- **epub2obsidian**: 用于生成 indexed 版本
- **ocr-to-obsidian**: 用于处理扫描版，提升 text_quality

## 总结

**新定位**：书籍是可检索的知识节点，不是待办任务。

**核心指标**：AI能否准确引用 > 我是否读完。

**管理重点**：质量评分决定处理深度，高质书籍优先向量化。

# 工具选择速查表

拿到一本书，30秒内决定用什么工具处理。

---

## 源头判断 → 工具选择

| 你看到的书 | 判断 | 使用工具 | 预期质量 | 后续操作 |
|-----------|------|----------|----------|----------|
| **EPUB 文件** | 格式好，文字可选 | `epub2obsidian` | ★★★★★ | 直接 chunked |
| **微信读书网页** | 需要抓取 | `wechat-book-to-pdf` | ★★★☆☆ | 按需OCR |
| **PDF 可选中文字** | 有文字层 | 不转换，直接读 | ★★★★☆ | indexed |
| **PDF 纯图片/扫描版** | 无文字层 | `ocr-to-obsidian` | ★★☆☆☆ | 校对后 indexed |
| **手机拍的实体书** | 图片质量参差 | `ocr-to-obsidian` | ★★☆☆☆ | 需大量校对 |
| **图册/漫画/画册** | 视觉为主 | 不OCR | ★☆☆☆☆ | ref 备用 |

## 工具命令速查

### epub2obsidian（EPUB → Markdown）

```bash
# 基本用法
node .agents/tools/epub2obsidian/index.js \
  "Library/书名.epub" \
  --category "02"

# 分类编号
01 = Python与编程实战
02 = 软件工程与代码质量
03 = 架构设计与系统思维
04 = AI编程与前沿技术
05 = 认知成长与软技能
06 = 测试与质量保障
00 = 待分类
```

**输出**：`Areas/书籍/02_分类/书名(作者)/书名(作者) -merged.md`

**质量预期**：text_quality = 5

---

### wechat-book-to-pdf（微信读书 → PDF）

```bash
# 首次登录（只需一次）
node .agents/tools/wechat-book-to-pdf/index.js login

# 前台抓取（短篇）
node .agents/tools/wechat-book-to-pdf/index.js crawl "URL"

# 后台抓取（长篇书籍推荐）
node .agents/tools/wechat-book-to-pdf/index.js crawl "URL" -bg
```

**输出**：`tools/output/book_[ID]/book_result.pdf`

**后续**：复制到 Library/，按需OCR

---

### ocr-to-obsidian（扫描版 → Markdown）

```bash
# 基本用法
python .agents/tools/ocr-to-obsidian/main.py \
  "Library/扫描版.pdf" \
  --output "Areas/书籍/00_待分类/书名.md"

# 大文件后台模式
python .agents/tools/ocr-to-obsidian/main.py \
  "Library/大书.pdf" \
  --output "Areas/书籍/00_待分类/" \
  --background

# 保存图片
python .agents/tools/ocr-to-obsidian/main.py \
  "扫描图片/" \
  --output "Areas/书籍/00_待分类/书名.md" \
  --save-images
```

**输出**：指定路径的 Markdown 文件

**质量预期**：text_quality = 2-4（需校对）

## 处理流程速查

### 场景1：Z-Library下载了EPUB

```
1. 放入 Library/
2. node epub2obsidian --category "02"
3. 评估：text_quality=5, structure=?, density=?
4. quality_score ≥4.5 → chunked
   quality_score <4.5 → indexed
```

**时间**：5分钟

---

### 场景2：微信读书想保存

```
1. node wechat-book-to-pdf crawl "URL" -bg
2. 等待完成（可能30分钟-2小时）
3. 复制PDF到 Library/
4. 决定是否OCR：
   ├─ 需要深度阅读 → ocr-to-obsidian → 校对 → indexed
   └─ 仅参考 → 标记 ref
```

**时间**：抓取30-120分钟 + OCR 10-30分钟

---

### 场景3：扫描版PDF需要处理

```
1. 先打开看清晰度
2. 值得OCR？
   ├─ 值得 → python ocr-to-obsidian --background
   │          → 等待完成 → 校对 → indexed
   └─ 不值得 → 标记 ref
```

**时间**：OCR 10-60分钟 + 校对时间不定

## 常见问题速答

**Q: EPUB转换失败？**  
A: 尝试解包后重新打包，或找PDF版本用OCR

**Q: OCR太慢？**  
A: 大文件用 `--background` 后台模式

**Q: OCR错字太多？**  
A: text_quality <3 的书不值得校对，标记 ref 备用

**Q: 微信读书抓取中断？**  
A: 重新运行 crawl，会自动继续

**Q: 分不清是什么格式？**  
A: 看文件扩展名：.epub / .pdf / .mobi

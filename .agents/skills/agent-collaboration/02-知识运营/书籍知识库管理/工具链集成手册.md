# 工具链集成管理手册

> 基于 `.agents/tools` 工具链的书籍管理完整工作流

## 工具链概览

| 工具 | 功能 | 输入 | 输出 | 适用场景 |
|------|------|------|------|----------|
| **epub2obsidian** | EPUB → Markdown | .epub | 结构化 Markdown | EPUB格式好书 |
| **ocr-to-obsidian** | OCR识别 | .pdf / 图片 | Markdown（需校对） | 扫描版PDF |
| **wechat-book-to-pdf** | 微信读书 → PDF | 微信读书URL | A4 PDF（扫描版） | 微信读书资源 |

## 按来源的标准化处理流程

### 流程 A: EPUB 书籍（推荐）

**来源**：Z-Library、其他电子书站
**质量预期**：通常最好
**工具**：`epub2obsidian`

#### 操作步骤

1. **放入 Library/**
   ```bash
   mv xxx.epub Areas/书籍/Library/
   ```

2. **执行转换**
   ```bash
   node .agents/tools/epub2obsidian/index.js \
     "Areas/书籍/Library/xxx.epub" \
     --category "02"
   ```

3. **输出位置**
   - Markdown: `Areas/书籍/02_软件工程与代码质量/xxx/xxx -merged.md`

4. **质量评估**（基于内容）
   - 通常为 `text_quality: 5`
   - 检查 structure 和 density

5. **后续处理**
   ```yaml
   quality_score: ≥4.5  →  chunked
   quality_score: 3.5-4.4 → indexed
   ```

6. **创建卡片**（可选，由工具自动生成）
   ```yaml
   ---
   title: "书名"
   author: "作者"
   source: "epub"
   ai_status: "indexed"  # 或 chunked
   # 4维度评分
   text_quality: 5
   structure: [1-5]
   density: [1-5]
   ai_ready: [1-5]
   ---
   ```

---

### 流程 B: 微信读书

**来源**：weread.qq.com
**质量预期**：扫描版PDF（纯图片）
**工具**：`wechat-book-to-pdf` → `ocr-to-obsidian`（可选）

#### 操作步骤

1. **抓取PDF**（首次需登录）
   ```bash
   # 登录（只需一次）
   node .agents/tools/wechat-book-to-pdf/index.js login
   
   # 抓取（后台模式推荐）
   node .agents/tools/wechat-book-to-pdf/index.js crawl \
     "https://weread.qq.com/web/reader/xxx" -bg
   ```

2. **获取PDF**
   - 输出: `.agents/tools/wechat-book-to-pdf/output/book_[ID]/book_result.pdf`
   - 复制到: `Areas/书籍/Library/`

3. **决定处理方式**

   **路径 B1 - 仅需参考**
   ```yaml
   # 仅保留PDF，不OCR
   ai_status: "ref"
   source: "wechat"
   ```

   **路径 B2 - 深度使用**
   ```bash
   # OCR转换
   python .agents/tools/ocr-to-obsidian/main.py \
     "Areas/书籍/Library/xxx.pdf" \
     --output "Areas/书籍/05_认知成长与软技能/xxx.md"
   ```

4. **质量评估**
   ```yaml
   text_quality: 3-4    # OCR有错字但可理解
   structure: 3-4       # 取决于原书
   density: [1-5]       # 按内容评估
   ai_ready: 3-4        # 需校对后使用
   ```

5. **后续处理**
   ```yaml
   quality_score: ≥3.5  →  校对后 indexed
   quality_score: <3.5  →  标记 "ocr草稿"
   ```

### 流程 C: PDF扫描版（其他来源）

**来源**：其他网站下载的扫描版PDF
**质量预期**：不确定，需评估
**工具**：`ocr-to-obsidian`

#### 操作步骤

1. **放入 Library/**
   ```bash
   mv xxx.pdf Areas/书籍/Library/
   ```

2. **初步评估**（先不OCR）
   - 打开PDF看清晰度
   - 判断是否值得OCR

3. **决定处理方式**

   **路径 C1 - 不值得OCR**
   ```yaml
   ai_status: "ref"
   note: "扫描版质量差，仅备用"
   ```

   **路径 C2 - 值得OCR**
   ```bash
   python .agents/tools/ocr-to-obsidian/main.py \
     "Areas/书籍/Library/xxx.pdf" \
     --output "Areas/书籍/00_待分类/xxx.md" \
     --background  # 大文件用后台模式
   ```

4. **质量评估**
   ```yaml
   text_quality: [1-5]   # 基于OCR结果
   structure: [1-5]
   density: [1-5]
   ai_ready: [1-5]
   ```

5. **后续处理**
   ```yaml
   quality_score: ≥3.5  →  校对 → 分类目录 → indexed
   quality_score: <3.5  →  标记 "ocr草稿" → 用时再校对
   ```

---

### 流程 D: 实体书扫描

**来源**：自己扫描的书页
**质量预期**：取决于扫描质量
**工具**：`ocr-to-obsidian`

#### 操作步骤

1. **扫描为图片/PDF**

2. **OCR转换**
   ```bash
   python .agents/tools/ocr-to-obsidian/main.py \
     "scanned_pages/" \
     --output "Areas/书籍/00_待分类/xxx.md" \
     --save-images
   ```

3. **人工校对**（必需）
   - OCR错误较多
   - 对照原图逐页校对

4. **质量评估**
   ```yaml
   text_quality: 3-4    # 校对后
   structure: 2-3       # 需手动添加标题
   density: [1-5]
   ai_ready: 3-4        # 校对后可用
   ```

## 质量评估与工具输出的关联

### EPUB转换（epub2obsidian）

```yaml
# 通常结果
text_quality: 5      # EPUB转换质量高
structure: 4-5       # 取决于原书

# 处理建议
if structure >= 4:
    ai_status: "chunked"
else:
    ai_status: "indexed"
```

### OCR转换（ocr-to-obsidian）

```yaml
# 通常结果
text_quality: 2-4    # 取决于扫描质量
structure: 2-3       # 可能丢失层级

# 处理建议
if text_quality >= 3:
    校对后 ai_status: "indexed"
else:
    标记 "需重新OCR或仅参考"
```

### 微信读书PDF

```yaml
# 特性
format: "pdf"
is_scan: true
source: "wechat"

# 处理建议
if 需要深度使用:
    OCR后 quality_score >= 3.5:
        ai_status: "indexed"
    else:
        ai_status: "ref"
else:
    ai_status: "ref"
```

## 常见问题

### Q1: EPUB转换失败怎么办？

**可能原因**：
- EPUB加密
- 格式不规范

**解决方案**：
1. 尝试其他工具解压EPUB
2. 或用OCR工具处理PDF版本
3. 标记 "需人工处理"

### Q2: OCR效果很差怎么办？

**评估**：
- text_quality < 3？
- 错字率 > 15%？

**解决方案**：
1. 检查PDF清晰度
2. 尝试其他OCR引擎
3. 或放弃OCR，保留PDF备用（ai_status: ref）

### Q3: 微信读书抓取失败？

**可能原因**：
- Cookie失效
- 需要登录
- 书籍下架

**解决方案**：
1. 重新运行 `login` 模式
2. 检查书籍URL是否有效
3. 或找其他来源

### Q4: 工具输出位置不一致？

**统一策略**：
- 所有原始文件：Library/
- 所有Markdown：分类目录或 00_待分类/
- 所有卡片：Cards/

## 总结

**核心原则**：
1. **工具驱动**：根据来源选择正确工具
2. **质量导向**：工具输出决定后续处理
3. **按需处理**：不是所有书都值得OCR

**快速决策**：
- EPUB → 直接 epub2obsidian → chunked/indexed
- 微信读书 → 抓取PDF → 按需OCR → 校对后 indexed
- 扫描版PDF → 评估 → 值得才OCR → 校对后 indexed
- 图册/低质量 → Library/ 备用 → ref

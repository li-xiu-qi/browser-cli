# 文章质量评估标准 V2

**定位**: Cliper 文章的质量筛选标准（配图无关版）  
**核心维度**: 思考深度 + 实用性

---

## 质量分级

| 等级 | 标准 | 处理建议 |
|------|------|----------|
| **S级** | 见解独到 + 可操作性强 + 信息密度高 | 精读 + 收藏 + 做笔记 |
| **A级** | 有干货 + 能落地 + 逻辑清晰 | 提取重点 + 归档 |
| **B级** | 有启发但较浅 / 部分有价值 | 扫读 + 选择性保留 |
| **C级** | 水文 / 搬运 / 无实质内容 | 直接删除 |

---

## 评估维度（无配图版）

### 1. 思考深度（权重35%）

不是看说了什么，看**怎么说的**。

| 指标 | 具体标准 | 评分 |
|------|---------|------|
| **有独特见解** | 不是简单复述，有作者自己的判断 | +10 |
| **有分析框架** | 用结构化的方式拆解问题（如：问题→原因→方案→验证） | +10 |
| **有底层逻辑** | 解释「为什么」而不仅是「怎么做」 | +10 |
| **有反常识观点** | 挑战 common sense，提供新视角 | +5 |
| **有批判性** | 不盲从权威，指出局限性 | +5 |

**反面指标**（扣分）:
- 纯粹翻译/搬运：-15
- 只是罗列信息无分析：-10
- 全是正确的废话：-10

---

### 2. 实用性（权重35%）

看完能不能**立即用**。

| 指标 | 具体标准 | 评分 |
|------|---------|------|
| **可立即执行** | 提供具体步骤/命令/配置，复制可用 | +10 |
| **有真实案例** | 作者自己实践过，有数据/截图佐证 | +10 |
| **有踩坑记录** | 分享失败经验、边界条件、注意事项 | +10 |
| **省时间** | 提供工具/脚本/自动化方案 | +5 |
| **可迁移** | 方法不只适用于特定场景，能举一反三 | +5 |

**反面指标**（扣分）:
- 只说「要做什么」不说「怎么做」：-10
- 案例是假设性的（"假设你有一个项目..."）：-5
- 只有结论没有过程：-5

---

### 3. 信息密度（权重20%）

单位时间的收获。

| 指标 | 标准 | 评分 |
|------|------|------|
| 字数/时长比 | 同样2000字，信息量比别人大 | +5 |
| 无废话 | 每段都有信息点，没有「众所周知」式的填充 | +5 |
| 有数据支撑 | 用数字说话，而非「很多人」「大部分」 | +5 |
| 有引用来源 | 标明出处，可追溯验证 | +5 |

**快速判断**:
- 3分钟扫读，能划出3个以上重点 → 密度高
- 读完只记得「挺好的」但说不出好在哪 → 密度低

---

### 4. 表达质量（权重10%）

怎么写的。

| 指标 | 标准 | 评分 |
|------|------|------|
| 逻辑清晰 | 结构分明，层次分明 | +3 |
| 语言精炼 | 不啰嗦，无重复 | +3 |
| 例子贴切 | 类比/案例帮助理解 | +2 |
| 态度真诚 | 不装，不居高临下 | +2 |

## 快速评分表

读完文章后，问自己这5个问题：

```markdown
## 文章评估: [标题]

### 思考深度（35%）
- [ ] 有作者独特观点，非简单复述？（+10）
- [ ] 有分析框架，结构化思考？（+10）
- [ ] 解释了底层逻辑？（+10）
- [ ] 有反常识或批判性视角？（+5）

小计: __/35

### 实用性（35%）
- [ ] 可立即执行，有具体步骤？（+10）
- [ ] 有真实案例/数据支撑？（+10）
- [ ] 有踩坑记录/边界条件？（+10）
- [ ] 能节省我时间或能迁移？（+5）

小计: __/35

### 信息密度（20%）
- [ ] 无废话，每段有信息点？（+5）
- [ ] 有数据而非模糊表述？（+5）
- [ ] 有来源可验证？（+5）

小计: __/15

### 表达质量（10%）
- [ ] 逻辑清晰，结构好？（+3）
- [ ] 语言精炼？（+3）
- [ ] 例子贴切？（+2）

小计: __/8

---

**总分**: __ / 93

**评级**: 
- [ ] S级 (80+) → 精读收藏
- [ ] A级 (60-79) → 提取重点
- [ ] B级 (40-59) → 扫读即可
- [ ] C级 (<40) → 删除

**一句话评价**: [为什么给这个分]
```

## 实战判断流程

```
看到文章
    ↓
3分钟扫读
    ↓
能说出作者的核心观点吗？
    → 不能 → 可能是C级 → 删除
    → 能 → 继续
    ↓
这个观点对我有用吗？
    → 无 → B级以下
    → 有 → 继续
    ↓
有具体可执行的步骤吗？
    → 无 → B级
    → 有 → A级或S级
    ↓
作者是自己实践过吗？
    → 无 → A级
    → 有 → S级
```

---

## 一句话原则

> **"看完这篇文章，我的认知或行动有变化吗？"**
> 
> - 认知变化（思考深度）+ 行动变化（实用性）= 高质量
> - 两者皆无 = 低质量

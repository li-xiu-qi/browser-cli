## 关键教训

### 1. 避免跟踪重复内容

AI 工具目录结构：
```
.agents/           ← ✅ 唯一事实来源（提交）
.agent/            ← ❌ 软链接（忽略）
.claude/           ← ❌ 软链接（忽略）
.gemini/           ← ❌ 软链接（忽略）
.kimi/             ← ❌ 软链接（忽略）
```

### 2. 大文件管理策略

| 文件类型 | 策略 | 原因 |
|----------|------|------|
| PDF 书籍 | ❌ 忽略 | 占用空间大（3.6GB）|
| 媒体文件 | ❌ 忽略 | 不适合 Git 管理 |
| Markdown 索引 | ✅ 提交 | 轻量、可版本控制 |

### 3. 定期清理历史

当仓库体积失控时：
1. 备份重要内容
2. 删除 `.git` 目录
3. 重新初始化
4. 按策略筛选添加文件
5. 新的初始化提交

## 文件命名规范

所有变更日志文件必须使用**日期前缀**命名：

```
YYYYMMDD-变更简述.md
```

| 格式 | 示例 | 说明 |
|------|------|------|
| `YYYYMMDD-简述.md` | `20260211-身份设定整合.md` | 具体变更记录 |
| `YYYYMMDD-YYYYMMDD-阶段汇总.md` | `20260201-20260228-二月变更汇总.md` | 阶段性汇总 |

---

## 何时创建新的 Changelog

| 场景 | 操作 |
|------|------|
| 规范有重大变更 | 创建 `YYYYMMDD-简述.md` 详细说明 |
| 发现规范问题并修复 | 创建独立文件记录 |
| 阶段性总结 | 创建 `YYYYMMDD-YYYYMMDD-阶段汇总.md` |
| 个人反馈记录 | 在 `Areas/My_Persona/changelogs/` 中记录 |

## 变更记录模板

创建新的变更记录文件时，使用以下模板：

```markdown
# YYYY-MM-DD: 变更简述

**类型**: [规范新增/规范修改/问题修复/经验沉淀/基础设施]

## 背景

描述变更的上下文和原因。

## 问题/需求

- 原规范有什么问题？
- 什么场景下发现了不足？

## 改进方案

1. 具体修改了哪些内容？
2. 新方案如何解决原问题？

## 已更新文件

- `../XX-文件名.md` - 具体修改
- `../SKILL.md` - 版本号更新

## 验证方式

- [ ] 检查项 1
- [ ] 检查项 2

## 关联文档

- [[../01-开发指南/规范体系管理|规范体系管理]]: 关联的规范文档
```

## 管理流程

### 发现规范问题

```
发现问题 → 创建 YYYYMMDD-简述.md 记录
       → 更新相关规范文档
       → 更新本 README 索引
```

### 规范变更流程

```
1. 在 changelogs/ 创建 YYYYMMDD-简述.md
2. 更新对应的规范文档
3. 更新 SKILL.md 版本号
4. 更新本 README 索引
5. 更新 00-入口导航.md 如有新增文档
```

---

## 演进原则

1. **及时记录**: 发现问题立即记录，不拖延
2. **小步快跑**: 每次改进范围可控，快速验证
3. **经验沉淀**: 将实践验证的经验写入规范
4. **持续迭代**: 规范不是一成不变的，随实践演进

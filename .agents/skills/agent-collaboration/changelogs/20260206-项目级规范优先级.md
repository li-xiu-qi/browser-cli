# 2026-02-06: 新增项目级规范优先级规则

**类型**: 规范新增

## 背景

AI 在 Projects/ERNIE-SQL/ 项目协作时，未遵循项目 README 中的特定规范（如"禁止重复下载数据"），而是按照全局规范执行了数据下载。

## 问题

- 项目已有本地数据，但 AI 重复下载导致时间和带宽浪费
- 项目特定的流程规范（如训练顺序）未被遵循
- 全局规范与项目需求冲突时，AI 不知以何为准

## 改进

1. 建立"项目级规范优先级最高"原则
2. 更新 AGENTS.md，添加规范优先级表格
3. 更新 [[../01-开发指南/规范体系管理|规范体系管理]]，明确项目 README 的覆盖规则
4. 更新 [[../00-基础规范/_入口导航|入口导航]]，在规范地图前添加优先级说明
5. 制定项目 README 规范模板（含 Project-Specific Rules 章节）

## 已更新

- `AGENTS.md` - 添加规范优先级章节，版本升级至 2.8.0
- `01-开发指南/规范体系管理.md` - 添加项目级规范优先级规则

## 项目特定规范 (Project-Specific Rules)

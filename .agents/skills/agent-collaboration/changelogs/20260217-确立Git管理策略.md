# 2026-02-17: 确立 Git 管理策略

**类型**: 基础设施

## 背景

随着知识库规模扩大，需要明确的 Git 管理策略来控制仓库体积，同时保留核心资产（笔记和 Skill 定义）。

## 策略决策

经过讨论，确定采用 **方案 A：分层 Git 管理** - 在 Projects/Resources 目录使用 `.gitignore` 只保留 Markdown 文件。

## 实施内容

### 1. 更新根目录 `.gitignore`

- 添加注释说明 Projects 和 Resources 的独立管理策略
- 保持已有的忽略规则（Obsidian、AI 工具、大文件等）

### 2. 创建 `Projects/.gitignore`

策略：
- 忽略所有文件 (`*`)
- 保留目录结构 (`!*/`)
- 保留 Markdown 文件 (`!*.md`, `!*.markdown`)
- 保留项目级 Skill (`!.agents/`)
- 保留 `.gitignore` 本身

效果：只提交笔记和索引，不提交代码项目。

### 3. 创建 `Resources/.gitignore`

策略：
- 同上基础规则
- 额外明确忽略各类资源文件：
  - 电子书 (pdf, epub, mobi)
  - 媒体文件 (mp4, mp3, wav)
  - 图片 (jpg, png, gif)
  - 压缩包 (zip, 7z, tar.gz)
  - 数据文件 (json, csv, xml)
  - 代码文件 (js, py, java, go 等)
  - 开发相关 (node_modules, .env, build/)

效果：只提交资源索引（Markdown），不提交实际资源文件。

## Git 管理原则

### 需要跟踪的内容

| 类别 | 路径 | 说明 |
|------|------|------|
| 笔记核心 | `Areas/`, `Archives/`, `日记/` | PARA 结构的 Markdown 笔记 |
| Skill 定义 | `.agents/skills/` | 所有 Skill 规范（最重要） |
| 工作流 | `.agents/workflows/` | 公共工作流定义 |
| 项目笔记 | `Projects/**/*.md` | 项目说明和笔记（不含代码） |
| 资源索引 | `Resources/**/*.md` | 资源索引和笔记（不含资源文件） |
| 配置 | `AGENTS.md`, `package.json`, `pyproject.toml` | 项目配置 |

### 需要忽略的内容

| 类别 | 说明 |
|------|------|
| 依赖 | `node_modules/`, `.venv/` |
| AI 工具配置 | `.claude/`, `.cursor/`, `.gemini/` 等 |
| 浏览器数据 | `.agents/browser_user_data/` |
| 大文件 | `.pdf`, `.mp4`, `.zip` 等 |
| 代码项目 | Projects 中的代码（只保留 md） |
| 资源文件 | Resources 中的实际资源（只保留 md 索引） |

## 已更新文件

- `.gitignore` - 添加策略说明注释
- `Projects/.gitignore` - 新建，只保留 md
- `Resources/.gitignore` - 新建，只保留 md

## 后续操作

执行以下命令完成 Git 配置：

```bash
# 1. 检查 Git 状态
git status

# 2. 添加核心文件
git add AGENTS.md
.gitignore
.agents/skills/
.agents/workflows/
Areas/
Archives/
日记/
templates/

# 3. 添加 Projects 和 Resources 的 .gitignore
git add Projects/.gitignore Resources/.gitignore

# 4. 添加 Projects 和 Resources 中的 md 文件
git add Projects/**/*.md Resources/**/*.md

# 5. 提交
git commit -m "chore: 确立 Git 管理策略，只保留核心笔记和 Skill"
```

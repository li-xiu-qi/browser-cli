# 写作规范

**定位**: 笔记写作的基础规范  
**适用范围**: 所有 Markdown 笔记文件

---

## 语言风格

### 基本原则

- 使用直白词语，不用成语和营销词
- 不用生活化比喻
- 标点尽量少，句子要清楚
- 对话统一使用中文，必要时用英文缩写，不做中英对照

### 禁止事项

| 禁止 | 说明 | 示例 |
|------|------|------|
| 括号 | 严禁使用任何形式的括号 | ~~（补充说明）~~ → 重写句子 |
| Emoji | 零容忍 | ~~~~ → 删除 |
| 箭头符号 | 正文不使用 | ~~→~~ → 用"导致"、"指向"等词 |
| 连续加粗 | 严禁通篇加粗 | ~~**这**是**错误**的~~ |
| 长从句 | 一行一个主逻辑 | 拆分为多个短句 |
| 引号与加粗混用 | 加粗内不套引号，避免渲染问题 | ~~**"个人博客"**~~ → **个人博客** |

### 格式规范

- **线性输出**: 一行聚焦一个主逻辑
- **加粗克制**: 仅极少数核心强调词使用加粗
- **段落短小**: 用短段落和列表，提高可读性

## 标题层级

### 使用规范

| 层级 | 用途 | 示例 |
|------|------|------|
| 一级 `#` | 文件主标题 | `# 写作规范` |
| 二级 `##` | 主要章节 | `## 语言风格` |
| 三级 `###` | 子章节 | `### 基本原则` |
| 四级 `####` | 细分内容 | `#### 禁止事项` |

### 规则

- 一级标题仅用于文件主标题
- 避免超过四级标题
- 同级标题之间要有逻辑连贯性

## 列表使用

### 有序列表

用于步骤、流程、优先级：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 无序列表

用于并列项、列举：

```markdown
- 项目 A
- 项目 B
- 项目 C
```

### 规则

- 列表项不宜过长，保持在一行内
- 同级列表项保持平行结构
- 避免列表嵌套超过两层

## Markdown 格式陷阱

| 问题 | 错误示例 | 正确写法 |
|------|----------|----------|
| 加粗与正文粘连无空行 | `**加粗**正文` | `**加粗** 正文` 或加粗单独成行 |
| 加粗内部有空格 | `** 加粗 **` | `**加粗**`（星号内侧不能有空格） |
| 加粗后紧跟标点无空格 | `**加粗**：` | `**加粗**：`（某些渲染器会失效） |
| 加粗与列表混用无空行 | `- **加粗**`<br>`正文` | `- **加粗**`<br><br>`正文` |

**原则**：加粗标记 `**文本**` 与后续内容之间建议保留空行，确保 Obsidian 等工具能正确渲染。

## 文档来源规范（强制）

基于官方文档生成的笔记，**必须**附上原始文档地址。

**Frontmatter 要求**：

```yaml
---
source: https://official-docs.example.com/guide
---

# 笔记标题

**官方文档**: https://official-docs.example.com/guide
```

**多来源情况**：

```yaml
---
source:
  - https://docs.example.com/guide
  - https://github.com/example/repo/blob/main/README.md
---
```

**原则**：
- 任何从外部官方文档提取的内容，都要可追溯
- 优先使用官方文档而非第三方转载
- 在笔记开头显式标注官方文档链接，便于后续核对更新

## 图表绘制规范（强制）

### 优先使用 Graphviz

**禁止**：使用 ASCII 图（字符画）表示流程、层级或关系图。

**错误示例**（不要使用）：
```markdown
```
L0 → L1 → L2 → L3
  ↓    ↓    ↓
  A    B    C
```
```

**正确做法**：使用 Graphviz 生成专业 SVG 图表。

### Graphviz 使用规范

1. **阅读最佳实践**
   - 绘图前必须阅读 [[text-visualization|Graphviz 最佳实践]]
   - 特别注意 viewBox 设置（必须为 "0,0" 开头）

2. **本地环境**
   - 本项目已配置本地 dot 环境
   - 无需连接 Kroki 服务器

3. **生成命令**
   ```bash
   # 单个文件
   dot -Tsvg assets/diagram.dot -o assets/diagram.svg
   
   # 或使用 npm 脚本
   npm run diagram:render assets/diagram.dot
   npm run diagram:batch assets/
   ```

4. **存放位置**
   - `.dot` 源文件：`assets/` 文件夹
   - 生成的 `.svg`：同目录
   - 笔记中引用：`![描述](assets/diagram.svg)`

### 适用场景

| 场景 | 推荐方式 | 示例 |
|------|---------|------|
| 层级结构 | Graphviz SVG | 六层模型图 |
| 流程步骤 | Graphviz SVG | 工作流图 |
| 对比关系 | Graphviz SVG | 对比矩阵 |
| 简单线性流程 | 文字描述或表格 | "步骤1 → 步骤2 → 步骤3" |

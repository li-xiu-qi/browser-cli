# PARA 方法论

**定位**: 知识库的组织原则  
**核心**: Projects, Areas, Resources, Archives 四类别

---

## 四类别定义

| 类别 | 用途 | 特点 | 问题问自己 |
|------|------|------|-----------|
| **Inbox** | 临时收集未分类内容 | 定期清理（建议每周） | 这个需要立即处理吗？ |
| **Projects** | 短期任务，有明确截止日期 | 完成后必须移至 Archives | 有明确截止时间和目标吗？ |
| **Areas** | 长期维护的责任领域 | 核心资产库，结构稳定 | 需要长期持续维护吗？ |
| **Resources** | 参考资料、附件、课程资料 | 可长期保存的参考材料 | 只是参考资料吗？ |
| **Archives** | 已完成项目的归档 | 只读状态，不删除只归档 | 项目已结束吗？ |

## 目录结构

```
笔记专用/
├── 0_Inbox/              # 收集箱
├── Projects/             # 项目
│   └── YYYY-MM-项目名/   # 按日期命名
├── Areas/                # 领域
│   ├── AI协作学/         # AI 协作规范
│   ├── My_Persona/       # AI 身份设定
│   ├── Scripts/          # 脚本工具
│   ├── 文本制图/         # 图表工具与示例
│   └── ...               # 其他领域
├── Resources/            # 资源
│   ├── 课程资料/
│   ├── 书籍/
│   └── 附件/
├── Archives/             # 归档
│   └── YYYY/             # 按年份归档
└── 日记/                 # 日常记录
    └── YYYY-MM-DD.md
```

## 文件流转规则

### 流入规则

```
新内容 → 0_Inbox → 分类整理 → PARA 四类别
```

### 流出规则

```
Projects 完成 → Archives
Areas 过时 → Archives
Resources 无用 → 删除
```

### 定期维护

| 周期 | 任务 |
|------|------|
| 每周 | 清理 0_Inbox |
| 每月 | 检查 Projects，完成的归档 |
| 每季度 | 回顾 Areas，更新长期目标 |

---

## 分类决策树

```
这个文件应该放哪里？
├── 需要立即处理？
│   └── 是 → 0_Inbox
│
├── 是短期任务，有明确截止时间？
│   └── 是 → Projects/
│
├── 需要长期维护，是责任领域？
│   └── 是 → Areas/
│
├── 只是参考资料？
│   └── 是 → Resources/
│
└── 项目已结束？
    └── 是 → Archives/
```

## 命名规范

### 项目

格式: `YYYY-MM-项目名/`

示例:
- `2026-02-案例库建设/`
- `2026-03-网站重构/`

### 领域

格式: `领域名/`

示例:
- `AI协作学/`
- `科学决策/`

### 笔记文件

格式: `主题名.md`

示例:
- `写作规范.md`
- `五层决策模型.md`

---

## 标签使用

### 标签 vs 文件夹

- **文件夹**: 用于 PARA 分类，结构性强
- **标签**: 用于主题归类，灵活性高

### 建议标签

| 类型 | 示例 |
|------|------|
| 状态 | #待处理 #进行中 #已完成 |
| 主题 | #决策 #写作 #项目管理 |
| 来源 | #一堂 #书籍 #文章 |
| 优先级 | #P0 #P1 #P2 |

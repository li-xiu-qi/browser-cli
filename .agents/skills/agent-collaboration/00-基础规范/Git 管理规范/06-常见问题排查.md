# 常见问题排查

> Git 使用中常见问题的诊断和解决方案

---

## 提交相关

### 问题：提交信息写错了

**场景**：刚提交完发现信息写错了

**解决**：

```bash
# 修改最后一次提交信息
git commit --amend -m "正确的提交信息"

# 如果已经推送到远程（强制推送，谨慎使用）
git push origin master --force
```

---

### 问题：漏添加了文件

**场景**：提交后发现漏了一个文件

**解决**：

```bash
# 添加漏掉的文件
git add 漏掉的文件

# 追加到最后一次提交
git commit --amend --no-edit
```

---

### 问题：不小心提交了敏感信息

**场景**：提交了含 API Key 或密码的文件

**解决**：

```bash
# 1. 从 Git 历史中完全移除文件
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch 文件路径' \
  HEAD

# 2. 添加到 .gitignore
echo "文件路径" >> .gitignore

# 3. 提交更改
git add .gitignore
git commit -m "chore: 移除敏感信息并更新忽略规则"

# 4. 强制推送（覆盖远程历史）
git push origin master --force
```

## 推送相关

### 问题：推送被拒绝（非快进）

**场景**：`git push` 显示 `rejected: non-fast-forward`

**原因**：远程仓库有本地没有的提交

**解决**：

```bash
# 先拉取远程变更
git pull origin master

# 如果有冲突，解决后提交
git add .
git commit -m "chore: 合并远程变更"

# 再推送
git push origin master
```

---

### 问题：推送被拒绝（权限不足）

**场景**：`git push` 显示 `Permission denied`

**原因**：没有推送权限或认证失败

**解决**：

```bash
# 检查远程地址
git remote -v

# 如果是 HTTPS，检查凭据
git config --list | grep credential

# 如果是 SSH，检查密钥
ls -la ~/.ssh/

# 测试连接
ssh -T git@github.com
```

## 冲突相关

### 问题：合并冲突

**场景**：`git pull` 或 `git merge` 显示冲突

**症状**：
```
CONFLICT (content): Merge conflict in 文件.md
Automatic merge failed; fix conflicts and commit
```

**解决**：

```bash
# 1. 查看冲突文件
git status

# 2. 打开冲突文件，找到冲突标记
#    <<<<<<< HEAD
#    本地版本的内容
#    =======
#    远程版本的内容
#    >>>>>>> branch-name

# 3. 手动编辑，保留需要的内容，删除冲突标记

# 4. 标记冲突已解决
git add 冲突文件

# 5. 完成合并
git commit -m "chore: 解决合并冲突"
```

---

### 问题：目录结构冲突

**场景**：两边同时移动了同一个文件

**解决**：

```bash
# 查看冲突状态
git status

# 决定保留哪个版本
# 或者手动合并内容

# 添加解决后的文件
git add .

# 提交
git commit -m "chore: 解决目录结构冲突"
```

## 撤销相关

### 问题：改错了文件，想恢复

**场景**：修改了文件但还没提交，想放弃修改

**解决**：

```bash
# 恢复单个文件
git checkout -- 文件路径

# 恢复所有未暂存的修改
git checkout -- .

# 恢复已暂存但未提交的文件
git reset HEAD 文件路径
git checkout -- 文件路径
```

---

### 问题：提交错了，想撤销

**场景**：提交了错误的内容

**解决**：

```bash
# 撤销最后一次提交，保留修改
git reset --soft HEAD~1

# 撤销最后一次提交，丢弃修改
git reset --hard HEAD~1

# 撤销已推送的提交（创建反向提交）
git revert 提交哈希
git push origin master
```

---

### 问题：误删了文件

**场景**：不小心删除了文件

**解决**：

```bash
# 如果还没提交删除操作，直接恢复
git checkout -- 删除的文件

# 如果已提交删除
git log --diff-filter=D --summary  # 查找删除记录

# 恢复到删除前的版本
git checkout 删除前哈希~1 -- 文件路径
git commit -m "chore: 恢复误删文件"
```

## 仓库相关

### 问题：仓库太大

**场景**：`.git` 目录占用空间很大

**诊断**：

```bash
# 查看仓库大小
du -sh .git

# 查看大对象
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ {print $3" "$4}' | \
  sort -rn | head -20
```

**解决**：

```bash
# 垃圾回收
git gc --aggressive --prune=now

# 清理大文件（如果已删除但未清理）
git reflog expire --expire=now --all
git gc --aggressive --prune=now
```

---

### 问题：.gitignore 不生效

**场景**：添加了 .gitignore 规则，但文件仍被跟踪

**原因**：文件已经被 Git 跟踪

**解决**：

```bash
# 从 Git 中移除，但保留本地文件
git rm --cached 文件路径

# 如果是目录
git rm -r --cached 目录路径

# 提交更改
git commit -m "chore: 更新忽略规则"
```

---

### 问题：换电脑后同步

**场景**：在新电脑上克隆仓库

**解决**：

```bash
# 克隆仓库
git clone 仓库地址

# 进入目录
cd 仓库名

# 配置本地用户名（可选）
git config user.name "你的名字"
git config user.email "你的邮箱"

# 配置提交模板
git config commit.template .gitmessage
```

## 历史相关

### 问题：查看某个文件的历史

**解决**：

```bash
# 查看文件的修改历史
git log -p 文件路径

# 查看文件的简略历史
git log --oneline 文件路径

# 查看谁在什么时候修改了文件
git blame 文件路径
```

---

### 问题：回退到历史版本

**解决**：

```bash
# 查看历史
git log --oneline

# 临时查看历史版本（不切换分支）
git show 提交哈希:文件路径 > 临时文件

# 恢复到历史版本
git checkout 提交哈希 -- 文件路径
git commit -m "chore: 恢复文件到历史版本"
```

## 多设备同步

### 问题：设备 A 和设备 B 冲突

**场景**：在两台设备上都修改了内容

**预防**：

```bash
# 工作前总是先拉取
git pull origin master

# 工作后及时推送
git push origin master
```

**解决冲突**：

```bash
# 拉取时出现冲突
git pull origin master

# 解决冲突（见上文"合并冲突"）
# ...

# 推送解决后的版本
git push origin master
```

## 诊断命令速查

```bash
# 查看当前状态
git status

# 查看详细差异
git diff

# 查看提交历史
git log --oneline -20

# 查看分支
git branch -a

# 查看远程
git remote -v

# 查看配置
git config --list

# 查看仓库大小
du -sh .git

# 查看哪些文件被跟踪
git ls-files

# 查看忽略规则是否生效
git check-ignore -v 文件路径
```

## 最佳实践

### 避免问题的建议

1. **经常提交**：小步快跑，便于回退
2. **及时推送**：减少多设备冲突
3. **写清楚提交信息**：方便后续查找
4. **使用 .gitignore**：避免提交不该提交的文件
5. **工作前拉取**：保持与远程同步

### 紧急恢复

如果搞砸了，记住这些救命命令：

```bash
# 查看所有操作记录（包括已删除的提交）
git reflog

# 恢复到任意历史状态
git reset --hard 哈希值

# 或者创建新分支保存当前状态
git checkout -b 备份分支
git checkout master
git reset --hard 哈希值
```

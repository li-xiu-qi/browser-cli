## 核心原则

### 提交什么？

| 类别 | 是否提交 | 说明 | 示例 |
|------|---------|------|------|
| **Markdown 笔记** | ✅ 必须 | 核心知识资产 | `Areas/哲学/*.md` |
| **Agent Skills** | ✅ 提交 | 可复用协作规范 | `.agents/skills/**` |
| **Agent Tools** | ✅ 提交 | 脚本和工具代码 | `.agents/tools/**` |
| **工作流配置** | ✅ 提交 | 标准化流程 | `.agents/workflows/**` |
| **MCP 模板** | ✅ 提交 | 示例配置 | `mcp.json.example` |
| **个人配置** | ❌ 忽略 | 敏感信息/偏好 | `settings.local.json`, `mcp.json` |
| **浏览器数据** | ❌ 忽略 | 登录态/缓存 | `browser_user_data/` |
| **开发环境** | ❌ 忽略 | 临时文件 | `node_modules/`, `__pycache__/` |
| **大文件** | ❌ 忽略 | 二进制资源 | PDF, 视频, 图片 > 1MB |

### 为什么这样设计？

知识库 Git 管理 ≠ 代码库 Git 管理

- **代码库**：协作、版本、回滚是关键
- **知识库**：内容沉淀、结构清晰、跨设备同步是关键

## 快速开始

### 1. 首次配置

```bash
# 配置用户名（每个仓库独立）
git config user.name "你的名字"
git config user.email "你的邮箱"

# 设置默认分支名
git config init.defaultBranch master

# 设置提交模板
git config commit.template .gitmessage
```

### 2. 日常提交流程

```bash
# 查看变更
git status

# 添加文件（建议分批）
git add Areas/哲学/新知识.md

# 提交（遵循规范）
git commit -m "feat: 添加康德伦理学笔记"

# 推送
git push origin master
```

### 3. 提交信息示例

```bash
# ✅ 好的提交
feat: 添加思维模型五层架构分类
refactor: 迁移经济学笔记到独立目录
chore: 更新哲学导航索引
fix: 修正辩证思维的概念定义
docs: 完善 Git 管理规范文档

# ❌ 避免的提交
update
修改
一大堆内容更新
```

## 场景指南

### 场景 1：新建学习项目

```bash
# 创建学习区文件
touch "Areas/_学习区/2026-Q2-机器学习.md"

# 编辑内容...

# 提交
git add Areas/_学习区/
git commit -m "feat: 启动机器学习学习项目"
```

### 场景 2：整理归档旧内容

```bash
# 移动文件到 Archives
mv Areas/过时内容.md Archives/2025-过时内容.md

# 分批提交
git add Areas/Archives/
git commit -m "refactor: 归档过期内容到 Archives"
```

### 场景 3：添加新的 Agent Skill

```bash
# 创建 Skill 目录和文件
mkdir -p .agents/skills/new-skill
touch .agents/skills/new-skill/SKILL.md

# 提交
git add .agents/skills/new-skill/
git commit -m "feat: 添加 xxx Skill"
```

## 目录结构规范

### 推荐的知识库目录

```
笔记专用/
├── .git/                   ← Git 仓库（自动生成）
├── .gitignore              ← 忽略规则（必须配置）
├── .gitattributes          ← 文件属性（可选）
├── .gitmessage             ← 提交模板（推荐）
│
├── Areas/                  ← 知识领域（核心内容，必须提交）
│   ├── _探索中/             ← 探索缓冲区
│   ├── _学习区/             ← 学习项目区
│   ├── 哲学/
│   ├── 经济学/
│   └── ...
│
├── Projects/               ← 项目产出（核心内容，必须提交）
│
├── MOC/                    ← 导航地图（核心内容，必须提交）
│
├── .agents/                ← Agent 配置（提交 Skills/Tools，忽略配置）
│   ├── skills/             ← ✅ 提交
│   ├── tools/              ← ✅ 提交
│   ├── workflows/          ← ✅ 提交
│   ├── mcp.json            ← ❌ 忽略（个人配置）
│   └── browser_user_data/  ← ❌ 忽略（缓存数据）
│
├── resources/              ← 资源文件（忽略大文件）
│   └── 参考项目/            ← ❌ 忽略（已在 .gitignore）
│
└── Archives/               ← 归档区（可选提交）
```

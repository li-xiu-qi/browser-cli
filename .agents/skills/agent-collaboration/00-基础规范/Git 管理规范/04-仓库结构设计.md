## 目录详细说明

### Areas/ - 知识领域（必须提交）

**用途**：存储核心知识内容

**结构**：
```
Areas/
├── _探索中/           ← 跨学科探索缓冲区
│   ├── _INDEX-探索中.md
│   └── 各种探索中的主题.md
│
├── _学习区/           ← 有期限的学习项目
│   ├── _INDEX-学习区.md
│   └── 2026-Q1-xxx.md
│
├── 哲学/              ← 学科领域
│   ├── _INDEX-哲学导航.md
│   └── 具体笔记.md
│
├── 经济学/
├── 心理学/
└── ...
```

**规范**：
- 使用 `_INDEX-` 前缀表示索引导航
- 使用 `_` 前缀表示特殊目录（缓冲区）
- 学科目录使用正式学科名称

### Projects/ - 项目产出（必须提交）

**用途**：存储产出导向的项目

**结构**：
```
Projects/
├── 📁 项目名称/
│   ├── README.md
│   └── 项目文件...
```

**与 Areas/ 的区别**：
- Areas = 知识积累（长期维护）
- Projects = 产出交付（有明确完成标准）

### MOC/ - 导航地图（必须提交）

**用途**：全局导航索引

**结构**：
```
MOC/
├── _INDEX-总览.md          ← 最顶层入口
├── _INDEX-思维模型.md       ← 思维模型导航
└── ...
```

### .agents/ - Agent 配置（混合）

**提交策略**：

| 目录/文件 | 策略 | 说明 |
|-----------|------|------|
| `skills/` | ✅ 提交 | Skill 定义和文档 |
| `tools/` | ✅ 提交 | 工具脚本和代码 |
| `workflows/` | ✅ 提交 | 工作流定义 |
| `mcp.json` | ❌ 忽略 | 个人 MCP 配置（含 API Key）|
| `settings.local.json` | ❌ 忽略 | 个人偏好设置 |
| `browser_user_data/` | ❌ 忽略 | 浏览器缓存和登录态 |

## 文件大小限制

### 推荐限制

| 文件类型 | 大小限制 | 处理方式 |
|---------|---------|---------|
| Markdown | 无限制 | 直接提交 |
| 图片 | < 1MB | 提交或忽略 |
| 图片 | > 1MB | 使用图床/云存储 |
| PDF | 任意 | 默认忽略，需要时例外 |
| 视频/音频 | 任意 | 必须忽略 |

### 检查大文件

```bash
# 查找大于 1MB 的文件
find . -type f -size +1M -not -path "./.git/*"

# 查找最大的 10 个文件
find . -type f -not -path "./.git/*" -exec ls -lh {} \; | sort -k5 -hr | head -10
```

## .gitattributes 配置（可选）

### 基础配置

```gitattributes
# 自动处理行尾符
* text=auto

# Markdown 文件使用 LF
*.md text eol=lf

# 二进制文件不处理
*.png binary
*.jpg binary
*.pdf binary
```

### 用途

- 统一行尾符（LF vs CRLF）
- 标记二进制文件
- 配置合并策略

## 提交模板配置

### .gitmessage 文件

```
# 类型: 简短描述（50字以内）
#
# 详细说明：
# -
# -
#
# 类型选项:
# feat:     新增内容/功能
# refactor: 重构/结构调整
# chore:    日常维护
# fix:      修正错误
# docs:     文档更新
# style:    格式调整
# merge:    合并分支
```

### 配置命令

```bash
git config commit.template .gitmessage
```

## 更新时必须同步的环节

### 1. 文件重命名/序号调整（必须同步）

**场景**: 调整规范文档的序号或文件名  
**影响**: AGENTS.md 中的链接会失效  
**操作**:
```
1. 修改 agent-collaboration 中的文件名
2. 同步更新 SKILL.md 中的目录结构
3. 同步更新 AGENTS.md 中的所有链接引用
4. 更新本文档的示例链接（如有）
```

**示例**:
```markdown
<!-- 修改前 -->
| 开发新的 Skill | [[agent-collaboration/01-开发指南/00-Skill开发规范]] |

<!-- 修改后 -->
| 开发新的 Skill | [[agent-collaboration/01-开发指南/00-Skill开发规范]] |
```

### 2. 新增规范文档（建议同步）

**场景**: 在 agent-collaboration 中新增规范  
**操作**:
```
1. 在 agent-collaboration 中创建文档
2. 更新 SKILL.md 目录结构和导航表
3. 在 AGENTS.md 的快速导航表中添加条目（如果是核心规范）
4. 更新版本号和变更记录
```

### 3. 修改核心原则（必须同步）

**场景**: 修改铁律、禁止事项、规范优先级等  
**影响**: AGENTS.md 和 Skill 中可能都有相关描述  
**操作**:
```
1. 确定唯一事实来源（通常在 Skill 中）
2. 修改 Skill 中的详细规范
3. 同步更新 AGENTS.md 中的精简版描述
4. 确保两者描述一致，不冲突
```

### 4. 版本号更新（独立维护）

| 文件 | 版本号规则 | 何时更新 |
|------|-----------|----------|
| AGENTS.md | 主版本.次版本.修订号 | 内容变更时 |
| SKILL.md | 主版本.次版本.修订号 | 内容变更时 |

**注意**: 两者版本号**独立维护**，不要求一致。

## 同步检查清单

修改以下任何内容后，执行此检查：

- [ ] **文件重命名**: AGENTS.md 中的链接是否更新？
- [ ] **目录结构调整**: SKILL.md 中的目录树是否更新？
- [ ] **新增文档**: AGENTS.md 快速导航是否需要添加？
- [ ] **核心原则变更**: 两处描述是否一致？
- [ ] **版本号**: 是否已递增？
- [ ] **变更记录**: 是否已记录？

## 常见错误

### ❌ 错误1: 只改一处

只修改了 agent-collaboration 中的文件名，忘记更新 AGENTS.md 的链接。

**后果**: AI 助手点击链接时找不到文件。

### ❌ 错误2: 路径混淆

AGENTS.md 中使用相对路径：
```markdown
<!-- 错误！在 AGENTS.md 中这样写无法正确跳转 -->
[[写作规范]]
```

### ❌ 错误3: 版本号不一致

强制让 AGENTS.md 和 SKILL.md 版本号保持一致。

**正确做法**: 两者独立维护，各自记录自己的变更。

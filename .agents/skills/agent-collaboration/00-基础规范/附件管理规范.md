# 附件管理规范

**定位**: 知识库附件的统一管理规则  
**适用范围**: 所有 Markdown 笔记的附件（图片、PDF、音频等）

---

## 核心原则

**强制规则**: 附件必须与笔记同级存放

```
笔记路径: 路径/笔记名.md
附件目录: 路径/assets/笔记名/
引用方式: ![描述](assets/笔记名/图片.png)
```

---

## 存放规则

### 必须遵守

| 规则 | 说明 | 示例 |
|------|------|------|
| 同级存放 | 附件目录与笔记文件在同一目录 | `Projects/xxx/笔记.md` + `Projects/xxx/assets/笔记/` |
| 命名一致 | 附件子目录名必须与笔记文件名一致 | 笔记 `写作规范.md` → 目录 `assets/写作规范/` |
| 相对路径 | 使用相对路径引用，非绝对路径 | `assets/笔记名/图片.png` |

### 禁止事项

- 全局附件文件夹（如 `attachments/` 或 `附件/`）
- 图片直接放笔记同级（未放入 `assets/` 子目录）
- 通用文件夹名如 `assets/images`（必须匹配文件名）

---

## 执行协议

### 处理违规笔记

当处理违反此规则的笔记时：

1. 若 `assets/<文件名>/` 目录不存在，则创建它
2. 将被引用的本地文件移动至该目录
3. 更新 Markdown 中的链接，使用相对路径 `assets/<文件名>/<文件名>`

### 重构现有笔记流程

当被要求修复或整理笔记图片时：

1. 识别所有本地图片链接
2. 检查其存放位置是否符合附件强制同级存放规则
3. 必要时执行文件移动并更新 Markdown 链接

---

## 检查清单

- [ ] 附件是否放在 `assets/笔记名/` 目录
- [ ] 子文件夹名是否与笔记文件名一致
- [ ] 引用路径是否为相对路径 `assets/笔记名/图片.png`

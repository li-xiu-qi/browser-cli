# 排版规范手册

**定位**: Obsidian Markdown 笔记的排版规范  
**核心**: 平衡简洁与可读性，格式服务于内容

---

## 核心原则

### 克制优先

- **格式服务于内容**，不喧宾夺主
- **能不用则不用**，必须用才用
- **少即是多**

### 一致性

- 同类元素使用相同格式
- 全篇风格统一

---

## 格式层级

### 一级：极简（默认）

**特点**:
- 仅使用标题层级
- 无加粗、无引用、无特殊符号

**适用场景**: 快速记录、草稿

**示例**:
```markdown
# 会议记录

## 参会人员
张三、李四、王五

## 讨论要点

### 议题一
内容...

### 议题二
内容...
```

---

### 二级：轻度装饰（推荐）

**特点**:
- 核心概念使用引用符号 `>`
- 关键结论适度加粗
- 数字、金额使用加粗

**适用场景**: 知识笔记、案例整理

**示例**:
```markdown
# 五层决策模型

> 决策质量取决于信息层、认知层、判断层、执行层、反馈层的协同

## 核心观点

普通人做决策只考虑**第一层**（表面信息），高手会深入到**第五层**（反馈优化）。

## 关键数据

- 决策失误率：**73%** 源于信息层问题
- 优化成本：后期修正成本是前期的 **10倍**
```

---

### 三级：结构化

**特点**:
- 步骤标题加粗
- 小节使用四级标题
- 关键段落引用

**适用场景**: 教程、操作指南

**示例**:
```markdown
# 如何建立个人知识库

## 第一步：确定收集范围

**核心原则**: 只收集对你有价值的

#### 信息源评估

- 权威性：作者是否有相关经验
- 时效性：内容是否过时
- 可操作性：是否能指导行动

## 第二步：选择工具

...
```

## 元素使用规范

### 引用符号 `>`

**使用场景**:
-  核心定义
-  金句总结
-  需要突出的段落

**禁止**:
-  大段引用（超过3行）
-  连续多段引用

---

### 加粗 `**文本**`

**使用场景**:
-  关键数字、金额（**100万**、**73%**）
-  核心结论（**关键是...**）
-  步骤序号（**第一步**：）

**禁止**:
-  连续多行加粗
-  整段加粗
-  为了加粗而加粗

---

### 四级标题 `####`

**使用场景**:
-  案例内部的小节划分
-  长段落的结构化

**禁止**:
-  滥用导致层级过深
-  仅包含一两句话的小节

---

### 链接 `[[ ]]`

Obsidian 使用 **wikilink** 格式进行内部链接：

**基本链接**:
```markdown
[[笔记名称]]
[[笔记名称|显示文本]]
```

**标题链接**:
```markdown
[[笔记名称#标题]]
[[#同一笔记内的标题]]
```

**块链接**:
```markdown
[[笔记名称#^块ID]]
```

**嵌入**:
```markdown
![[嵌入笔记]]
```

 **禁止标准 Markdown 链接语法 `[文本](路径)`，统一使用 wikilink。**

---

## 禁止事项

| 禁止项 | 原因 | 替代方案 |
|--------|------|----------|
| 连续多行加粗 | 视觉疲劳，失去重点 | 仅关键处加粗 |
| 大段引用 | 破坏阅读流 | 只引用核心句 |
| Emoji 符号 | 不专业，跨平台显示不一致 | 用文字表达 |
| 括号注释 | 打断阅读 | 脚注或另起一句 |
| 箭头符号 `->` | 非标准符号 | 用文字描述 |
| 标准 Markdown 链接 | 与 wikilink 混用 | 统一用 `[[ ]]` |
| 过多层级标题 | 结构混乱 | 控制在四级以内 |

## 常见错误示例

###  错误：过度装饰

```markdown
# **五层决策模型**

> **决策质量**取决于**信息层**、**认知层**、**判断层**、**执行层**、**反馈层**的**协同**

## **核心观点**

**普通人**做决策只考虑**第一层**（**表面信息**），**高手**会深入到**第五层**（**反馈优化**）。
```

**问题**:
- 标题不需要加粗
- 引用中过多加粗
- 正文连续加粗失去重点

---

###  正确：克制使用

```markdown
# 五层决策模型

> 决策质量取决于信息层、认知层、判断层、执行层、反馈层的协同

## 核心观点

普通人做决策只考虑**第一层**（表面信息），高手会深入到**第五层**（反馈优化）。
```

## 排版检查清单

发布笔记前，检查以下项目：

- [ ] 标题没有不必要的加粗
- [ ] 引用只用于核心观点
- [ ] 加粗不超过全文的 10%
- [ ] 使用 wikilink 而非标准链接
- [ ] 没有 Emoji 和特殊符号
- [ ] 层级不超过四级标题

# 规范体系管理

**定位**: AI 协作规范体系的元规则  
**作用**: 定义规范如何被创建、修改和维护

---

## 层级架构

```
项目 README.md (Projects/xxx/README.md) ← 最高优先级
    ↓ 覆盖
项目级 Skill (Projects/xxx/.agents/skills/)
    ↓ 覆盖
Areas/My_Persona/ (个人偏好)
    ↓ 覆盖
AGENTS.md (根目录全局规范)
    ↓ 引用
全局 Skill (.agents/skills/)
    ↓ 包含
具体规范文档
```

### 各层职责

| 层级 | 文件 | 职责 | 修改频率 | 优先级 |
|------|------|------|----------|--------|
| 项目层 | `Projects/xxx/README.md` | 项目特定规范 | 按项目 | **1 (最高)** |
| 项目 Skill | `Projects/xxx/.agents/skills/` | 项目专属 Skill | 按项目 | **2** |
| 个人偏好 | `Areas/My_Persona/` | 个人偏好设定 | 低 | **3** |
| 入口层 | `AGENTS.md` | 轻量级总入口 | 低 | **4** |
| 全局 Skill | `.agents/skills/` | 全局 Skill 定义 | 中 | **5** |
| 默认行为 | Skill 内部 | 默认规则 | - | **6** |

### 项目级规范优先级规则（重要！）

**规则：项目 README.md 中的规范优先级高于全局规范**

当 AI 在 Projects/ 目录下执行具体项目任务时：

1. **首先检查** 项目根目录是否有 README.md
2. **如果存在** "项目特定规范" 或 "Project-Specific Rules" 部分 → 优先遵循
3. **如果不存在** 或规范未覆盖 → 遵循全局 AGENTS.md 规范

**示例场景**：
- 全局规范："数据下载后存放到 data/raw/"
- 项目 README："使用本地已有数据，禁止重复下载"
- **结果**：以项目 README 为准，不下载数据

**项目 README 规范模板**：
```markdown
## 项目特定规范 (Project-Specific Rules)

 **重要**: 本项目的以下规范优先级高于全局 AGENTS.md 规范。

### 1. 数据使用规范
- 禁止重复下载数据
- 使用本地已有数据: `data/alpaca_zh/train.json`

### 2. 代码修改规范  
- 修改 train.py 时必须保持三种模式支持
- 数值必须使用 float() 转换

### 3. 执行顺序规范
- 必须按 SFT → Plan A → Plan B 顺序执行
```

## 修改规则（元规则）

### 规则 1：单一事实来源

每类信息只应存在于一个位置：

| 信息类型 | 唯一存放位置 | 禁止存放位置 |
|----------|--------------|--------------|
| 目录结构 | `06-项目目录结构.md` | AGENTS.md, 00-入口导航.md |
| 写作规范 | `01-写作规范.md` | AGENTS.md |
| PARA 分类 | `02-PARA方法论.md` | AGENTS.md |
| 规范索引 | `00-入口导航.md` | AGENTS.md |

### 规则 2：AGENTS.md 只放导航

AGENTS.md 内容限制：
- 只能包含**快速任务对照表**
- 只能做**链接引用**，不展开具体内容
- 版本号跟随 `agent-collaboration/` 内规范的重大更新

**修改 AGENTS.md 的场景**：
- 新增 Skill 类型
- 调整入口导航结构
- 版本号升级（当规范体系有重大变更）

### 规则 3：规范文档的自我管理

修改本目录下的规范文档时，必须：

1. **检查影响范围**
   ```
   修改 01-写作规范.md
   ├── 检查 00-入口导航.md 是否有相关引用 → 同步更新
   ├── 检查 SKILL.md 是否有相关描述 → 同步更新
   └── 检查 AGENTS.md 是否受影响 → 通常不需要
   ```

2. **更新版本日志**
   - 在 `changelogs/agent-collaboration-changelog.md` 中记录变更
   - 说明变更原因和影响

3. **验证一致性**
   - 修改后检查 `00-入口导航.md` 的链接是否有效
   - 检查相关文档中的引用是否一致

## 新增规范的流程

新增一个规范文档时：

1. **确定编号**
   - 查看现有编号，按顺序分配新号
   - 编号格式：`NN-规范名称.md`（NN 为两位数）

2. **更新索引**
   - 在 `00-入口导航.md` 中添加对应条目
   - 在 `SKILL.md` 中补充功能描述（如适用）

3. **建立关联**
   - 在文档底部添加"关联文档"章节
   - 使用 wikilink 链接到相关规范

4. **记录变更**
   - 在 `05-反馈与演进日志.md` 中记录新增

## 一致性检查清单

修改任何规范后，检查：

- [ ] 00-入口导航.md 中的引用和描述是否准确
- [ ] SKILL.md 中的功能描述是否过时
- [ ] 关联文档的 wikilink 是否有效
- [ ] changelogs/ 是否记录变更
- [ ] AGENTS.md 是否需要同步（通常是版本号）

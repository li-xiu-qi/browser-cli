# 内容排版规范

**定位**: 统一的内容排版标准，覆盖从写作到发布的全流程  
**适用场景**: 
- 场景1: Obsidian 笔记排版（基础格式）
- 场景2: 公众号发布适配（平台限制处理）
- 场景3: 转写文稿润色（口语清洗）

---

## 场景1: Obsidian Markdown 排版

### 核心原则

**克制优先**
- 格式服务于内容，不喧宾夺主
- 能不用则不用，必须用才用
- 少即是多

**一致性**
- 同类元素使用相同格式
- 全篇风格统一

### 格式层级

| 层级 | 特征 | 适用场景 |
|------|------|----------|
| **极简** | 仅标题层级，无装饰 | 快速记录、草稿 |
| **轻度装饰** | 核心概念引用、关键数字加粗 | 知识笔记、案例整理 |
| **结构化** | 四级标题、步骤加粗 | 教程、操作指南 |

### 使用规范

**引用符号 `>` 的使用场景**
- 核心定义
- 金句总结
- 需要突出的段落

**加粗 `** **` 的使用场景**
- 关键数字、金额
- 核心结论
- 步骤序号（如 **第一步**：）

**四级标题 `####` 的使用场景**
- 案例内部的小节划分
- 长段落的结构化

### 禁止事项

- 连续多行加粗
- 大段引用
- Emoji 符号
- 括号注释
- 箭头符号 ->
- 标准 Markdown 链接 `[文本](路径)`（使用 wikilink）

### 链接规范

**Obsidian wikilink 格式**:
- `[[笔记名称]]`
- `[[笔记名称|显示文本]]`
- `[[笔记名称#标题]]`
- `[[#同一笔记内的标题]]`
- `![[嵌入笔记]]`

禁止标准 Markdown 链接语法 `[文本](路径)`。

## 场景2: 微信公众号发布适配

### 核心限制与解决方案

| 限制 | 问题 | 解决方案 |
|------|------|----------|
| 表格嵌图片 | 无法渲染表格内的 `<img>` | 使用**散开格式**（见下方示例） |
| 视频标签 | 不支持 `<video>` | 视频转 GIF（<5MB, <300帧） |
| LaTeX 公式 | 显示为源码或乱码 | 公式转图片插入 |

### 散开格式示例

**推荐做法**（表格嵌图片的替代方案）：

```markdown
DeepSeek-V3.2 回答：正确

![DeepSeek-V3.2 回答截图](deepseek_answer.png)

Kimi-K2.5 回答：正确

![Kimi-K2.5 回答截图](kimi_answer.png)
```

**避免做法**：
```markdown
| DeepSeek-V3.2 | Kimi-K2.5 |
|---------------|-----------|
| ![图](a.png)  | ![图](b.png) |
```

### 标点符号规范

| 符号 | 正确用法 | 错误用法 | 说明 |
|-----|---------|---------|------|
| 引号 | `"内容"` 或 `'内容'` | `「内容」` | 避免直角引号 |
| 破折号 | 少用，用句号替代 | `——` | 可能显示异常 |
| 加粗 | 克制使用 | `**大量加粗**` | 可能与主题色冲突 |

### 图片引用规范

**标准语法**：
```markdown
![Step-3.5-Flash 数据仪表盘：包含折线图、饼图、柱状图](../images/dashboard.png)
```

**不要用引用块包裹图片**：
```markdown
 不推荐：> ![图片描述](image.png)
 推荐：![图片描述](image.png)
```

**图片序号不加粗**：
```markdown
 不推荐：**图3：** 这是说明
 推荐：图3：这是说明
```

### 发布前检查清单

- [ ] 没有直角引号 `「」`
- [ ] 没有 `<video>` 标签（已转 GIF）
- [ ] 表格中没有嵌套图片
- [ ] LaTeX 公式已转为图片
- [ ] 图片大小 < 5MB
- [ ] 模型名使用全称
- [ ] 破折号使用克制

## 场景3: 转写文本润色

### 使用场景

- 处理语音转写后的粗糙文稿
- 自动添加标点、分段
- 提取标题
- 清洗口语废话

### 核心处理逻辑

**步骤一：读取与分析**
- 读取用户提供的文本或文件
- 通读内容，识别主题和逻辑脉络

**步骤二：结构重塑**
- **添加标题**: 插入 `#`, `##`, `###` 标题，确保清晰目录结构
- **智能分段**: 将大段语音流切分为逻辑独立的自然段，段落之间保留空行

**步骤三：格式美化**
- **标点补全**: 修正缺失或错误的标点符号
- **重点突出**: 
    - `**加粗**` 强调核心概念
    - `>` 引用标记案例、故事、引用语
    - `- ` 或 `1. ` 整理并列观点或步骤

**步骤四：内容净化**
- **去口语化**: 删除填充词（"那个"、"就是说"、"呃"、"啊"）
- **去重**: 精简重复啰嗦的表达，**严禁改变原意**
- **纠错**: 修正明显的语音识别错误

### 输出规范

- **直接修改模式**: 如用户指定文件，覆盖写入原文件（或指定新路径）
- **保留原意**: 编辑任务，不是重写任务。不随意删减实质性内容

## 高级模板

### 索引+案例文档模式

适用于收集和整理一类案例。

```
主题目录/
├── _INDEX-主题索引.md      # 入口文件，分类导航
├── 案例A.md                 # 单个案例详细记录
└── ...
```

### 原则/方法论收集模板

适用于收集体系化的方法论、原则框架。

```markdown
# 原则名称

来源：课程/书籍/实践  
适用场景：XXX

---

## 一句话概括
用一句话说明核心。

## 核心定义
> 金句式定义

## 为什么重要
- 没有会怎样
- 有了会怎样

## 具体做法
| 场景 | 做法 |
|------|------|
| 场景A | 具体行动 |

## 常见误区
| 误区 | 纠正 |
|------|------|
| 错误做法 | 正确做法 |
```

### 术语与定律表

适用于收集专业术语、定律、概念。

```markdown
| 术语 | 定义 | 用途 | 关联 |
|------|------|------|------|
| CAC | 获客成本 | 衡量营销效率 | [[营销笔记]] |
```

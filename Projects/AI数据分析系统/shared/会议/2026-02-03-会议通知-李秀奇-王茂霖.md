## 会议议题

### 一、项目背景与战略定位（10分钟）

**项目概述**：
AIASys 是面向工业制造的智能数据分析平台，采用 Host-Worker 双 Agent 架构，支持设备数据分析、智能故障诊断、参数预测优化等核心能力。

**核心功能**：
- 设备数据分析（振动、频谱、趋势、异常检测）
- 智能故障诊断（检测、分类、严重性评估、维护建议）
- 参数预测优化（时间序列预测、参数辨识、优化）
- 智能对话（基于大语言模型的自然语言交互）
- Notebook 回放（分析过程可追溯、可复现）

**支持设备**：轧机、电机、减速器、轴承

**需对齐的基础问题**：

| 问题 | 目的 | 当前待确认 |
|------|------|------------|
| 项目起源 | 为什么启动智能制造分析平台？解决什么业务痛点？ | 待对齐 |
| 战略定位 | 在工业互联网/智能制造战略中的位置？ | 待对齐 |
| 商业目标 | 对内降本增效 vs 对外产品化创收？ | 待对齐 |
| ROI预期 | 投入产出比预期？盈亏平衡点？ | 待对齐 |

**关键确认点**：
- 这个项目是必须完成，还是锦上添花？
- 如果延期或失败，对智能制造转型的影响？
- 公司愿意为工业 AI 平台投入多少资源上限？

### 二、项目现状与风险评估（10分钟）

同步当前团队配置与实际风险：

| 模块 | 技术栈 | 当前配置 | 风险说明 |
|------|--------|----------|----------|
| 前端 | React 19 + TypeScript | 1名实习生（身体不适，产出低） | 高 — 可能断档 |
| 后端 | Python + FastAPI + smolagents | 项目负责人1人 | 高 — 单点故障 |
| 算法 | Jupyter + LLM | 实际2人参与（名义3人） | 中 — 资源虚高 |

**核心矛盾**：
- 实际可用人力：2.5人（后端+算法 + 前端实习生有风险）
- 项目需求人力：至少4人（前后端分离、Agent 架构、算法工程）
- 技术复杂度：Host-Worker 双 Agent 架构、Jupyter 集成、多设备支持

**需确认**：是否了解这些风险？对当前状况的看法？

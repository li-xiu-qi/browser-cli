# AIASys 招聘计划与面试指南

> 科学招聘四部曲：做判断 → 写JD → 找线索 → 做评估

---

## 一、做判断 — 要不要招？

### 当前缺口分析

| 缺口 | 类型 | 判断 | 结论 |
|------|------|------|------|
| 前端开发 | 缺执行（人手不足） | 实习生风险高，需backup或替换 | **需要招** |
| 后端Backup | 缺执行（单点故障） | 我请假项目停摆 | **需要招/培养** |
| 算法专家 | 缺认知（Agent架构） | 可用咨询替代 | 暂时不招全职 |

### 招聘决策

**确定要招**：
1. **前端开发实习生/初级**（1名）— 替代或补充现有高风险实习生
2. **后端开发实习生**（1名）— 作为backup培养

**暂不招**：
- 算法专家 — 用外部咨询+自学解决

### 替代方案对比

| 缺口 | 招聘方案 | 替代方案 | 推荐 |
|------|----------|----------|------|
| 前端 | 招实习生（3K/月，培养3个月） | 我接手前端（耗时50%） | **招聘**，否则后端无人做 |
| 后端Backup | 招实习生（3K/月，培养3个月） | 王茂霖做决策backup | **两者并行**，短期靠A，长期招人 |

**与A对齐**：
- "前端确定需要招人，否则我兼顾前后端，项目进度不可控"
- "后端backup建议短期靠您决策，长期招实习生培养，预算约 __________/月"

## 二、写JD — 职位描述

### 岗位1：前端开发实习生

**JD公式**：输入 + 过程 + 输出

```
【岗位】前端开发实习生（AI数据分析平台）

【输入】
- 设计稿和产品需求文档
- 后端API接口文档
- 现有组件库和代码规范

【过程】
- 使用React+TypeScript开发Web界面
- 使用Tailwind CSS实现响应式布局
- 与后端联调API接口
- 按Git工作流提交代码和PR

【输出】
- 高质量的前端页面（还原度≥90%）
- 可复用的React组件
- 代码通过Code Review
- 2周内能独立交付小功能

【要求】
- 计算机相关专业，有一定React基础
- 熟悉HTML/CSS/JavaScript
- 有学习意愿，能接受14天魔鬼训练
- 实习期≥3个月，每周≥4天

【培养路径】
- 第1-2周：14天魔鬼训练（C级→B级）
- 第3-8周：独立执行任务（稳定在B级）
- 第9-12周：独立负责模块（冲击A级）

【我们能提供】
- 完整的培养体系和导师带教
- 参与工业AI平台的实战项目
- React 19 + TypeScript 5.9前沿技术栈
- 实习证明和推荐信（表现优异者）
```

### 岗位2：后端开发实习生

```
【岗位】后端开发实习生（AI数据分析平台）

【输入】
- 产品需求文档

- 3个月后能独立负责模块

【要求】
- 计算机相关专业，熟悉Python
- 了解HTTP协议和RESTful API
- 有数据库基础（SQL）
- 实习期≥3个月，每周≥4天

【培养路径】
- 第1个月：熟悉技术栈，开发简单接口
- 第2-3个月：独立负责功能模块
- 长期：向A级独立开发者进阶

【我们能提供】
- FastAPI + smolagents + Jupyter实战

- 实习证明和转正机会
```

## 四、做评估 — 面试筛选

### 面试流程

```
简历筛选 → 电话初筛（15min）→ 技术面试（30min）→ 综合面试（30min）→ Offer
```

### 筛选模型（优先级排序）

| 优先级 | 考察项 | 前端 | 后端 | 评估方式 |
|--------|--------|------|------|----------|
| **P0** | 学习能力 | 快速掌握新技术 | 快速理解架构 | 过往项目经历、学习路径 |
| **P0** | 基础扎实 | React/JS基础 | Python/HTTP基础 | 现场编程题 |
| **P1** | 自驱力 | 主动解决问题 | 主动查文档 | 询问遇到困难如何处理 |
| **P1** | 代码规范 | Git使用规范 | 代码可读性 | 看GitHub代码 |
| **P2** | 项目经验 | 有完整项目 | 有API开发经验 | 项目细节追问 |

**红线（一票否决）**：
- 简历造假
- 无法解释自己写的代码
- 态度不端正（敷衍、傲慢）

### 前端面试题

**基础题**（必问）：
1. 解释React的useState和useEffect
2. 什么是组件，如何复用？
3. CSS如何实现水平垂直居中？

**现场编程**（15分钟）：
```
实现一个简单的计数器组件：
- 显示当前数字
- 点击+1按钮数字加1
- 点击-1按钮数字减1
- 使用React+TypeScript
```

**追问**：
- 如何优化组件性能？
- 如果数字要在多个组件共享，怎么办？

### 后端面试题

**基础题**（必问）：
1. HTTP的GET和POST有什么区别？
2. 什么是RESTful API？
3. 数据库表如何设计？（给一个简单场景）

**现场编程**（15分钟）：
```
用Python实现一个简单的API：
- GET /items 返回物品列表
- POST /items 创建新物品
- 数据存在内存中（list或dict）
- 可以使用Flask或FastAPI
```

**追问**：
- 如果数据要持久化，怎么办？
- 如何处理并发请求？

### 行为面试（STAR原则）

**问题**：说说你遇到最大的技术挑战是什么？

**追问**（考察思维）：
- Situation：当时是什么情况？
- Task：你的任务是什么？
- Action：你做了什么？
- Result：结果如何？

### 面试评估表

| 候选人 | 基础 | 学习力 | 自驱力 | 代码 | 态度 | 综合 | 结论 |
|--------|------|--------|--------|------|------|------|------|
| __________ | /10 | /10 | /10 | /10 | /10 | /50 | ⬜ 通过 / ⬜ 待定 / ⬜ 拒绝 |
| __________ | /10 | /10 | /10 | /10 | /10 | /50 | ⬜ 通过 / ⬜ 待定 / ⬜ 拒绝 |

**通过标准**：综合≥35分，单项不低于6分

## 五、14天魔鬼训练（Landing计划）

### 入职即启动

**Day 1**：
- 确定段位：C级（学习期）
- 明确目标：14天后达到B级（能独立执行）
- 签署训练承诺

**训练内容**（详见 [[团队培养机制]]）：

| 时间 | 任务 | 验收标准 |
|------|------|----------|
| Day 1-3 | 熟悉项目 | 能独立运行，提交首个PR |
| Day 4-6 | 仿写实现 | 还原度90%+ |
| Day 7-9 | API联调 | 数据展示功能完成 |
| Day 10-12 | Bug修复 | 独立修复3个Bug |
| Day 13-14 | 独立交付 | 小功能模块验收通过 |

**不通过处理**：
- 延长1周继续训练
- 或协商离职

# AIASys 里程碑验收标准

> 明确"什么算完成"，避免后期扯皮

---

## 里程碑总览

| 阶段 | 里程碑 | 计划时间 | 验收人 |
|------|--------|----------|--------|
| 1 | 需求与方案确认 | W1 | A + 项目负责人 |
| 2 | 后端接口完成 | W3 | 项目负责人 |
| 3 | 前端框架完成 | W4 | 项目负责人 |
| 4 | 核心功能联调 | W6 | 项目负责人 + A |
| 5 | 内测通过 | W7 | 项目负责人 |
| 6 | 终版交付 | W8 | A |

---

## 详细验收标准

### 里程碑 1：需求与方案确认（W1）

**交付物**：
- [ ] PRD（产品需求文档）
- [ ] 技术方案文档
- [ ] 原型图/UI 设计稿

**验收标准**：
- [ ] A 确认功能范围与优先级
- [ ] 技术方案评审通过
- [ ] 原型图确认无重大修改

**不通过标准**：
- 核心功能有争议
- 技术方案存在不可行风险

---

### 里程碑 2：后端接口完成（W3）

**交付物**：
- [ ] API 接口文档（Swagger）
- [ ] 核心模块代码
- [ ] 单元测试报告

**验收标准**：
- [ ] __________ 个核心接口开发完成
- [ ] 单元测试覆盖率 ≥ __________%
- [ ] 接口文档完整可调用

**测试方法**：
- Postman 调用全部接口，返回符合预期
- 代码 review 无阻塞问题

---

### 里程碑 3：前端框架完成（W4）

**交付物**：
- [ ] 页面框架代码
- [ ] 组件库搭建
- [ ] 与后端接口联调 demo

**验收标准**：
- [ ] __________ 个核心页面框架完成
- [ ] 路由/状态管理搭建完成
- [ ] 至少 1 个端到端流程跑通

**测试方法**：
- 本地运行可看到页面框架
- 能调用后端接口获取数据

---

### 里程碑 4：核心功能联调（W6）

**交付物**：
- [ ] 可运行版本（deployed）
- [ ] 功能测试报告
- [ ] 已知问题清单

**验收标准**：
- [ ] 20% 核心功能全部可用
- [ ] __________ 个场景端到端跑通
- [ ] 无 P0/P1 级 bug

**测试方法**：
- 按照用户流程操作，能完成核心任务
- 性能指标：页面加载 < __________s

---

### 里程碑 5：内测通过（W7）

**交付物**：
- [ ] 测试报告
- [ ] Bug 修复记录
- [ ] 部署文档

**验收标准**：
- [ ] 内测用户 ≥ __________ 人
- [ ] 严重 bug 全部修复
- [ ] 部署文档完整，可独立部署

**不通过标准**：
- 存在影响使用的严重 bug
- 部署文档缺失关键步骤

---

### 里程碑 6：终版交付（W8）

**交付物**：
- [ ] 生产环境可访问链接
- [ ] 用户操作手册
- [ ] 项目总结文档

**验收标准**：
- [ ] A 确认功能符合预期
- [ ] 核心用户可独立使用
- [ ] 文档齐全，可交接维护

**签字确认**：
> 项目负责人：__________ 日期：__________  
> 批准者：__________ 日期：__________

---

## 验收流程

```
开发完成 → 自测 → 提交验收 → 验收人检查 → 
  ├─ 通过 → 进入下一阶段
  └─ 不通过 → 列出问题 → 修复 → 重新验收
```

**时间盒**：每个里程碑验收不超过 __________ 天

3. "终版交付时，您需要亲自操作验证，还是看测试报告即可？"

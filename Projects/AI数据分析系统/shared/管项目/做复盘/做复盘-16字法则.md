# AIASys 做复盘 — 16字法则

> 项目启动前准备复盘框架，确保项目结束后有效沉淀

---

## 复盘的价值

**不是追责，是进化**

- 好的：提炼成方法，下次复用
- 不好的：找到根因，避免再犯
- 模糊的：还原事实，形成共识

---

## 16字法则

```
目标回顾 → 结果评估 → 过程分析 → 总结提炼
```

## 复盘会议指南

### 会前准备

- [ ] 数据准备：进度、质量、成本数据
- [ ] 人员邀请：核心参与者必须到场
- [ ] 氛围营造：强调学习而非追责

### 会议流程

1. **目标回顾**（10分钟）：当初的目标是什么
2. **结果评估**（15分钟）：实际结果如何
3. **过程分析**（20分钟）：关键决策和得失
4. **总结提炼**（15分钟）：方法和教训
5. **行动计划**（10分钟）：下一步做什么

### 关键原则

- **对事不对人**：分析原因，不追究责任
- **数据说话**：用客观数据支撑判断
- **聚焦未来**：总结是为了下次更好

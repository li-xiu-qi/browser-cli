## 第三页：AIASys 产品定位

**幻灯片内容：**
- 一句话定位：面向工业制造的智能数据分析平台
- 核心价值：设备数据分析、智能故障诊断、参数预测优化
- 技术架构图：React + FastAPI + Host-Worker Agent + Jupyter
- 支持设备：轧机、电机、减速器、轴承

**演讲稿：**
首先回顾 AIASys 的现状。我们的产品定位是面向工业制造的智能数据分析平台，专注于设备数据分析、故障诊断和参数优化三大核心场景。

技术上我们采用 Host-Worker 双 Agent 架构，前端是 React，后端是 FastAPI，核心分析能力通过 Jupyter Kernel 实现。这种架构的优势是分析过程完全可追溯、可复现，这在工业场景中非常重要。

目前我们主要支持四类设备：轧机、电机、减速器和轴承。

**演讲稿：**
AIASys 的核心能力可以归纳为四个方面。第一是设备数据分析，包括振动分析、频谱分析等工业特有的分析方法。第二是智能故障诊断，这是我们的核心竞争力之一。第三是参数预测优化。第四是 Notebook 可复现性，这是工业级应用的关键需求。

## 第六页：DB-GPT 深度分析

**幻灯片内容：**
- 定位：AI 原生数据应用开发框架
- 核心特性：AWEL 工作流 + 多模型管理 + Text2SQL + RAG
- 技术架构图

**演讲稿：**
先看 DB-GPT。这是一个开源项目，目标是构建大模型领域的基础设施。它的核心特性包括 AWEL 工作流引擎、多模型管理、Text2SQL 和 RAG 框架。

DB-GPT 的优势是通用数据库接入能力强，SQL 生成能力成熟，开源生态活跃。但劣势也很明显：领域性弱，不适合工业场景的专业分析需求。

## 第七页：DB-GPT vs AIASys

**幻灯片内容：**
对比表格：
| 维度 | DB-GPT | AIASys |
|------|--------|--------|
| 目标用户 | 通用开发者 | 工业工程师 |
| 数据源 | 通用数据库 | 工业设备数据文件 |
| 分析能力 | SQL + 文本 | 信号处理 + 频谱分析 |
| Notebook | 无 | 内置 |
| 领域深度 | 通用 | 垂直深耕 |

**演讲稿：**
对比来看，DB-GPT 面向通用数据应用开发者，我们面向工业制造工程师。他们擅长数据库查询和文本分析，我们擅长信号处理和频谱分析。他们没有 Notebook 支持，这是我们的一大优势。

关键洞察：我们不需要在通用数据库能力上与 DB-GPT 竞争，而应该强化工业领域的垂直深度。

## 附录：可能的问答准备

**Q1：为什么我们不直接集成 DB-GPT，而要自己开发？**
A：DB-GPT 是通用框架，工业场景有很多特殊需求（信号处理、频谱分析、故障诊断模式）无法通过通用方案满足。我们需要在特定领域建立深度优势。

**Q2：InfiniSQL 和我们计划的 AIASysQL 有什么区别？**
A：InfiniSQL 面向企业通用数据分析，优化的是关系型数据库查询。AIASysQL 面向工业时序数据，需要针对 InfluxDB、TDengine 等时序数据库进行优化，同时内置工业领域的常用分析算子。

**Q3：工具市场的优先级是否过高？**
A：工具市场是中期的目标，短期重点是建立工具注册中心。这能为内部团队提供更好的工具共享机制，为后续开放做准备。

**Q4：Notebook 架构为什么必须保持核心地位？**
A：工业分析对可追溯性有严格要求，Notebook 可以完整记录分析过程，这是其他竞品不具备的优势，也是工业客户的关键需求。

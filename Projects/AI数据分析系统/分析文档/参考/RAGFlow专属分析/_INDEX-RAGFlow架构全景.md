##  核心洞察

### 1. 架构设计亮点

```
┌─────────────────────────────────────────────────────────────────┐
│                    RAGFlow 核心设计                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Canvas DSL (领域特定语言)                                    │
│     • 工作流定义 + 执行状态一体                                  │
│     • 支持断点恢复和历史回溯                                     │
│     • JSON序列化，易于存储和传输                                 │
│                                                                 │
│  2. 检索架构 (DeepDoc + Hybrid Search)                          │
│     • 文档解析：MinerU/Docling/OCR多策略                         │
│     • 索引：BM25 5% + 向量相似度 95%                             │
│     • 高级：GraphRAG + RAPTOR                                    │
│                                                                 │
│  3. 多租户 (行级隔离 + 索引隔离)                                 │
│     • 数据层：tenant_id字段过滤                                  │
│     • 检索层：ragflow_{tenant_id}独立索引                        │
│     • 权限：OWNER/NORMAL角色 + me/team资源权限                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 关键技术选型

| 层级 | 技术栈 | 选型理由 |
|------|--------|----------|
| 前端 | React 18 + Vite + Tailwind | 现代、快速、易维护 |
| 状态 | Zustand + React Query | 简洁、类型安全 |
| 后端 | Flask + Peewee | 轻量、Python生态 |
| 任务 | Celery + Redis | 成熟异步队列 |
| 存储 | MySQL + ES/Infinity + MinIO | 结构化+检索+对象存储 |
| 部署 | Docker + K8s Helm | 云原生支持 |

### 3. 一句话总结

> **RAGFlow = 基于 Canvas DSL 的可视化 Agent 编排平台 + DeepDoc 文档解析 + 多策略 RAG 检索 + 基础多租户隔离**

## 一、从一个经典误区开始

### 1.1 冰淇淋与溺水

**现象**: 
- 冰淇淋销量增加时，溺水死亡人数也增加
- 两者高度相关（相关系数 0.95+）

**错误结论**: 
> "吃冰淇淋会导致溺水！应该禁止销售冰淇淋！"

**真相**: 
- 冰淇淋销量 ↑ ← **夏天到来** → 游泳人数 ↑ → 溺水人数 ↑
- 冰淇淋和溺水只是**相关**，不是**因果**
- 真正的共同原因是：**气温/季节**

```
        夏天（气温高）
           /        \
          ↓          ↓
    冰淇淋销量    游泳人数
                          ↓
                      溺水人数
```

### 1.2 相关 ≠ 因果

| 关系类型 | 含义 | 例子 |
|---------|------|------|
| **相关** | 两个变量同时变化 | 冰淇淋销量 vs 溺水人数 |
| **因果** | A的变化导致B的变化 | 吸烟 vs 肺癌 |

**核心问题**: 相关性不等于因果性！

## 二、什么是因果发现（Causal Discovery）

### 2.1 定义

**因果发现** = 从观测数据中**自动找出**变量之间的因果关系

**通俗解释**: 
> 就像侦探破案，通过观察各种线索（数据），找出"谁是真凶"（真正的原因）

### 2.2 为什么要做因果发现？

**传统机器学习的局限**:
```python
# 传统ML：输入特征 → 预测结果
# 知道"是什么"，不知道"为什么"

模型.predict(X)  # 预测准确率95%
# 但它不知道为什么，也不知道干预会有什么效果
```

**因果发现的价值**:
```python
# 因果发现：找出变量间的因果关系
# 知道"为什么"，可以预测干预效果

因果图 = 因果发现算法(data)
# 发现：营销投入 → 销售额 ↑
#       季节因素 → 营销投入 ↑
#       季节因素 → 销售额 ↑

# 可以回答：如果我增加营销预算，销售额会增加多少？
```

### 2.3 因果发现算法示例

#### 算法1: PC算法（Peter-Clark算法）

**原理**: 通过条件独立性检验，逐步构建因果图

**简单例子**:

```
数据：1000个人的 [吸烟, 肺癌, 黄手指, 饮酒] 记录

Step 1: 发现相关性
- 吸烟 ↔ 肺癌 （相关）
- 吸烟 ↔ 黄手指 （相关）
- 肺癌 ↔ 饮酒 （相关？）

Step 2: 条件独立性检验
- 控制"吸烟"后，黄手指和肺癌不再相关
  → 黄手指 ← 吸烟 → 肺癌 （吸烟是共同原因）
  
- 控制"饮酒"后，吸烟和肺癌仍然相关
  → 吸烟 → 肺癌 （直接因果）

最终因果图：
        吸烟
       /    \
      ↓      ↓
    黄手指   肺癌
              ↑
           饮酒？（无直接因果）
```

#### 算法2: GES算法（贪婪等价搜索）

**原理**: 搜索评分最高的因果图结构

**例子**: 
```
数据：教育水平、收入、父母财富

可能因果图1: 父母财富 → 教育水平 → 收入
评分: 85分

可能因果图2: 教育水平 ← 父母财富 → 收入
            教育水平 → 收入
评分: 92分 ✓ （选择这个）
```

### 2.4 因果发现能做什么？

| 应用场景 | 例子 | 价值 |
|---------|------|------|
| **医学研究** | 发现药物→疗效的真正因果路径 | 避免副作用 |
| **营销策略** | 区分"营销→销售"还是"季节→两者" | 优化预算分配 |
| **产品优化** | 发现功能A→留存率提升 | 确定优先级 |
| **风险控制** | 发现风险因子→违约的因果链 | 精准风控 |

## 三、什么是因果推断（Causal Inference）

### 3.1 定义

**因果推断** = 基于已知的因果关系，**预测干预效果**

**通俗解释**: 
> 知道了"谁导致谁"，就可以回答"如果我做了什么，会发生什么"

### 3.2 核心问题

因果推断回答三类问题：

#### 问题1: 关联（Association）
> "吸烟的人肺癌发病率是多少？"

```
P(肺癌 | 吸烟) = 15%  （条件概率）
```

#### 问题2: 干预（Intervention）⭐
> **"如果强制所有人戒烟，肺癌发病率会降到多少？"**

```
P(肺癌 | do(戒烟)) = 2%  （干预后概率）

注意：不是 P(肺癌 | 戒烟)
因为"主动戒烟的人"和"被迫戒烟的人"可能不同
```

#### 问题3: 反事实（Counterfactual）⭐⭐
> **"如果小明当初没有吸烟，他现在会得肺癌吗？"**

```
现实：小明吸烟 → 得了肺癌
反事实：P(肺癌 | 小明, 如果不吸烟) = ?
```

### 3.3 为什么需要因果推断？

**例子**: 某电商平台发现
- 使用优惠券的用户，复购率 70%
- 不使用优惠券的用户，复购率 30%

**错误结论**: 
> "给所有用户发优惠券，复购率会从30%提升到70%！"

**考虑因果**: 
- 使用优惠券 ← **高价值用户** → 高复购率
- 优惠券本身可能只是"标记"了高价值用户
- 给低价值用户发优惠券，可能无效

**因果推断解决的问题**:
```
真实因果效应 = P(复购 | 发优惠券) - P(复购 | 不发优惠券)
            = 干预效果 - 自发效果
            = 实际提升幅度
```

### 3.4 常用因果推断方法

#### 方法1: Do-Calculus（Judea Pearl）

**核心**: 用"do算子"表示干预

```
P(Y | do(X))  # 干预X后Y的概率

vs 

P(Y | X)      # 观察到X时Y的概率（可能包含混杂因素）
```

**例子**: 
```
P(肺癌 | do(吸烟)) = 15%  # 干预：强制吸烟
P(肺癌 | 吸烟) = 12%      # 观察：吸烟者中（可能包含其他因素）
```

#### 方法2: 倾向得分匹配（Propensity Score Matching）

**问题**: 吸烟者和不吸烟者本身就不一样（年龄、性别等）

**解决**: 找"相似"的人对比

```
吸烟者小明：男，30岁，程序员
匹配的非吸烟者：男，31岁，程序员（尽量相似）

对比这两组，才能看出"吸烟"的真实效果
```

#### 方法3: 双重差分（Difference-in-Differences）

**场景**: 某城市实施新政策，看效果

```
              政策前    政策后    差分
实验组城市    100      150      +50
对照组城市    100      120      +20
                      
政策效果 = 50 - 20 = 30 （排除了整体趋势）
```

#### 方法4: 工具变量（Instrumental Variable）

**问题**: 教育水平 → 收入，但两者都受"能力"影响

**工具变量**: 找只影响教育、不直接影响收入的因素

```
工具变量：家附近是否有大学（影响上学便利性）

家附近有大学 → 更可能上大学 → 收入更高
        ↑                    ↓
      工具变量              结果

通过工具变量，可以分离出教育的"纯净"因果效应
```

## 四、因果发现 vs 因果推断

| 维度 | 因果发现 | 因果推断 |
|------|---------|---------|
| **阶段** | 第一步：找关系 | 第二步：用关系 |
| **输入** | 原始数据 | 因果图 + 数据 |
| **输出** | 因果图（谁导致谁） | 干预效果（如果...会怎样） |
| **问题** | "X和Y是什么关系？" | "如果我改变X，Y会怎样？" |
| **算法** | PC、GES、NOTEARS | Do-Calculus、PSM、DID |

### 流程图

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   原始数据       │ →  │   因果发现       │ →  │   因果图        │
│  [X, Y, Z, ...] │    │  (PC/GES算法)   │    │   X → Y ← Z     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                                     ↓
                                              ┌─────────────────┐
                                              │   因果推断       │
                                              │ (Do-Calculus等) │
                                              └─────────────────┘
                                                     ↓
                                              ┌─────────────────┐
                                              │   干预效果       │
                                              │ "如果做X，Y会    │
                                              │  变成什么"       │
                                              └─────────────────┘
```

## 五、实际应用案例

### 案例1: 医疗（药物效果评估）

**背景**: 新药治疗高血压

**因果发现**:
```
数据：患者 [用药, 血压下降, 年龄, 运动习惯, 饮食]

发现因果图：
      年龄 → 用药 ← 病情严重程度
        ↓              ↓
      血压下降 ← 运动习惯 ← 健康意识

发现"病情严重程度"是混杂因素！
```

**因果推断**:
```
问题: "药物的真实效果是多少？"

混杂因素控制后：
P(血压下降 | do(用药)) - P(血压下降 | do(不用药)) = 15%

结论：药物确实有效，效果15%
```

### 案例2: 电商（营销ROI评估）

**背景**: 评估抖音广告投放效果

**因果发现**:
```
发现因果图：
      用户画像 → 广告投放 ← 营销预算
          ↓            ↓
        购买行为 ← 产品需求

发现"产品需求"是共同原因！
（需求高的人更容易看到广告，也更容易购买）
```

**因果推断**:
```
 naive分析：投广告的用户转化率 5%，没投的 1%
 → 看起来ROI = (5%-1%) / 成本 = 很高

 因果分析（控制产品需求后）：
 P(购买 | do(投广告)) - P(购买 | do(不投)) = 0.3%
 → 真实ROI很低！
 
 结论：大部分转化是"本来就会买"的人
```

### 案例3: 产品（功能影响评估）

**背景**: 新功能"智能推荐"对留存的影响

**因果发现**:
```
发现：使用推荐功能 → 留存率提高
但：活跃用户 → 更可能发现推荐功能
```

**因果推断（A/B测试）**:
```
实验组：强制显示推荐功能
对照组：不显示推荐功能

结果：
P(留存 | do(显示推荐)) = 45%
P(留存 | do(不显示)) = 42%
真实效果 = 3%

结论：功能有效，但不如表面看起来那么有效
```

## 六、总结

### 一句话理解

| 概念 | 一句话 |
|------|--------|
| **因果发现** | 从数据中"破案"，找出真正的因果关系 |
| **因果推断** | 基于因果关系"预测"，如果我做了什么会怎样 |

### 核心价值

```
传统ML：                因果分析：
"是什么"                "为什么" + "如果怎样"
   ↓                       ↓
知道相关性               知道因果性
   ↓                       ↓
预测不准（有混杂因素）   预测干预效果（控制混杂因素）
```

### 什么时候需要因果分析？

| 场景 | 是否需要因果分析 |
|------|----------------|
| 只是预测（明天销量） |  传统ML足够 |
| 要干预（要不要降价） |  需要因果推断 |
| 找原因（什么导致流失） |  需要因果发现 |
| 评估效果（新政策有用吗） |  需要因果推断 |

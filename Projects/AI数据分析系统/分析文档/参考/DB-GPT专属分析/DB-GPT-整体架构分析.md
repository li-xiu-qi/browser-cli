**版本**: DB-GPT v0.7.4

## 一、项目概述

### 1.1 一句话定义

> **DB-GPT 是一个开源的 AI 原生数据应用开发框架，基于 AWEL（Agentic Workflow Expression Language）和 Multi-Agent 架构，旨在通过大模型技术简化数据与业务的交互。**

### 1.2 项目定位

- **Data 3.0 时代的基础设施**: 基于模型和数据库，让企业和开发者用更少的代码构建定制化应用
- **数据智能中枢**: 连接多种数据源（数据库、Excel、数据仓库），提供自然语言交互能力
- **Text2SQL 专业户**: 在 Spider 数据集上达到 82.5% 的准确率
- **企业级 AI 底座**: 支持私有化部署，确保数据隐私安全

### 4.2 AWEL (Agentic Workflow Expression Language)

AWEL 是 DB-GPT 的核心创新，一种声明式的工作流编排语言：

```python
# AWEL 示例：简单的数据处理流程
from dbgpt.core.awel import DAG, HttpTrigger, MapOperator

with DAG("data_pipeline") as dag:
    # HTTP 触发器
    trigger = HttpTrigger("/api/process")
    
    # 数据处理算子
    process = MapOperator(process_data)
    
    # 结果输出算子
    output = MapOperator(format_output)
    
    # 编排流程
    trigger >> process >> output
```

**AWEL 特点**：
- **声明式**：用 Python DSL 描述工作流
- **可视化**：支持拖拽式流程编排
- **可复用**：算子可以组合复用
- **可追踪**：完整的执行日志和状态

### 4.3 SMMF (Service-oriented Multi-model Management Framework)

支持数十种大语言模型的统一管理：

| 类型 | 支持模型 |
|------|----------|
| **开源模型** | Llama 3.1/3、Qwen 2.5/3、GLM-4、DeepSeek、Yi、Gemma、Phi-3、Mixtral 等 |
| **API 模型** | OpenAI GPT-4/3.5、Claude、文心一言、通义千问、智谱 GLM、Kimi 等 |
| **本地部署** | vLLM、llama.cpp、MLX、Ollama 等 |

### 5.3 差异化优势

#### DB-GPT 独特优势

1. **AWEL 工作流编排**
   - 声明式 Python DSL，开发者友好
   - 算子可组合、可复用、可测试
   - 可视化编辑与代码编辑双模式

2. **Text2SQL 专业化**
   - 独立的 DB-GPT-Hub 微调项目
   - Spider 数据集 82.5% 准确率
   - 支持 LoRA/QLoRA/P-tuning 多种微调方式

3. **SMMF 多模型管理**
   - 统一接口支持 20+ 模型
   - 本地模型 + API 模型统一管理
   - 模型切换零成本

4. **GBI (生成式商业智能)**
   - 数据对话 + 报表生成
   - 数据分析洞察
   - 可视化图表自动推荐

5. **数据应用开发框架**
   - 不只是工具，更是开发框架
   - dbgpts 应用市场
   - 可构建自定义数据应用

#### RAGFlow 独特优势

1. **DeepDoc 文档解析**
   - MinerU/Docling/OCR 多策略
   - 复杂文档结构识别
   - 表格、图表提取

2. **GraphRAG 知识图谱**
   - 实体关系自动抽取
   - 结构化知识检索
   - RAPTOR 层次化摘要

3. **多租户隔离**
   - 行级数据隔离
   - ES 索引隔离
   - RBAC 权限控制

4. **Canvas DSL**
   - 工作流定义与状态一体
   - 断点恢复
   - 历史版本管理

### 5.4 适用场景建议

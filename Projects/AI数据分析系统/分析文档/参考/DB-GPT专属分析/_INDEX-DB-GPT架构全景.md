##  核心洞察

### 1. 一句话定义

> **DB-GPT = 面向数据的 AI 应用开发框架（Data 3.0 基础设施）**

与 RAGFlow 的核心差异：
- **RAGFlow** = 深度文档理解 + RAG 知识库平台
- **DB-GPT** = Text2SQL + 多模型管理 + Agent 框架

### 2. 架构设计亮点

```
┌─────────────────────────────────────────────────────────────────┐
│                    DB-GPT 核心设计                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. AWEL（Agentic Workflow Expression Language）                 │
│     • 声明式工作流编排语言                                        │
│     • 类似 Kubernetes YAML，开发者友好                           │
│     • 支持 DAG 工作流和对话流                                     │
│                                                                 │
│  2. SMMF（Service-oriented Multi-Model Management Framework）    │
│     • 支持 30+ 种大模型统一接入                                   │
│     • ProxyLLMClient 统一抽象层                                   │
│     • 支持本地模型 + 云模型混合部署                                │
│                                                                 │
│  3. Text2SQL（Text-to-SQL）                                      │
│     • Spider 数据集 82.5% 准确率                                  │
│     • 支持 10+ 种数据库方言                                       │
│     • 内置 SQL 生成 + 验证 + 执行                                 │
│                                                                 │
│  4. Role-Based Agent 框架                                        │
│     • 三层记忆系统（感官/短期/长期）                              │
│     • 计划驱动的多 Agent 协作                                     │
│     • AutoPlanChatManager 动态调度                                │
│                                                                 │
│  5. GBI（Generative Business Intelligence）                      │
│     • 数据对话 + 报表生成 + 可视化                                │
│     • 支持多种图表类型（AntV系列）                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 技术选型对比（vs RAGFlow）

| 维度 | DB-GPT | RAGFlow |
|------|--------|---------|
| **后端框架** | FastAPI（异步现代） | Flask（同步传统） |
| **前端框架** | Next.js（SSR） | React + Vite（SPA） |
| **项目架构** | Monorepo（7子包） | 模块化单体 |
| **向量数据库** | 6+ 种（Milvus/Chroma/ES等） | ES/Infinity |
| **文档解析** | 基础支持 | DeepDoc深度解析 |
| **Text2SQL** | ⭐⭐⭐ 专业级 | ⭐ 基础支持 |
| **Agent框架** | ⭐⭐⭐ 完善多Agent | ⭐⭐ Canvas工作流 |
| **GraphRAG** | ⭐⭐⭐ 完整实现 | ⭐⭐ 基础支持 |

## 🆚 DB-GPT vs RAGFlow 选型建议

| 场景 | 推荐选择 | 理由 |
|------|---------|------|
| **深度文档理解** | RAGFlow | DeepDoc 解析更强，支持复杂版面 |
| **Text2SQL/数据分析** | DB-GPT | 82.5%准确率，支持10+数据库 |
| **多模型管理** | DB-GPT | 30+模型统一接入，SMMF框架 |
| **知识图谱** | DB-GPT | GraphRAG完整实现，Text2GQL |
| **快速RAG应用** | RAGFlow | 部署简单，开箱即用 |
| **复杂Agent工作流** | DB-GPT | AWEL更灵活，多Agent协作完善 |
| **可视化流程编排** | RAGFlow | Canvas低代码更直观 |

# DB-GPT / RAGFlow 历史会话回顾功能分析

> 详细分析两者如何支持历史会话回顾，包括实现机制、功能对比和上下文管理。

---

## 一、核心结论

### 两者都支持历史会话回顾 

| 功能 | DB-GPT | RAGFlow |
|------|--------|---------|
| **历史会话列表** |  支持 |  支持 |
| **会话详情查看** |  支持 |  支持 |
| **继续历史对话** |  支持 |  支持 |
| **多轮上下文** |  支持 |  支持 |
| **引用溯源** | ⚠️ 基础 |  强（检索来源） |

### 核心差异

```
DB-GPT:  完整的对话历史管理（Conversation + Message）
RAGFlow: 对话历史 + 引用溯源（展示检索来源）
```

### 2.1 数据模型对比

#### DB-GPT 数据表

```sql
-- 对话表
CREATE TABLE conversation (
    id VARCHAR(255) PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    name VARCHAR(255),           -- 对话名称（可编辑）
    model_name VARCHAR(100),     -- 使用的模型
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 消息表
CREATE TABLE chat_message (
    id VARCHAR(255) PRIMARY KEY,
    conv_id VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL,   -- 'user' | 'assistant' | 'system'
    content TEXT,                -- 消息内容
    created_at TIMESTAMP,
    FOREIGN KEY (conv_id) REFERENCES conversation(id)
);
```

#### RAGFlow 数据表

```sql
-- 对话表
CREATE TABLE dialog (
    id VARCHAR(255) PRIMARY KEY,
    tenant_id VARCHAR(255),
    name VARCHAR(255),
    message_count INT,           -- 消息数量统计
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 消息表
CREATE TABLE message (
    id VARCHAR(255) PRIMARY KEY,
    dialog_id VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL,
    content TEXT,
    reference TEXT,              -- 引用来源（JSON格式）
    created_at TIMESTAMP,
    FOREIGN KEY (dialog_id) REFERENCES dialog(id)
);
```

### 2.2 关键差异

| 字段 | DB-GPT | RAGFlow | 说明 |
|------|--------|---------|------|
| `reference` |  无 |  有 | RAGFlow 存储检索来源 |
| `message_count` |  无 |  有 | RAGFlow 统计消息数 |
| `model_name` |  有 |  无 | DB-GPT 记录使用模型 |

## 三、历史会话功能流程

![功能流程](assets/conversation-history-flow.svg)

### 3.1 场景1: 查看历史会话列表

```
用户点击"历史会话"
    ↓
GET /api/v1/conversations
    ↓
查询数据库（按时间倒序）
    ↓
返回会话列表
    ↓
侧边栏展示
```

**代码示例（DB-GPT）**：
```python
@app.get("/api/v1/conversations")
async def list_conversations(user_id: str):
    """获取用户的历史会话列表"""
    conversations = await ConversationService.get_user_conversations(
        user_id=user_id,
        order_by="updated_at DESC",
        limit=50
    )
    return {
        "data": [
            {
                "id": conv.id,
                "name": conv.name or f"对话 {conv.created_at}",
                "last_message": conv.last_message,
                "updated_at": conv.updated_at
            }
            for conv in conversations
        ]
    }
```

### 3.2 场景2: 查看会话详情

```
用户点击某个历史会话
    ↓
GET /api/v1/conversations/{id}/messages
    ↓
查询该会话的所有消息
    ↓
按时间顺序返回
    ↓
渲染对话界面
```

### 3.3 场景3: 继续历史对话

```
用户在历史会话中发送新消息
    ↓
加载上下文（最近 N 条历史消息）
    ↓
构建 Prompt（system + history + new_message）
    ↓
发送给模型（携带上下文）
    ↓
保存新消息到数据库
```

**关键：上下文加载策略**

```python
async def load_context(conv_id: str, max_history: int = 10):
    """加载历史上下文"""
    # 1. 获取最近 N 条消息
    messages = await ChatMessage.get_by_conv_id(
        conv_id=conv_id,
        order_by="created_at DESC",
        limit=max_history
    )
    
    # 2. 按时间正序排列
    messages.reverse()
    
    # 3. 构建上下文
    context = []
    for msg in messages:
        context.append({
            "role": msg.role,
            "content": msg.content
        })
    
    return context
```

## 四、上下文窗口管理

![上下文管理](assets/context-window-management.svg)

### 4.1 问题：模型有上下文长度限制

| 模型 | 上下文长度 |
|------|-----------|
| GPT-3.5 | 4K tokens |
| GPT-4 | 8K / 32K tokens |
| Qwen | 8K / 32K tokens |
| Llama2 | 4K tokens |

**问题**：历史消息太多，超出模型上下文限制怎么办？

### 4.2 解决方案

#### 策略1: 直接截断（最简单）

```python
# 只保留最近 N 条消息
recent_messages = all_messages[-10:]  # 保留最近10条
```

**优点**：简单快速  
**缺点**：丢失早期上下文

#### 策略2: 滑动窗口（DB-GPT/RAGFlow 常用）

```python
def build_context_with_window(
    system_prompt: str,
    history: List[Message],
    new_message: str,
    max_tokens: int = 4000
):
    """滑动窗口策略"""
    context = [{"role": "system", "content": system_prompt}]
    
    # 从后往前添加，直到达到限制
    total_tokens = count_tokens(system_prompt)
    
    for msg in reversed(history):
        msg_tokens = count_tokens(msg.content)
        if total_tokens + msg_tokens > max_tokens:
            break
        context.insert(1, {"role": msg.role, "content": msg.content})
        total_tokens += msg_tokens
    
    # 添加新消息
    context.append({"role": "user", "content": new_message})
    
    return context
```

**优点**：
- 保留系统提示
- 保留最近对话
- 实现简单

#### 策略3: 摘要 + 最近对话（高级）

```python
async def build_context_with_summary(
    system_prompt: str,
    history: List[Message],
    new_message: str,
    max_tokens: int = 4000
):
    """摘要策略"""
    context = [{"role": "system", "content": system_prompt}]
    
    # 如果历史太长，先摘要
    if len(history) > 20:
        # 早期对话生成摘要
        early_messages = history[:-10]
        summary = await generate_summary(early_messages)
        context.append({
            "role": "system",
            "content": f"之前对话摘要：{summary}"
        })
        
        # 保留最近完整对话
        recent_messages = history[-10:]
        context.extend(recent_messages)
    else:
        context.extend(history)
    
    context.append({"role": "user", "content": new_message})
    return context
```

**优点**：保留关键信息，支持超长对话  
**缺点**：需要额外计算摘要

### 4.3 DB-GPT 实现

```python
# DB-GPT 配置
CONVERSATION_MAX_MESSAGES = 20  # 默认保留最近20条
CONVERSATION_TOKEN_LIMIT = 4000  # token 限制

def build_conversation_context(
    conversation_id: str,
    new_message: str,
    app_code: Optional[str] = None
):
    """DB-GPT 构建对话上下文"""
    # 1. 获取系统提示
    system_prompt = get_system_prompt(app_code)
    
    # 2. 获取历史消息（滑动窗口）
    history = ChatMessage.get_recent_messages(
        conv_id=conversation_id,
        limit=CONVERSATION_MAX_MESSAGES
    )
    
    # 3. 构建上下文
    context = [{"role": "system", "content": system_prompt}]
    context.extend([
        {"role": msg.role, "content": msg.content}
        for msg in history
    ])
    context.append({"role": "user", "content": new_message})
    
    # 4. 检查 token 数，必要时截断
    if count_tokens(context) > CONVERSATION_TOKEN_LIMIT:
        context = truncate_context(context, CONVERSATION_TOKEN_LIMIT)
    
    return context
```

# 自动归档旧对话
async def archive_old_conversations(user_id: str, days: int = 30):
    """归档30天前的对话"""
    old_conversations = await Conversation.find({
        "user_id": user_id,
        "updated_at": {"$lt": datetime.now() - timedelta(days=days)},
        "is_archived": False
    })
    
    for conv in old_conversations:
        conv.is_archived = True
        await conv.save()

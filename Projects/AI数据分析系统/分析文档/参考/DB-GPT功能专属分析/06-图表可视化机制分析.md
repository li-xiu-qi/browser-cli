# DB-GPT 图表可视化机制分析

> 分析 DB-GPT 的图表可视化是如何实现的：后端 AI 决策图表类型，前端负责渲染

---

## 一、核心机制概述

DB-GPT 的图表可视化采用 **"后端 AI 决策 + 前端渲染"** 的协作模式：

```
┌─────────────────────────────────────────────────────────────────┐
│                      图表生成流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户提问 ──► 后端 AI 分析 ──► 决策图表类型 ──► 执行查询       │
│       │                            │                │          │
│       │                            │                ▼          │
│       │                            │         获取数据          │
│       │                            │                │          │
│       │                            ▼                ▼          │
│       │                    组装 ChartData 对象                  │
│       │                    {                                    │
│       │                      chart_type: "BarChart",            │
│       │                      chart_name: "区域销售额",          │
│       │                      values: [...],                     │
│       │                      ...                                │
│       │                    }                                    │
│       │                            │                            │
│       ▼                            ▼                            │
│   看到可视化图表 ◄─── 前端根据 chart_type 渲染                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、后端：AI 智能决策

### 2.1 决策流程

后端 LLM 根据以下因素决策图表类型：

| 因素 | 分析内容 | 示例 |
|------|----------|------|
| **用户意图** | 用户想做什么分析 | "对比"、"趋势"、"占比" |
| **数据特征** | 查询结果的字段类型 | 时间字段、分类字段、数值字段 |
| **数据量** | 返回的记录数 | 10条 vs 1000条 |
| **场景模板** | 预设的图表推荐规则 | Dashboard 场景的模板配置 |

### 2.2 Prompt 中的图表选择规则

```python
# chat_db/auto_execute/prompt.py（节选）
"""
请根据以下规则选择合适的展示方式:

1. 如果是趋势分析（有时间维度，如按月、按季度），使用'LineChart'
   示例："展示过去一年的销售趋势"

2. 如果是分类对比（有类别维度，需要比较大小），使用'BarChart'
   示例："对比各区域的销售额"

3. 如果是占比分析（需要看部分与整体的关系），使用'PieChart'
   示例："各品类销售额占比"

4. 如果是明细数据（字段较多，需要查看具体数值），使用'Table'
   示例："列出所有订单详情"

请在响应中指定 display_type 字段。
"""
```

### 2.3 代码实现

```python
# chat_dashboard/chat.py
class ChatDashboard(BaseChat):
    def do_action(self, prompt_response):
        chart_datas: List[ChartData] = []
        for chart_item in prompt_response:
            # 执行 SQL 获取数据
            field_names, values = dashboard_data_loader.get_chart_values_by_conn(
                self.database, chart_item.sql
            )
            
            # 组装图表数据（包含 AI 决策的图表类型）
            chart_datas.append(
                ChartData(
                    chart_uid=str(uuid.uuid1()),
                    chart_name=chart_item.title,
                    chart_type=chart_item.showcase,  # <-- AI 决策的图表类型
                    chart_desc=chart_item.thoughts,
                    chart_sql=chart_item.sql,
                    column_name=field_names,
                    values=values,
                )
            )
        return ReportData(charts=chart_datas)
```

### 2.4 支持的可视化类型

| 图表类型 | 决策条件 | 适用场景 |
|----------|----------|----------|
| **BarChart** | 分类维度 + 数值指标 + 对比意图 | 各区域销售额对比、各品类销量排名 |
| **LineChart** | 时间维度 + 连续数据 + 趋势意图 | 月度销售趋势、用户增长曲线 |
| **PieChart** | 单维度 + 占比关系 | 市场份额占比、费用构成 |
| **Table** | 多字段明细 + 精确数值 | 订单详情、客户列表 |
| **IndicatorValue** | 单一关键指标 | 总销售额、用户总数 |
| **ScatterChart** | 两个数值维度 + 分布关系 | 价格与销量关系 |

## 三、前端：负责渲染

### 3.1 数据结构定义

```typescript
// types/chat.ts
export type ChartData = {
  chart_uid: string;      // 图表唯一标识
  chart_name: string;     // 图表标题
  chart_type: string;     // 图表类型（后端 AI 指定）
  chart_desc: string;     // 图表描述
  chart_sql: string;      // 生成图表的 SQL
  column_name: string[];  // 列名
  values: ChartValue[];   // 数据值
};

type ChartValue = {
  name: string;
  type: string;
  value: number;
};
```

### 3.2 渲染逻辑

```typescript
// components/chart/index.tsx
function Chart({ chartsData }: Props) {
  return (
    <div className='flex flex-col gap-3'>
      {chartsData.map(chart => {
        // 根据后端指定的 chart_type 选择对应组件
        if (chart.chart_type === 'LineChart') {
          return <LineChart key={chart.chart_uid} chart={chart} />;
        } else if (chart.chart_type === 'BarChart') {
          return <BarChart key={chart.chart_uid} chart={chart} />;
        } else if (chart.chart_type === 'PieChart') {
          return <PieChart key={chart.chart_uid} chart={chart} />;
        } else if (chart.chart_type === 'Table') {
          return <TableChart key={chart.chart_uid} chart={chart} />;
        } else if (chart.chart_type === 'IndicatorValue') {
          return <IndicatorCard key={chart.chart_uid} chart={chart} />;
        }
      })}
    </div>
  );
}
```

### 3.3 图表组件实现

以柱状图为例：

```typescript
// components/chart/bar-chart.tsx
import ReactECharts from 'echarts-for-react';

function BarChart({ chart }: { chart: ChartData }) {
  const option = {
    title: { text: chart.chart_name },
    tooltip: {},
    xAxis: {
      type: 'category',
      data: chart.values.map(v => v.name),
    },
    yAxis: { type: 'value' },
    series: [{
      type: 'bar',
      data: chart.values.map(v => v.value),
    }],
  };
  
  return <ReactECharts option={option} />;
}
```

## 四、完整示例

### 场景：用户查询各区域销售额

**步骤 1：用户提问**
```
用户："查询各区域销售额排名"
```

**步骤 2：后端 AI 分析**
```
AI 分析：
- 意图：分类对比（排名）
- 维度：区域（分类）
- 指标：销售额（数值）
- 决策：适合柱状图

生成响应：
{
  "sql": "SELECT region, SUM(amount) as sales FROM orders GROUP BY region ORDER BY sales DESC",
  "display_type": "BarChart",
  "thoughts": "使用柱状图对比各区域销售额"
}
```

**步骤 3：执行查询**
```sql
执行 SQL → 返回数据：
[
  {"region": "华东", "sales": 5000000},
  {"region": "华南", "sales": 3800000},
  {"region": "华北", "sales": 2900000}
]
```

**步骤 4：组装 ChartData**
```json
{
  "chart_uid": "chart-001",
  "chart_name": "各区域销售额排名",
  "chart_type": "BarChart",
  "chart_desc": "使用柱状图对比各区域销售额",
  "chart_sql": "SELECT region...",
  "column_name": ["region", "sales"],
  "values": [
    {"name": "华东", "type": "bar", "value": 5000000},
    {"name": "华南", "type": "bar", "value": 3800000},
    {"name": "华北", "type": "bar", "value": 2900000}
  ]
}
```

**步骤 5：前端渲染**
```
前端接收 chart_type="BarChart" → 渲染 ECharts 柱状图

用户看到：
```

## 七、局限与注意事项

### 7.1 AI 决策可能不准确

```
问题：AI 可能选择不合适的图表类型

示例：
用户："看一下销售数据"
AI 可能返回表格（过于保守）
而用户实际想要柱状图对比

解决：
- 在 Prompt 中明确意图
- 后续版本支持用户手动切换图表
```

### 7.2 大数据量渲染性能

```
问题：返回 10000 条数据的折线图会卡

解决：
- 后端做数据聚合（按周/月汇总）
- 超出阈值自动切换为表格
- 前端虚拟滚动优化
```

### 7.3 复杂图表支持有限

```
当前支持：基础图表（柱状、折线、饼图、表格）
暂不支持：
- 桑基图
- 热力图
- 地理地图
- 自定义可视化

扩展方式：
在 chart/index.tsx 中新增组件映射
```

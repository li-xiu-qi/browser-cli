### 1.2 AWEL 架构解析

```
┌─────────────────────────────────────────────────────────────────┐
│                     AWEL 三层架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌───────────────┐                                            │
│   │   DSL 层       │  ← 声明式工作流定义 (SQL-like)              │
│   │   业务人员     │                                            │
│   └───────┬───────┘                                            │
│           │ 解析                                                │
│           ▼                                                     │
│   ┌───────────────┐                                            │
│   │ AgentFream 层 │  ← 链式算子组合 (Pandas/Spark 风格)         │
│   │   Python 开发者│                                           │
│   └───────┬───────┘                                            │
│           │ 编译                                                │
│           ▼                                                     │
│   ┌───────────────┐                                            │
│   │  Operator 层  │  ← 基础计算原子 (Embedding/LLM/Retrieval)   │
│   │   框架开发者   │                                            │
│   └───────┬───────┘                                            │
│           │ 执行                                                │
│           ▼                                                     │
│   ┌───────────────┐                                            │
│   │   DAG 引擎    │  ← 拓扑排序 + 调度 + 执行                   │
│   └───────────────┘                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 核心优势

| 优势 | 说明 | 学习价值 |
|------|------|----------|
| **三层抽象** | DSL → AgentFream → Operator | 渐进式复杂度管理 |
| **DAG 原生** | 有向无环图执行模型 | 复杂流程可视化 |
| **声明式 DSL** | SQL-like 语法 | 降低非程序员门槛 |
| **分布式支持** | Ray 集成 | 企业级扩展能力 |

### 1.4 代码示例对比

**AIASys (命令式)**:
```python
team = DataAnalysisTeam(session_id="123")
result = team.run("分析销售数据")
```

**DB-GPT AWEL (声明式)**:
```sql
CREATE WORKFLOW analysis AS
BEGIN
    DATA raw = LOAD FROM db_source('sales_db');
    DATA clean = TRANSFORM raw USING clean_data();
    DATA result = APPLY LLM 'gpt-4' WITH DATA clean;
    RESPOND TO user WITH result;
END;
```

### 1.5 对 AIASys 的启示

1. **DSL 层**: 为高级用户提供快捷配置方式
2. **DAG 可视化**: 复杂分析流程展示执行图
3. **Operator 复用**: 工具标准化为可复用算子

### 2.4 Workforce 架构

```python
# Workforce 使用示例
from camel.workforce import Workforce
from camel.societies import RolePlaying

# 创建 Workforce
workforce = Workforce(
    description="数据分析团队",
    new_agent_kwargs={
        "model": ModelFactory.create("gpt-4o"),
    }
)

# 添加 Worker
workforce.add_single_agent_worker(
    "数据分析师",
    agent=data_analyst_agent,
    tools=[python_tool, viz_tool]
)

workforce.add_single_agent_worker(
    "可视化专家", 
    agent=viz_expert_agent,
    tools=[plot_tool]
)

# 执行任务
task = Task(content="分析销售数据并生成图表")
result = workforce.process_task(task)
```


### 3.3 代码示例

```python
from smolagents import CodeAgent, HfApiModel, tool

# 定义工具
@tool
def get_weather(location: str) -> str:
    """获取指定位置的天气"""
    return f"{location} 的天气是晴朗的"

# 创建 Agent
agent = CodeAgent(
    tools=[get_weather],
    model=HfApiModel(),
    additional_authorized_imports=["pandas", "numpy"]
)

# 运行
agent.run("分析天气数据并生成报告")
```

### 3.4 对 AIASys 的价值

AIASys 基于 smolagents 构建，继承其优势：
-  简洁 API
-  工具装饰器
-  多种 LLM 支持
-  代码执行能力

| 属性 | 详情 |
|------|------|
| **来源** | Anthropic (官方) |
| **安装** | `pip install claude-agent-sdk` |
| **定位** | 程序化 Claude Code 交互 |

### 4.2 双模式架构

```
┌─────────────────────────────────────────────────────────────────┐
│               Claude Agent SDK 双模式                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   模式 1: query() - 简单查询                                     │
│   ┌─────────────────────────────────────────────────────┐      │
│   │  async for message in query(prompt="..."):          │      │
│   │      print(message)  # 流式输出                      │      │
│   └─────────────────────────────────────────────────────┘      │
│                                                                 │
│   模式 2: ClaudeSDKClient - 双向会话                            │
│   ┌─────────────────────────────────────────────────────┐      │
│   │  async with ClaudeSDKClient(options) as client:     │      │
│   │      await client.query("...")                      │      │
│   │      async for msg in client.receive_response():    │      │
│   │          process(msg)                               │      │
│   └─────────────────────────────────────────────────────┘      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 核心特性

| 特性 | 说明 | 学习价值 |
|------|------|----------|
| **内置 MCP** | 进程内 MCP Server | 避免外部进程管理 |
| **自定义工具** | @tool 装饰器 | 类型安全 |
| **Hooks** | Agent 循环钩子 | 执行拦截与反馈 |
| **工具白名单** | allowed_tools | 权限控制 |

### 4.4 MCP 集成示例

```python
from claude_agent_sdk import tool, create_sdk_mcp_server

# 1. 定义工具
@tool("analyze_data", "分析数据", {"data": str})
async def analyze_data(args):
    result = process_data(args['data'])
    return {"content": [{"type": "text", "text": result}]}

# 2. 创建 MCP Server
server = create_sdk_mcp_server(
    name="analysis-tools",
    version="1.0.0",
    tools=[analyze_data]
)

# 3. 使用
options = ClaudeAgentOptions(
    mcp_servers={"analysis": server},
    allowed_tools=["mcp__analysis__analyze_data"]
)
```

### 4.5 对 AIASys 的启示

1. **MCP 生态**: 接入标准化工具协议
2. **Hooks 机制**: 执行前后拦截
3. **进程内 MCP**: 避免 IPC 开销

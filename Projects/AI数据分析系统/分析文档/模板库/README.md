# 前端功能分析模板库

本目录包含一套标准化的前端功能分析模板，用于系统化分析 AI 数据分析系统相关的开源项目。

##  模板清单

| 模板 | 文件名 | 使用场景 | 预计耗时 |
|------|--------|---------|---------|
| **模板1** | `模板1-单个项目前端深度分析.md` | 深度分析单个项目的前端架构、交互设计、技术实现 | 1-2小时 |
| **模板2** | `模板2-项目间对比评估.md` | 横向对比多个同类项目，辅助技术选型 | 2-3小时 |
| **模板3** | `模板3-前端技术选型决策.md` | 基于参考项目分析，制定技术选型方案 | 2-3小时 |
| **模板4** | `模板4-交互设计模式提取.md` | 从项目中提取可复用的交互设计模式 | 1-2小时 |
| **模板5** | `模板5-可视化方案分析.md` | 分析可视化方案，辅助技术选型 | 1-2小时 |

##  使用指南

### 场景1：分析一个新项目

**推荐模板**: 模板1 - 单个项目前端深度分析

**使用流程**:
1. 复制 `模板1-单个项目前端深度分析.md` 到新文件
2. 按照模板提示填写项目信息
3. 分析项目的前端架构、交互设计、技术实现
4. 输出价值评估和借鉴建议

**示例输出位置**:
```
Projects/AI数据分析系统/分析文档/报告/前端功能分析/
├── AI-Data-Science-Team-前端功能分析.md
├── DB-GPT-前端功能分析.md
└── Data-Formulator-前端功能分析.md
```

---

### 场景2：对比多个项目做选型

**推荐模板**: 模板2 - 项目间对比评估

**使用流程**:
1. 选择2-5个同类项目（如前端框架、可视化方案等）
2. 使用模板1先完成各项目的深度分析
3. 复制 `模板2-项目间对比评估.md`
4. 填写多维度对比矩阵
5. 给出明确的推荐结论

**示例输出位置**:
```
Projects/AI数据分析系统/分析文档/报告/前端功能分析/
└── 03-交互设计对比评估报告.md
```

---

### 场景3：制定技术选型决策

**推荐模板**: 模板3 - 前端技术选型决策

**使用流程**:
1. 明确业务需求和技术约束
2. 基于模板2的对比结果
3. 复制 `模板3-前端技术选型决策.md`
4. 评估候选方案，计算加权得分
5. 制定实施路线图

**示例输出位置**:
```
Projects/AI数据分析系统/分析文档/
└── 前端技术选型决策.md
```

---

### 场景4：提取交互设计模式

**推荐模板**: 模板4 - 交互设计模式提取

**使用流程**:
1. 分析多个项目的交互设计
2. 识别重复的交互模式
3. 复制 `模板4-交互设计模式提取.md`
4. 抽象出通用的设计模式
5. 形成可复用的设计模式库

---

### 场景5：分析可视化方案

**推荐模板**: 模板5 - 可视化方案分析

**使用流程**:
1. 确定分析的可视化类型（图表/仪表盘/探索式）
2. 复制 `模板5-可视化方案分析.md`
3. 分析图表类型支持、性能、交互能力
4. 给出选型建议

---

##  分析工作流建议

### 完整分析流程

```
Step 1: 快速扫描（使用快速扫描模板）
    ↓ 筛选出重点项目
Step 2: 深度分析（使用模板1）
    ↓ 对每个重点项目分析
Step 3: 对比评估（使用模板2）
    ↓ 横向对比同类项目
Step 4: 模式提取（使用模板4）
    ↓ 提取可复用设计模式
Step 5: 技术选型（使用模板3）
    ↓ 制定技术方案
Step 6: 可视化分析（使用模板5）
    ↓ 可视化方案选型
```

##  填写规范

### 必填项
- 元信息（项目名称、日期、分析者）
- 核心架构图
- 技术栈清单
- 价值评估（评分）
- 结论与建议

### 可选项
- 详细的代码示例
- 性能测试数据
- 截图/示意图
- 附录（参考资源）

### 评分标准

| 评分 | 含义 |
|:----:|------|
| ⭐⭐⭐⭐⭐ (5分) | 优秀，强烈推荐 |
| ⭐⭐⭐⭐ (4分) | 良好，值得借鉴 |
| ⭐⭐⭐ (3分) | 一般，参考即可 |
| ⭐⭐ (2分) | 较差，谨慎参考 |
| ⭐ (1分) | 差，不建议参考 |

##  最佳实践

1. **从模板1开始**：新项目分析先使用模板1建立基础认知
2. **保持一致性**：同类型分析使用相同模板，便于对比
3. **及时更新**：发现新信息后及时更新分析文档
4. **记录决策**：技术选型等重要决策记录在文档中
5. **分享知识**：有价值的发现分享给团队成员

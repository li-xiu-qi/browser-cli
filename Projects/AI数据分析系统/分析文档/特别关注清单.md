# 参考项目分析 - 特别关注清单

本清单列出 AI 数据分析系统项目中特别值得关注的借鉴点。

---

## 一、架构设计层面

### 1.1 LLM 抽象与多厂商支持

**值得借鉴的问题**:
- 如何设计 LLM 接口抽象层，支持 OpenAI、Azure、本地模型等多厂商？
- 如何实现模型 fallback 机制？
- 如何管理不同模型的 Prompt 模板差异？

**重点关注项目**: DB-GPT、Vanna、TaskWeaver

**使用模板**: 模板2（深度架构分析）+ 模板8（技术栈分析）

### 1.2 数据源抽象

**值得借鉴的问题**:
- 如何统一封装 MySQL、PostgreSQL、SQLite、DuckDB 等不同数据源？
- 如何自动获取表结构（Schema）信息？
- 如何处理大数据量下的性能问题？

**重点关注项目**: WrenAI、Chat2DB、SuperSonic

### 1.3 插件/扩展机制

**值得借鉴的问题**:
- 如何设计插件系统，支持自定义数据源？
- 如何支持自定义可视化类型？
- 如何支持自定义分析 Agent？

**重点关注项目**: DB-GPT、TaskWeaver

---

## 二、Text-to-SQL 实现

### 2.1 Prompt 工程

**值得借鉴的具体实现**:
- **Schema 描述格式**: 如何将表结构信息整理进 Prompt？
- **Few-shot 样例选择**: 如何根据问题动态选择样例 SQL？
- **领域知识注入**: 如何将业务术语映射到数据库字段？

**重点关注项目**: 
- Vanna（RAG + 向量检索样例）
- WrenAI（语义建模层）
- SQLCoder（专用模型微调）

### 2.2 SQL 正确性保障

**值得借鉴的机制**:
- SQL 执行前的合法性检查
- 执行错误后的自动修正流程
- 结果与预期不符时的反馈处理

**重点关注项目**: Vanna、DB-GPT

### 2.3 复杂查询支持

**值得借鉴的方案**:
- 多表 JOIN 的自动处理
- 子查询的生成策略
- 聚合函数与分组的使用

**重点关注项目**: WrenAI、SuperSonic

---

## 三、可视化与交互

### 3.1 图表智能推荐

**值得借鉴的算法**:
- 如何根据数据特征推荐图表类型？（如：时间序列→折线图，分类对比→柱状图）
- 如何根据问题语义推荐图表？
- 多维度数据的下钻交互设计

**重点关注项目**: 
- LIDA（Microsoft 的图表推荐引擎）
- Data Formulator（概念驱动的可视化）
- PyGWalker（类 Tableau 的交互）

### 3.2 自然语言到图表配置

**值得借鉴的映射方式**:
- "按月份展示销售额趋势" → 折线图
- "各地区的订单占比" → 饼图
- "对比去年同期的增长" → 双轴图

**重点关注项目**: Data Formulator、PandasAI

### 3.3 结果呈现方式

**值得借鉴的呈现策略**:
- SQL + 表格 + 图表的组合展示
- 关键指标的自动高亮
- 异常数据的自动标注

**重点关注项目**: Evidence、RATH

---

## 四、Agent 与自动化

### 4.1 多步分析流程

**值得借鉴的场景**:
- 用户问"为什么上月销售额下降"时的自动分析流程
- 数据清洗 → 探索分析 → 可视化 → 洞察总结的自动化

**重点关注项目**: 
- TaskWeaver（Microsoft 的数据分析 Agent 框架）
- DeepAnalyze（自治数据分析 Agent）
- AI Data Science Team（数据科学工作流 Agent）

### 4.2 工具调用设计

**值得借鉴的设计**:
- 工具的定义方式（Schema 描述）
- 工具选择的决策逻辑
- 工具结果的整合方式

**重点关注项目**: TaskWeaver、Open Interpreter

### 4.3 人机协作模式

**值得借鉴的交互**:
- 何时需要人工确认？
- 如何展示 Agent 的思考过程？
- 如何支持人工干预和修正？

**重点关注项目**: TaskWeaver、DataMind

## 五、工程实践

### 5.1 错误处理

**值得借鉴的策略**:
- LLM 输出解析失败的容错
- SQL 执行失败的优雅降级
- 超时和重试机制

**重点关注项目**: Vanna、DB-GPT

### 5.2 缓存机制

**值得借鉴的方案**:
- 相似查询的结果缓存
- Schema 信息的缓存策略
- 向量化结果的缓存

**重点关注项目**: Vanna（RAG 缓存）、SuperSonic

### 5.3 安全与权限

**值得借鉴的机制**:
- SQL 注入防护
- 敏感数据脱敏
- 行级权限控制

**重点关注项目**: Chat2DB、WrenAI

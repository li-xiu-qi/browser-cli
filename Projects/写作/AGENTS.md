## Cliper 内容管理规范

### 抓取文章后的分类流程

当用户使用 Cliper 抓取文章后，按以下规则处理：

```
抓取文章 → 评估内容质量 → 评估写作形式 → 决定去向
```

#### 决策树

| 内容质量 | 写作形式 | 目标目录 | 处理方式 |
|----------|----------|----------|----------|
| 高价值 | 值得学习 | `05_参考范文/` | 全文保存 + 深度分析 (`06_范文分析/`) |
| 高价值 | 一般 | `08_素材收藏/` | 摘要记录 + 归类到观点/案例/金句 |
| 低价值 | - | 丢弃 | 不保存 |

#### 快速判断标准

**05_参考范文 标准**（需同时满足）：
- [ ] 结构清晰，逻辑严密
- [ ] 表达有亮点，值得学习
- [ ] 写作技巧可复用

**08_素材收藏 标准**（满足其一）：
- [ ] 观点独特，有启发性
- [ ] 案例典型，可作论据
- [ ] 金句精辟，值得引用

**丢弃标准**（满足其一）：
- [ ] 内容平庸，无新意
- [ ] 纯粹情绪宣泄
- [ ] 过时失效

#### 操作流程

1. **Cliper 默认存放位置**: `.agents/cliper_datas/`
2. **用户分类**：
   - 参考范文 → `05_参考范文/`
   - 素材收藏 → `08_素材收藏/`（按类型放入子目录）
3. **AI 协助**：
   - 范文：使用分析模板进行深度分析
   - 素材：使用摘要模板记录要点

#### 目录结构

```
05_参考范文/
├── 文章标题.md           # 原文全文
└── ...

06_范文分析/
└── [文章标题]/
    └── 分析.md           # 深度分析文档

08_素材收藏/
├── 观点收藏/             # 独特观点、洞察
├── 案例收藏/             # 典型事件、数据
├── 金句收藏/             # 精辟句子、比喻
└── _INDEX/
    └── 素材收藏索引.md    # 索引文档
```

## 目录规范（强制执行）

```
03_工作区/[文章标题]/
├── 捕获.md        # 输入：只管存，不管理
├── 提取.md        # 加工：结构优先
├── 输出.md        # 产出：遵循风格 Skill
├── 参考文献/       # PDF、论文等
└── assets/        # 配图
```

**命名**:
- 工作区目录：文章标题
- 三个核心文件：固定为 `捕获.md` / `提取.md` / `输出.md`

## 写作协作流程

### 1. 创建阶段

用户说"我要写一篇关于 XXX 的文章"时：

1. 确认 `03_工作区/[文章标题]/` 目录存在
2. 创建标准文件结构
3. 加载风格 Skill（如用户指定）

### 2. 写作阶段

按 **捕获 → 提取 → 输出** 流程：

**捕获**（可选协助）：
- 帮用户整理素材
- 分类归纳

**提取**（必须协助）：
- 头脑风暴角度
- 生成并确认大纲
- 将大纲写入 `提取.md`

**输出**（核心）：
- 分段写作
- 每段用户手动修改后再继续
- 严格遵循风格 Skill

### 3. 发布后

用户说"这篇文章已发布"时：

1. 在 `04_Skill素材库/_INDEX.md` 添加：
   ```markdown
   | 文章标题 | 03_工作区/文章/ | 已发布 | ⭐ 正例 | ⏳ 待复盘 |
   ```

2. 提醒用户：发布后 1-7 天内复盘

### 4. 复盘阶段

用户说"复盘这篇文章"时：

1. 读取原文（`03_工作区/[文章]/输出.md`）
2. 按 [[.agents/skills/ai-writing-collaborator/05-项目管理规范/03-复盘流程规范|复盘模板]] 分析
3. 确定分类（正例 ⭐ / 反例 ❌）
4. 更新 `_INDEX.md` 状态
5. 根据复盘结果更新风格 Skill（如需）

## 项目专属规则（覆盖全局）

### 署名
- 所有文章署名：筱可

### 选题标准
- 技术文章：必须有代码示例
- 观点文章：必须有明确立场

### 风格偏好
- 语气：口语化，像朋友聊天
- 开头：场景化，不从抽象概念开始
- 技术内容：术语后紧跟通俗解释

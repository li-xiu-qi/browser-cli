# 写作工作区规范

本目录是内容生产的核心车间。每个文件夹代表一个独立的**写作项目（工作空间）**，实行严格的物理隔离和标准化的文件结构。

##  项目结构标准

一个标准的工作空间（以文章标题命名）包含以下五个核心组件：

```text
[文章标题]/
├── 捕获.md        # [输入] 灵感收集、素材堆积
├── 提取.md        # [加工] 逻辑大纲、观点提炼
├── 输出.md        # [产出] 最终成稿
├── 参考文献/      # [支撑] 深度参考资料 (PDF, 论文, 研报)
└── assets/       # [资源] 文章配图、内嵌附件
```

##  组件详细说明

### 1.  捕获.md (Capture)
- **定位**: 混沌的输入缓冲区。
- **内容**: 
    - 网页剪藏链接（Cliper 数据）。
    - 脑海中闪过的碎片灵感。
    - 复制粘贴的原始段落。
- **原则**: **只管存，不管理**。不要在这里花时间排版，先把东西抓进来再说。

### 2.  提取.md (Distill)
- **定位**: 逻辑加工台与大纲视图。
- **内容**:
    - **思维导图/大纲**: 梳理文章脉络结构。
    - **核心观点**: 从捕获的素材中提炼出的“金句”或论点。
    - **TODO 列表**: 写作过程中需要查证的数据或待补充的环节。
- **原则**: **结构优先**。在这里完成思考，而不是在写正文时边写边想。

### 3. ✍️ 输出.md (Output)
- **定位**: 对外发布的成品。
- **内容**: 经过润色、排版完美、可直接发布的文章。
- **元数据**: 必须包含标准的 Frontmatter（status, date, tags）。
- **原则**: **读者视角**。这里的内容是为了让别人看懂，注重表达的流畅性和排版的美观。

### 4.  参考文献/ (References)
- **定位**: 高价值的原始参考资料库。
- **内容**: 
    - 论文原文 (PDF)。
    - 行业研报。
    - 电子书章节。
    - 深度长文的离线存档。
- **用法**: 在 `提取.md` 或 `输出.md` 中，通过 `[[参考文献/xxx.pdf]]` 的方式引用，作为论据支撑。

### 5. ️ assets/
- **定位**: 视觉资源库。
- **内容**: 文章中插入的图片、图表、图标等。
- **维护**: 通常由 Obsidian 自动管理（粘贴图片时自动存入）。

---

##  典型工作流

1.  **收集**: 在 `捕获.md` 中堆积素材，将下载的 PDF 丢入 `参考文献/`。
2.  **构思**: 在 `提取.md` 中梳理逻辑，引用 `捕获.md` 中的素材和 `参考文献/` 中的证据。
3.  **写作**: 根据 `提取.md` 的大纲，在 `输出.md` 中完成正文撰写。
4.  **归档**: 文章发布后，将 `输出.md` 标记为 `已完成`。

**项目**：将 PNG 格式的 CPU vs GPU 架构对比图转换为 SVG 矢量图  
**时间**：2026-02-06  
**参与人**：筱可（执行）、小可（AI助手）

| T11 | 创建复盘文档 |

### 关键决策点

**决策1：先画框架再填内容**
- 创建了 layout-cpu-section.svg 和 layout-gpu-section.svg 作为框架
- 效果：有助于定位问题，但最终布局还是多次调整

**决策2：组件化设计**
- 在 `<defs>` 中定义角色，用 `<use>` 复用
- 效果：代码量减少30%，小黄人矩阵实现简洁

**决策3：独立文件拆分**
- 将角色拆分为 professor.svg、minion.svg、commander.svg
- 效果：便于单独调试和对比原图

## 三、得失分析

### 做得好的（成功因素）

#### 1. 组件化设计思路 
**发现**：使用 `<defs>` + `<use>` 模式  
**原因**：12个小黄人如果分别绘制，代码会重复且难以维护  
**假设**：任何需要重复元素的 SVG 都应该采用此模式  
**沉淀**：svg-lessons-learned.md 中记录了具体实现方法

#### 2. 分文件调试策略 
**发现**：将角色拆分为独立 SVG 文件  
**原因**：便于单独对比原图、调整细节  
**假设**：复杂 SVG 项目都应该采用"组件+整合"的工作流  
**沉淀**：建立了 components/ 目录结构

#### 3. 云朵状对话框创新 
**发现**：用 path 实现柔和圆角 + 尖角  
**原因**：比矩形+三角形组合更美观  
**方法**：`l-6,10 l-2,-10` 创建小尖角  
**沉淀**：已记录在文档中

### 做得不好的（失败教训）

#### 1. 初期布局规划不足 
**问题**：第一版布局重叠严重，教授/Cache/文字相互覆盖  
**根因分析**：
- 没有用虚线框先占位验证
- y坐标计算未考虑元素完整高度链
- 急于填内容，没先验证框架

**真问题**：缺乏"先框架后内容"的执行纪律

**改进**：
- 下次先用 `stroke-dasharray` 画虚线框占位
- 计算坐标时考虑：y = 上一元素y + 高度 + 间距
- 使用半透明填充预览边界

#### 2. 角色设计反复调整 
**问题**：教授和指挥官形象多次返工  
**根因分析**：
- 第一次没仔细看原图细节（教授是圆头方眼镜）
- 指挥官做成了有五官的，原图是剪影
- 没有先定义"设计规范"

**真问题**：没有建立"参考图-设计规范-实现"的流程

**改进**：
- 创建 design-spec.md 记录颜色、尺寸、风格
- 每个角色先画草图对比原图确认
- 关键细节（如眼镜形状）先确认再实现

#### 3. 中文括号低级错误 
**问题**：`translate（720, 110）` 用了全角括号导致 GPU 区域不显示  
**根因**：注意力在布局上，忽视了语法细节
**改进**：代码审查清单中加入"符号检查"

### 关键认知提炼

#### 认知1：SVG 坐标系需要"从外到内"规划
```
画布 (1422x690)
  └── 卡片 (640x550) @ (50,100)
        └── 教授区域 @ (140,230)
              └── 对话框 @ (-60,-100) 相对坐标
```
**模型**：先绝对定位大容器，再相对定位内部元素

#### 认知2：复刻项目的"三层验证"
1. **框架层**：虚线框占位，验证布局
2. **组件层**：独立文件，验证角色
3. **整合层**：最终组装，验证整体

#### 认知3：对话框设计的"云朵公式"
```svg
M10,0 h90 a8,8 0 0 1 8,8 v18 a8,8 0 0 1 -8,8 h-75 
l-6,10 l-2,-10 h-7 a8,8 0 0 1 -8,-8 v-18 a8,8 0 0 1 8,-8 z
```
**关键参数**：圆角半径 `a8,8`、尖角 `l-6,10 l-2,-10`

## 四、总结提炼

### 认知共识

| 类型 | 内容 |
|------|------|
| **目标标准** | SVG 复刻项目成功 = 视觉还原 + 无重叠 + 代码可维护 |
| **做事标准** | 先框架（虚线框）→ 再组件 → 最后整合 |
| **做对了** | 组件化设计、分文件调试、云朵对话框 |
| **做错了** | 初期布局规划不足、角色设计反复、符号错误 |

### 行动计划

#### 坚持做 
- [ ] 使用 `<defs>` + `<use>` 模式复用元素
- [ ] 拆分独立组件文件便于调试
- [ ] 创建 svg-lessons-learned.md 记录经验
- [ ] 使用 path 实现云朵状对话框

#### 放弃做 
- [ ] 不使用全角括号（`（）`）在 SVG 属性中
- [ ] 不跳过框架验证直接填内容
- [ ] 不在没有对比原图的情况下凭记忆设计角色

#### 新开始 🆕
- [ ] 建立 SVG 项目模板（含框架、组件目录、设计规范）
- [ ] 创建代码审查 Checklist（符号、坐标、颜色）
- [ ] 沉淀"三层验证"工作流到 AGENTS.md

### 过程资产

#### 1. 代码模板
```svg
<svg viewBox="0 0 width height">
  <defs>
    <!-- 定义可复用组件 -->
    <g id="component">
      <!-- 内容 -->
    </g>
  </defs>
  
  <!-- 框架层：先用虚线框占位 -->
  <rect x="" y="" width="" height="" 
        fill="none" stroke="red" stroke-dasharray="5,3"/>
  
  <!-- 整合层：使用组件 -->
  <use href="#component" x="" y=""/>
</svg>
```

#### 2. 对话框模板
```svg
<path d="M10,0 h90 a8,8 0 0 1 8,8 v18 a8,8 0 0 1 -8,8 h-75 
         l-6,10 l-2,-10 h-7 a8,8 0 0 1 -8,-8 v-18 a8,8 0 0 1 8,-8 z" 
      fill="#fce029" stroke="#d4a00f"/>
```

#### 3. 颜色规范
| 用途 | 颜色 |
|------|------|
| 背景 | `#0d1117` → `#161b22` |
| 卡片 | `#1c2128` → `#13171d` |
| 强调文字 | `#58a6ff` |
| 小黄人 | `#fce029` |

#### 4. 布局计算公式
```javascript
// 元素位置计算
y_position = 上一个元素_y + 上一个元素高度 + 安全间距(20px)

// 示例
const titleHeight = 38 + 20; // 文字高度 + 行间距
const professorY = titleY + titleHeight + 40; // 40px 安全间距
```

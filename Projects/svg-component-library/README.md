# SVG 配图探索项目

> 从 PNG 复刻到方法论提炼，探索 SVG 在各类场景的适用边界

## 项目简介

本项目通过实际案例探索 SVG 矢量图在不同场景（PPT配图、文章配图、数据图表、流程图等）的应用，提炼可复用的方法论，明确 SVG 的适用边界。

**核心产出**：
-  6个场景 SVG 实践案例
-  可复用的 SVG 组件库（15+组件）
-  三层模型配图方法论
-  明确的适用场景边界指南

## 快速开始

### 查看效果
直接用浏览器打开任意 `.svg` 文件即可预览。

### 使用组件
```svg
<svg xmlns="http://www.w3.org/2000/svg">
  <defs>
    <!-- 引用组件库中的定义 -->
    <use href="components/minion.svg#minion"/>
  </defs>
</svg>
```

### 复刻方法
1. 阅读 `docs/svg-illustration-framework.md` 了解方法论
2. 复制 `components/layout-*.svg` 作为布局框架
3. 调整位置和样式
4. 使用 `docs/SVG配图检查清单.md` 验证

## 核心方法论：三层模型

```
┌─────────────────────────────────────────┐
│           SVG 配图三层模型               │
├─────────────────────────────────────────┤
│  第一层：框架层（Layout）                │
│  • 确定画布尺寸和网格系统                 │
│  • 划分功能区域                          │
│  • 用虚线框占位验证                       │
├─────────────────────────────────────────┤
│  第二层：组件层（Components）            │
│  • 识别可复用元素                         │
│  • 在 <defs> 中定义                       │
│  • 独立文件调试                          │
├─────────────────────────────────────────┤
│  第三层：样式层（Style）                 │
│  • 统一配色方案                          │
│  • 字体和排版规范                         │
│  • 视觉层次（大小、颜色、对比度）          │
└─────────────────────────────────────────┘
```

## 场景适用指南

| 场景 | 推荐工具 | 说明 |
|------|----------|------|
| **技术架构图** |  SVG | 适合，可控性强 |
| **PPT 配图** |  SVG | 适合，可导出高清 |
| **文章配图** |  SVG | 适合，矢量无损 |
| **时间线** |  SVG | 适合，布局清晰 |
| **数据图表** | ⚠️ ECharts/D3 | 简单图表可用SVG，复杂建议用专业库 |
| **流程图** |  Graphviz | **不适合SVG**，Graphviz自动布局更优 |
| **时序图** |  PlantUML | **不适合SVG**，PlantUML/Mermaid更合适 |

### 选择建议

**使用 SVG 的情况**：
- 需要精确控制视觉样式
- 静态展示，无复杂交互
- 作为其他工具的输出载体（如 Graphviz 导出 SVG）

**不使用 SVG 的情况**：
- 复杂数据可视化（用 ECharts/D3）
- 复杂流程/架构图（用 Graphviz/PlantUML）
- 需要交互（用 D3.js + React/Vue）

## 经验教训

###  成功经验

1. **组件化设计**
   - 使用 `<defs>` + `<use>` 复用元素
   - 12个小黄人只定义一次，代码量减少30%

2. **三层模型**
   - 先框架、再组件、最后样式
   - 有效避免返工

3. **分文件调试**
   - 复杂角色单独调试
   - 便于对比原图修改

###  失败教训

1. **流程图不适合SVG**
   - 坐标计算复杂，极易出错
   - Graphviz 已解决自动布局问题
   - **结论**：不要重复造轮子

2. **缺乏检查清单**
   - 初期没有发布前验证流程
   - 导致多个重叠、方向错误等问题
   - **改进**：建立强制检查清单

3. **负数坐标问题**
   - Q4标签使用 y="-10"
   - 虽然显示但非最佳实践
   - **规范**：保持坐标为正

## 检查清单（发布前必检）

### 逻辑正确性
- [ ] 连接线指向正确目标
- [ ] 架构关系符合实际
- [ ] 图标方向正确

### 视觉正确性
- [ ] 无重叠元素（计算边界框）
- [ ] 半透明元素 opacity ≥ 0.3
- [ ] 相邻元素间距 ≥ 15px
- [ ] 文字与背景对比度足够

### 代码正确性
- [ ] 无全角括号（）
- [ ] 坐标值为正
- [ ] viewBox 与 width/height 匹配

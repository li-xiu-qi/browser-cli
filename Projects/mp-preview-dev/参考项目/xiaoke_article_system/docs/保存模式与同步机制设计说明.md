# 保存模式与最近修改同步机制设计说明

## 1. 概述

本系统的前端编辑器支持两种保存模式：**自动保存（auto）** 和 **最近修改同步（sync）**。用户可在界面切换保存模式，系统会根据不同模式自动处理本地与后端的数据一致性。

## 2. 保存模式说明

### 2.1 自动保存模式（auto）

- 用户编辑内容后，0.5 秒内无操作会自动将内容保存到后端。
- 自动保存成功后，前端会同步后端返回的 mtime（最后修改时间）。
- 用户也可随时手动保存。

### 2.2 最近修改同步模式（sync）

- 系统每 0.5 秒自动轮询当前文件的本地和后端内容。
- 比较本地和后端的 mtime（最后修改时间），谁新用谁：
  - **后端新**：用后端内容覆盖本地缓存和编辑器内容。
  - **本地新**：自动推送本地内容到后端。
- 这样可防止多端编辑冲突，保证数据一致性。

## 3. 关键实现位置

- 主要逻辑集中在 `frontend/src/hooks/useFileOperations.js`：
  - `useFileOperations(saveMode = 'sync')` 接收保存模式参数。
  - 自动保存和最近修改同步的核心逻辑均在该 Hook 内部通过 `useEffect` 实现。
- 保存模式的状态管理和切换在 `frontend/src/App.jsx`，并通过 localStorage 持久化。

## 4. 关键代码片段

### 4.1 自动保存

```js
useEffect(() => {
  if (!currentFile || !markdown) return;
  if (saveMode === 'auto') {
    const autoSaveTimer = setTimeout(async () => {
      // ...保存到后端逻辑...
    }, 500);
    return () => clearTimeout(autoSaveTimer);
  }
}, [markdown, currentFile, saveMode]);
```

### 4.2 最近修改同步

```js
useEffect(() => {
  if (saveMode !== 'sync' || !currentFile) return;
  const interval = setInterval(async () => {
    // ...获取本地和后端mtime...
    if (remoteMtime > localMtime) {
      setFileContent(currentFile, data.content, remoteMtime);
      setMarkdown(data.content);
    } else if (localMtime > remoteMtime) {
      // ...推送本地内容到后端...
    }
  }, 500);
  return () => clearInterval(interval);
}, [currentFile, saveMode]);
```

## 5. 设计优势

- **数据安全**：避免编辑丢失，保证本地和后端内容一致。
- **用户体验好**：无需手动同步，自动保持最新。
- **灵活切换**：支持自动保存和同步两种模式，满足不同用户需求。

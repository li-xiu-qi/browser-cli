# 图片上传服务配置工具使用教程

本教程介绍如何使用 `backend/configure-upload.js` 工具快速切换项目的图片上传服务提供商。

## 工具简介

`configure-upload.js` 是一个用于配置图片上传服务的命令行工具，支持在“阿里云服务”和“本地存储”之间一键切换。该工具会自动修改 `.env.local` 文件中的相关环境变量，无需手动编辑配置。

## 支持的上传服务提供商

- **alicloud**：阿里云图片上传服务（api.alitest.yiyongcad.com）
- **local**：本地存储（./public/uploads）

## 使用方法

1. 打开命令行，进入 `backend` 目录：

   ```cmd
   cd backend
   ```

2. 执行以下命令切换上传服务：

   - 切换到阿里云服务：
     ```cmd
     node configure-upload.js alicloud
     ```
   - 切换到本地存储：
     ```cmd
     node configure-upload.js local
     ```

   执行后会看到类似如下提示：
   ```
   ✅ 已切换到 本地存储 (local)
   ✅ 已切换到 阿里云服务 (api.alitest.yiyongcad.com) (alicloud)
   ```

3. 查看当前配置：

   ```cmd
   node configure-upload.js status
   ```
   会输出当前使用的图片上传服务。

4. 查看帮助信息：

   ```cmd
   node configure-upload.js help
   ```

## 配置文件说明

- 工具会自动在 `backend` 目录下生成或更新 `.env.local` 文件。
- 切换服务时，会自动写入对应的环境变量，例如：
  - 阿里云：
    ```env
    IMAGE_UPLOAD_PROVIDER=alicloud
    ALI_UPLOAD_URL=https://api.alitest.yiyongcad.com/v1/image/upload
    ```
  - 本地存储：
    ```env
    IMAGE_UPLOAD_PROVIDER=local
    LOCAL_UPLOAD_DIR=./public/uploads
    ```

## 注意事项

- 切换服务后，前端/后端如有依赖环境变量，需重启服务以使配置生效。
- 仅支持 `alicloud` 和 `local` 两种模式，如需扩展请修改 `configure-upload.js`。

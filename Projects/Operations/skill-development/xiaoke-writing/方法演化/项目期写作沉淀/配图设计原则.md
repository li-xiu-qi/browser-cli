# 配图设计原则（内容创作与写书）

> 配图是内容的「翻译器」——把抽象文字转化为直观认知。

---

## 一、核心原则：3C 法则

### 1. Clarity（清晰）
- **一眼能懂**：读者不需要解释就能理解配图传达的信息
- **去除噪音**：删除与核心信息无关的装饰元素
- **层次分明**：主要信息 > 次要信息 > 背景

### 2. Consistency（一致）
- **风格统一**：同一篇文章/书籍的配图保持视觉风格一致
- **配色协调**：建立配色系统，避免每图一色
- **语言统一**：图标、箭头、线型风格统一

### 3. Constraint（克制）
- **一图一义**：一张图只传达一个核心观点
- **能省则省**：不是每个概念都需要配图
- **留白呼吸**：元素之间保持适当间距

## 二、不同载体的配图策略

### 公众号文章配图

| 类型 | 特点 | 建议 |
|------|------|------|
| **头图** | 决定点击率 | 高对比、强情绪、有悬念 |
| **概念图** | 解释抽象概念 | 简洁符号化，避免复杂 |
| **数据图** | 增强说服力 | 突出关键数字，简化坐标 |
| **流程图** | 步骤说明 | 不超过5步，线性展示 |
| **金句卡** | 便于转发 | 大字号，品牌色，留白多 |

**公众号特殊考虑**：
- 移动端阅读，宽度 900px 足够
- 深色模式适配，避免纯白背景
- 加载速度，SVG 优于 PNG

### 书籍配图

| 类型 | 作用 | 设计要点 |
|------|------|----------|
| **章节头图** | 承上启下 | 抽象概念，情绪渲染 |
| **概念图解** | 系统讲解 | 信息完整，标注清晰 |
| **案例图示** | 实例说明 | 真实感，细节丰富 |
| **数据图表** | 论证支撑 | 专业精确，来源标注 |
| **思维导图** | 知识结构 | 层级清晰，便于记忆 |

**书籍特殊考虑**：
- 印刷版与电子版统一
- 黑白打印测试（灰度对比）
- 分辨率要求（印刷 ≥300dpi）

### 产品宣传海报

| 元素 | 作用 | 设计要点 |
|------|------|----------|
| **主标题** | 抓眼球、传达核心价值 | 大字报风格，7字以内最佳 |
| **卖点卡片** | 分点展示功能亮点 | 3-5个，便签/对话框形式 |
| **成果数据** | 建立可信度 | 数字化呈现，对比强烈 |
| **行动召唤** | 引导下一步 | 明确、可执行 |

**海报设计公式**：
```
[一句话成果] + [3-5个卖点] + [可信背书] = 高转化海报
```

**示例**："不懂代码的人，7天做出一个能上架的App"
- 主标题：降低门槛（"不懂代码"）+ 成果量化（"7天"）+ 具体产出（"上架App"）
- 卖点卡片：用便签分散布局，每个卡片一个功能点
- 成果数据："一个人 × 七天 × 零代码 = 完整iOS应用"

## 三、设计风格类型

### 拼贴/手账风格（Collage/Scrapbook）

**适用场景**：个人项目展示、轻松话题、创意产品、降低专业门槛感

**视觉元素**：
- 胶带、撕纸边缘、回形针、夹子
- 便签纸、便利贴、对话框
- 手写体、手绘图标
- 网格纸、牛皮纸背景

**设计优势**：
- **亲切感**：打破科技产品的冷峻感，显得有温度
- **可信度**：暗示"这是我亲手做的"，不是大公司流水线产品
- **低门槛**：与"零代码""普通人能做"等主题天然契合

**注意事项**：
- 装饰元素需与主题相关（如春节主题用红包、烟花元素）
- 避免过度堆砌，保持信息可读性
- 统一手账元素的"手绘精度"（不要有的很精致、有的很潦草）

### 扁平插画风格
- 简洁几何形状，无阴影或微阴影
- 适合概念图解、流程说明
- 强调色彩区块而非细节纹理

### 写实/截图风格
- 真实产品界面、数据截图
- 增强可信度，适合案例展示
- 注意脱敏处理（隐藏敏感信息）

## 四、配图的「四不原则」

### 1. 不重复文字
❌ 配图只是文字的重复（如：文字说"三步流程"，图里还是写"第一步、第二步、第三步"）
✅ 配图补充文字无法表达的信息（如：三步之间的依赖关系、时间占比）

### 2. 不增加负担
❌ 复杂的图例说明、需要学习的视觉语言
✅ 自解释设计，所见即所得

### 3. 不虚构数据
❌ 没有来源的统计数字、虚构的对比结果
✅ 明确数据来源，或仅展示定性关系

### 4. 不追求酷炫
❌ 3D效果、过度渐变、复杂动画
✅ 扁平化、清晰、易于理解

## 五、信息可视化层级

```
第一层：核心观点（第一眼看到）
    ↓ 最大字号 / 最高对比 / 中心位置
    
第二层：支撑信息（第二眼看到）
    ↓ 中等字号 / 次要颜色 / 环绕布局
    
第三层：细节补充（主动寻找时看到）
    ↓ 小字号 / 灰色 / 边缘位置
    
第四层：装饰元素（可忽略的）
    ↓ 低对比 / 背景融合
```

**渐进披露原则**：
- 读者应该能选择看多少
- 简单场景一眼懂，复杂场景可以深入

### 分散式信息卡片布局

将多个独立信息点以"卡片"形式分散布局，而非线性堆叠：

```
┌─────────────────────────────┐
│        主标题区域            │
└─────────────────────────────┘
   ┌──────┐      ┌──────┐
   │卡片1 │      │卡片2 │
   └──────┘      └──────┘
         ┌──────┐
         │卡片3 │
         └──────┘
      ┌─────────────────┐
      │     总结区       │
      └─────────────────┘
```

**适用场景**：
- 产品功能卖点展示
- 多步骤流程（非严格顺序）
- 对比多个选项/方案

**设计要点**：
- 每张卡片一个核心观点
- 通过颜色/大小区分优先级
- 卡片间保持视觉关联（统一风格或连接线）
- 便签旋转 2-5° 增加手作感，但不宜超过 15°

## 六、配色心理学

### 情绪映射

| 颜色 | 情绪 | 适用场景 |
|------|------|----------|
| 蓝色 | 理性、信任、专业 | 技术讲解、数据分析 |
| 绿色 | 成功、增长、安全 | 正确做法、正向案例 |
| 橙色 | 活力、警示、温暖 | 重点提示、旧思维/错误 |
| 红色 | 危险、错误、紧急 | 错误做法、风险警示 |
| 紫色 | 创新、高端、神秘 | 新概念、未来展望 |
| 灰色 | 中性、次要、背景 | 辅助信息、减弱强调 |

### 氛围配色（情绪渲染）

| 场景 | 主色 | 辅助色 | 效果 | 适用内容 |
|------|------|--------|------|----------|
| **春节/喜庆** | 中国红 #C8102E | 暖黄、金色 | 热闹、节日感 | 年度总结、产品发布 |
| **温暖/亲切** | 暖黄 #F5E6C8 | 米色、棕色 | 手作、人性化 | 个人故事、教程分享 |
| **清爽/科技** | 薄荷绿 #98D8C8 | 白色、浅灰 | 简洁、未来感 | 新产品、技术概念 |
| **沉稳/专业** | 深蓝 #1E3A5F | 浅蓝、银灰 | 可靠、权威 | 行业分析、数据报告 |
| **活力/年轻** | 亮橙 #FF6B35 | 柠檬黄、草绿 | 动感、创意 | 活动宣传、创新项目 |

**氛围配色原则**：
- 先确定情绪关键词（喜庆？专业？亲切？），再选色
- 主色不超过 60%，辅助色 30%，点缀色 10%
- 从真实场景取色（如春节配色的红包、春联、金元宝）

### 对比 vs 协调

- **对比**：用于区分、警示、选择（旧 vs 新）
- **协调**：用于归属、关联、流程（步骤1 → 步骤2）

## 七、B端架构图设计范式

> 参考案例：数据+AI算法赋能项目架构图

### 7.1 设计公式

```
B端架构图 = 单色渐变主色 + 单一强调色 + 三段式布局 + 立体容器 + 流向箭头
```

### 7.2 配色策略：60-30-10 法则

| 比例 | 用途 | 具体应用 |
|------|------|----------|
| **60%** | 主色 | 蓝色系渐变（#4A90E2 → #1E3A5F），构建专业基调 |
| **30%** | 辅助色 | 浅蓝、白色、灰蓝，用于背景和次级元素 |
| **10%** | 强调色 | 橙色（#F5A623），仅用于关键标签和标题 |

**效果**：专业感强，视觉统一，重点突出

### 7.3 布局策略：F型三段式

```
┌─────────────────────────────────────────────────────┐
│  左侧：功能模块（输入层）                              │
│  ┌────┐ ┌────┐ ┌────┐                                │
│  │模块1│ │模块2│ │模块3│  ← 文件夹图标，6个功能入口      │
│  └────┘ └────┘ └────┘                                │
├─────────────────────────────────────────────────────┤
│  中间：数据源（处理层）                                │
│       ┌─────────┐     ┌─────────┐                    │
│       │ 交易市场 │ ←→ │ 施工现场 │  ← 场景插画          │
│       └─────────┘     └─────────┘                    │
├─────────────────────────────────────────────────────┤
│  右侧：数据仓库（输出层）                              │
│       ┌─────────────────────────────────┐            │
│       │    3D立体柱状图（项目/企业/行业）  │  ← 透视感 │
│       │    分层展示数据分类               │            │
│       └─────────────────────────────────┘            │
└─────────────────────────────────────────────────────┘
         ↓ 底部弧形装饰 + 主标题
```

**布局要点**：
- **左 → 右**：符合中文阅读习惯
- **近大远小**：右侧数据库用 3D 立体效果，暗示"容量"和"层级"
- **流向可视化**：箭头连接线 + 处理标签（清洗/汇算/聚类）

### 7.4 信息层级：四阶递进

| 层级 | 视觉处理 | 内容类型 |
|------|----------|----------|
| **第一阶** | 最大字号、强调色、中心位置 | 主标题：数据+AI算法赋能项目 |
| **第二阶** | 中等字号、白色、卡片背景 | 模块名称：交易市场、施工现场 |
| **第三阶** | 小字号、深蓝、列表形式 | 子项内容：具体功能点 |
| **第四阶** | 最低对比、背景融合 | 装饰元素：弧形底纹 |

### 7.5 细节处理清单

- [ ] **图标化**：左侧功能用文件夹图标，降低认知成本
- [ ] **标签微交互**：橙色标签带轻微阴影，暗示"可点击"
- [ ] **呼吸感**：模块间距统一（建议 16-24px），避免拥挤
- [ ] **立体感**：右侧数据库用渐变 + 透视营造 3D 效果
- [ ] **流向清晰**：箭头带标签，明确数据流向和处理步骤

### 7.6 可复用模式

| 场景 | 套用公式 |
|------|----------|
| 产品架构图 | 三段式（输入→处理→输出）+ 单色渐变 + 强调色点缀 |
| 数据流转图 | F型布局 + 箭头流向 + 处理节点标签 |
| 系统模块图 | 左侧功能列表 + 右侧立体容器 + 底部主标题 |

## 八、技术实现原则

### 格式选择

| 场景 | 推荐格式 | 理由 |
|------|----------|------|
| Web/公众号 | SVG | 矢量、可缩放、体积小 |
| 印刷书籍 | PDF/EPS | 高精度、CMYK支持 |
| 数据图表 | SVG + ECharts | 数据驱动、交互友好 |
| 照片/截图 | WebP/PNG | 真实感、细节丰富 |

### 响应式原则

- **最小可读**：在 320px 宽度下仍能看清
- **最大有效**：在 1920px 宽度下不模糊
- **打印友好**：黑白模式仍有足够对比度

## 八、审核清单

### 内容层
- [ ] 配图是否传达了文字未表达的信息？
- [ ] 是否有虚构/未标注来源的数据？
- [ ] 核心观点是否在 3 秒内能被捕捉？

### 视觉层
- [ ] 配色是否与文章整体风格一致？
- [ ] 文字与背景对比度是否足够？
- [ ] 移动端查看是否清晰？

### 技术层
- [ ] 文件大小是否合理（Web < 200KB）？
- [ ] 是否支持深色模式？
- [ ] 字体是否已转曲（避免字体缺失）？

## 九、常见问题

**Q：一篇文章配几张图合适？**
A：取决于内容密度。一般建议：
- 1500 字以下：1-2 张
- 1500-3000 字：2-4 张  
- 3000 字以上：每 800-1000 字 1 张

**Q：复杂系统图怎么做？**
A：分而治之：
1. 总体架构图（鸟瞰）
2. 分模块详图（特写）
3. 流程图串联（关系）

**Q：没有设计基础怎么办？**
A：从模仿开始：
1. 收集优秀案例（Pinterest、Behance）
2. 拆解配色、布局、字体
3. 建立个人组件库

# 写作规范版本对比

这个文件只回答一件事：

每个版本相对上一版，到底新增了什么，重点改了什么，当前适合拿来干什么。

如果你只想快速选版本，先看最后一列。

---

## 当前结论

- 当前最细、最适合继续使用的版本是 [v9-写作规范.md](./v9-写作规范.md)
- `v9` 不是推翻 `v8`，而是在 `v8` 的主线上继续补“怪词”这层句子自然度问题
- 这条线现在可以粗分成 4 个阶段：
  - `v0-v1`：技术博客转公众号的早期改写框架
  - `v2-v4`：先价值、再标题、再结构的主线成型
  - `v5-v6`：节奏和硬检查并入主线
  - `v7`：去假力度，处理评委腔、空心对比句和主体缺失
  - `v8`：防止替读者下定义，把“去掉 AI 味”推进到对象定义权
  - `v9`：清理怪词，把“去掉 AI 味”推进到句子自然度和判断落点

## 版本表

| 版本 | 当前定位 | 相对上一版主要变化 | 这一版重点解决什么问题 | 现在怎么看 |
|---|---|---|---|---|
| `v0` | 最早期探索版 | 从“理解原文 -> 搭骨架 -> 转公众号”出发，先把技术文档改写流程写出来 | 解决最早期的“技术内容怎么转成公众号文章” | 历史起点，不建议直接复用 |
| `v1` | `v0` 的整理版 | 语言和结构更规整，把一些早期提示改成更可执行的说明 | 让最初的改写流程更像一份可操作规范 | 仍然偏早期 |
| `v2` | 主线定型版 | 从“深度理解原文”切到“先判断文章卖什么价值”，明确 4 个起手问题 | 解决“文章改得很完整，但卖点不直给” | 是后面几版的真正基线 |
| `v4` | `v2` 的成熟展开版 | 保留 `v2` 主线，同时把文章类型、标题变量、骨架判断写得更完整 | 解决“知道先看价值，但不知道怎么把价值落到结构和标题” | 还是有参考价值，但不再是当前主入口 |
| `v5` | 硬检查增强版 | 在主线上加入一组更硬的表达检查：机械对比句、分点过多、整篇纯段落、破折号滥用、表情、蹩脚比喻 | 解决“结构对了，但读起来还是很像 AI 生成物” | 是从“方法”走向“质检”的关键节点 |
| `v6` | 合并版主线 | 把原先拆开的“价值优先”和“节奏优先”并回同一份规范，形成“先价值、再节奏、最后硬检查” | 解决“规则分叉太多，使用入口不清” | 是当前主线的结构基座 |
| `v7` | 句子层校正版 | 在 `v6` 基础上补“评委腔 / 空心对比句 / 主体缺失 / 主体 + 对象 + 判断”这组表达规则 | 解决“文章看上去更利落了，但句子还在替读者打分、替读者下定义、靠口气硬撑” | 句子层关键版本 |
| `v8` | 对象定义权校正版 | 在 `v7` 基础上补“不要替读者想 / 模型先补可解释对象 / 已知-推测-待确认”这组规则 | 解决“句子已经没那么假了，但解释还在抢跑、还在替对象下定义” | 上一版关键版本 |
| `v9` | 怪词清理版 | 在 `v8` 基础上补“怪词识别 / 对象-动作-关系补全 / 直接判断句替换”这组规则 | 解决“句子方向对了，但读起来还是发空、发飘、像在用词味撑力度” | 当前默认版本 |

## 关键分水岭

### 从 `v1` 到 `v2`

这里不是小修，而是整条线换了起点。

`v1` 还更像“先理解原文，再想怎么改写”。

`v2` 开始，真正的起点变成：

- 这篇文章卖什么价值
- 这篇文章写给谁
- 读者在什么场景下会打开
- 看完后应该带走什么

也就是说，写作规范开始从“内容处理流程”转向“读者价值设计”。

### 从 `v4` 到 `v5`

这里开始，规范不再只讲方法，也开始讲清哪些句子和排版动作会把文章带偏。

`v5` 的关键不是多了几条禁令，而是开始明确：

- 不要靠机械的 `不是……而是……` 推进结构
- 不要靠破折号制造假节奏
- 不要在“全篇列表”和“全篇纯段落”之间来回摆

这意味着规范开始真正碰“AI 味”的表层症状。

### 从 `v5` 到 `v6`

这里的核心是收口，不是扩张。

`v6` 把之前拆开的“价值优先”和“节奏优先”合回同一条线。以后不需要先找一份规范，再去找另一份节奏补充。

这一步解决的是：

- 入口太散
- 主线太多
- 用的人不知道当前到底该看哪一份

### 从 `v6` 到 `v7`

这里补的是句子层真正容易复发的问题。

`v6` 已经把节奏收回主线，但还没有把下面这组问题钉死：

- `最抓人的一句话`
- `最值得看的`
- `最有冲击力的`
- `不是……而是……`
- `不在……而在……`

`v7` 的价值在于，它不再只说“少用这些句子”，而是把底层原因写出来了：

- 主体不清楚
- 判断标准不清楚
- 句子在靠语气制造力度
- 对比骨架没有真的补新信息

也因为这个变化，`v7` 才开始更接近“去掉 AI 味”的深层版本。

如果说 `v5` 主要在清理表层症状，`v7` 已经开始碰写作姿态本身。

### 从 `v7` 到 `v8`

这里补的是 `v7` 还没直接写进去的那层：

- 为什么模型和写作者都容易先补一个“典型读者”或“真正问题”
- 为什么这种倾向会滑成代入式权威
- 怎么把这类问题拦在写作前

`v7` 主要在清理句子里的假力度。

`v8` 继续处理解释里的抢跑定义。

### 从 `v8` 到 `v9`

这里补的是 `v8` 之后很容易暴露出来的一层：

- 句子不一定在替读者下定义
- 也不一定在写评委腔
- 但还是读起来别扭

这类问题经常不是逻辑错了，而是句子在用怪词撑判断。

典型情况是：

- 词有气氛
- 关系不够清楚
- 句子像文章
- 但不像人会真的这么说

`v9` 的价值在于，它把这组问题正式写进主线，不再只停留在单篇文章反馈里。

这一版重点补的是：

- 什么叫怪词
- 为什么怪词的问题不只是“不自然”
- 怎么把句子改回对象、动作、关系清楚的直接判断句

这意味着规范开始不只问：

- 这句话像不像 AI

也开始问：

- 这个对象是谁先定义的
- 这个判断是不是抢在事实前面
- 作者有没有顺手拿走读者或用户的自我定义权

## 关于“去掉 AI 味”和 `v8` 的关系

如果你问得更直接一点：

`v8` 算不算已经在吸收“去掉 AI 味”这条线？

我的判断是：

- 算，而且比 `v7` 更完整
- 它现在已经不只在句子层处理假力度，也开始在解释层处理“替对象下定义”

目前已经明确写进 `v8` 的，是：

- 评委腔
- 空心对比句
- 主体缺失
- `主体 + 对象 + 判断`
- `最吸引人` 这类看上去变软、实际还是悬空的句子
- 不要替读者想
- 不要把用户理解做成用户定义
- 模型先补一个可解释对象，再围绕它补完整解释
- 输出前先拆开 `已知 / 推测 / 待确认`

也就是说，`v8` 已经不只是和这条线同向，而是开始把这条线的核心判断正式并进规范了。

继续往后升版本，一个自然方向会是：

- 把这组规则继续往标题、小标题和开头首屏单独拆成一套检查表

## 专题补充线

除了主线版本，还单独拆出了专题文档：

- [写作规范-我的视角-v1.md](./写作规范-我的视角-v1.md)
- [写作规范-我的视角-v2.md](./写作规范-我的视角-v2.md)

它们不替代主线版本。

解决的是另一件更窄、但很高频的问题：

- 当文章价值的一部分来自作者真实判断变化时，怎么把“我的视角”写对

**v1 到 v2 的变化**：

- `v1` 主要覆盖思考类、判断型、对谈重编排、方法论长文
- `v2` 在 `v1` 基础上新增“写法四：我原来怎么判断 -> 查资料后哪里被改了”，明确把**对比/汇总/整理类文章**纳入适用范围（前提是作者有真实认知变化）

## 选版本建议

| 你的需求 | 更适合看哪一版 |
|---|---|
| 想看这条线最早是怎么起步的 | `v0` 或 `v1` |
| 想理解为什么现在总强调“先判断价值” | `v2` |
| 想看主线比较完整的中期版本 | `v4` |
| 想看硬检查是怎么被引进来的 | `v5` |
| 想看节奏为什么并回主线 | `v6` |
| 想看句子层假力度是怎么被处理的 | `v7` |
| 想直接拿当前最实用的方法 | `v9` |
| 文章价值一部分来自作者判断变化，想补“我的视角”专题规则 | `写作规范-我的视角-v2`（对比/汇总类看 v2，其他看 v1 亦可） |

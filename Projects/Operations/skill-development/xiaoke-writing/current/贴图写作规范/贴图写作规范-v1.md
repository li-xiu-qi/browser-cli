# 贴图写作规范-v1

## 定位

本规范用于公众号「图片消息」（贴图/多图文）的写作。

与长文不同，贴图有严格的形态限制：字数极少、纯文本、无排版。需要一套专门的写作方法。

## 贴图的核心限制

| 维度 | 具体限制 |
|------|----------|
| 单图字数 | 50-150字 |
| 总字数 | 600-1000字（8-10张图）|
| 标题字数 | **不超过20个字符**（关键限制）|
| 排版 | 纯文本，不支持markdown语法 |
| 列表 | 不可用（1. 2. 3. 或 - 都会显示异常）|
| 表情 | 不建议使用 |
| 大标题 | 不可用（# ## 等语法无效）|

## 贴图 vs 其他形态

| 对比项 | 长文 | 小红书图文 | 公众号贴图 |
|--------|------|-----------|-----------|
| 单页字数 | 不限 | 30-80字 | 50-150字 |
| 排版能力 | 强 | 中 | 无 |
| 列表格式 | 可用 | 可用 | **不可用** |
| 阅读方式 | 连续阅读 | 图文跳读 | 滑动浏览 |
| 标题限制 | 64字 | 20字 | **20字** |
| emoji | 可用 | 推荐 | 不建议 |

## 核心写作原则

### 1. 标题必须20字以内

这是硬限制，超过会被截断。

不稳：飞书CLI开源了，200+命令覆盖11个场景，OpenClaw用户有福了！（34字）

更稳：飞书CLI开源，200+命令11场景，claw有福（20字）

精简技巧：
- 去掉"了"、"的"等助词
- 数字+单位合并（"11个场景"→"11场景"）
- 用简称（"OpenClaw"→"claw"）
- 去掉感叹号、句号等标点

### 2. 段落式表达，不用列表

贴图是纯文本环境，列表符号（1. 2. 3. / - / *）会显示异常或占字符。

不稳：
```
安装很简单：
1. npm install
2. 配置凭证
3. 登录授权
```

更稳：
```
安装很简单，npm install一行命令。然后三步搞定：配置凭证、登录授权、开始使用。
```

### 3. 每页独立成段

读者可能从中间任何一张图开始看。每张图的文字要能独立理解大意。

### 4. 松散串联，允许跳读

图与图之间是松散的关联，不是严密的逻辑链。不要写"如前所述""接上页"等依赖前文的内容。

### 5. 第一张图必须交代视角

读者决定要不要继续滑，全靠第一张图。必须在第一句话交代"我是怎么发现这个东西的"。

不稳：今天给大家分享一个好东西。

更稳：刷GitHub的时候注意到一个数据。

### 6. 最后一张图必须留互动

贴图不支持超链接，读者的下一步通常是评论。最后要抛一个具体问题。

示例：
- 你最想用CLI实现什么功能？
- 你之前用过类似的工具吗？
- 哪个功能你最感兴趣？

## 单图结构模板

标准8图结构：

| 图序 | 功能 | 字数 | 内容要点 |
|------|------|------|----------|
| 01 | 封面/钩子 | 60-80字 | 我怎么发现这个东西的 |
| 02 | 核心亮点1 | 80-100字 | 最打动我的点 |
| 03 | 核心亮点2 | 80-100字 | 第二个关键价值 |
| 04 | 使用/安装 | 70-90字 | 怎么上手 |
| 05 | 我的用法 | 80-100字 | 我自己怎么用 |
| 06 | 更多场景 | 80-100字 | 其他用法 |
| 07 | 资源/总结 | 60-80字 | 去哪找更多信息 |
| 08 | 结尾互动 | 40-60字 | 抛问题引评论 |

## 语言风格

### 短句制造节奏

长句拆短，用句号和换行制造呼吸感。

不稳：这不是普通的CLI工具，它是专门为AI Agent设计的，自带19个AI技能，200多个命令，覆盖日历、消息、文档、表格、邮件、会议等11个业务场景。

更稳：这不是普通的CLI工具。它是为AI Agent设计的，自带19个AI技能，200多个命令，覆盖日历、消息、文档等11个场景。

### 口语化但有边界

可用：很简单、最让我惊喜的是、我自己现在用它

避免：绝绝子、yyds、啊啊啊啊、家人们

## 容易写坏的几种方式

### 1. 超过20字标题

会被截断显示，关键信息丢失。

### 2. 使用markdown列表

在纯文本环境会显示为乱码或占字符。

### 3. 单图超过150字

字太小或堆在一起，阅读体验差。

### 4. 我的视角变成产品说明书

满篇"支持""包含""涵盖"，没有"我"的存在。

### 5. 开头没有钩子

"今天给大家分享"属于无效开场。

## 标题撰写技巧

### 精简公式

原句：飞书CLI开源了，200+命令覆盖11个场景，OpenClaw用户有福了！

精简步骤：
1. 去掉"了"→飞书CLI开源
2. 去掉"覆盖"→200+命令11场景
3. 用简称→claw有福

结果：飞书CLI开源，200+命令11场景，claw有福（20字）

### 标题类型

| 类型 | 示例 | 适用场景 |
|------|------|----------|
| 数据型 | 飞书CLI开源，200+命令11场景 | 工具介绍、产品推荐 |
| 对比型 | 以前写代码，现在一行命令 | 效率提升、新旧对比 |
| 利益型 | claw用户有福了 | 蹭热点、精准人群 |
| 悬念型 | 字节开源了这个 | 新闻速递、突发事件 |

## 最后检查清单

- [ ] 标题不超过20个字符
- [ ] 单图字数50-150字
- [ ] 无markdown语法（# ## - 1. 等）
- [ ] 无emoji
- [ ] 段落式表达，无列表符号
- [ ] 第一张图交代"我怎么发现的"
- [ ] 每张图能独立理解
- [ ] 最后一张图抛问题引互动
- [ ] 总字数600-1000字

## 一句话收口

公众号贴图是在20字标题+纯文本的限制下，用800字分散在8张图里，每张图独立成段，靠内容本身传递价值。
